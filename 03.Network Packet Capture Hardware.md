# Network Packet Capture Hardware

## 1. Core Device (Compute / IoT Platform)

### Raspberry Pi (your main controller)

#### Why it matters:
- Acts as your **sniffer box / attack box / IoT controller**
- Runs tools like:
  - Wireshark / tcpdump
  - Bettercap
  - Tshark
- Has **Gigabit Ethernet + Wi-Fi + GPIO (Raspberry Pi)**

#### Use cases:
- Portable network sniffer
- IoT honeypot
- Packet logging device
- DIY network TAP (with limitations)

👉 Good for: **Beginners / budget labs / field testing**

---

## 2. Network TAP Devices (Passive Monitoring)

These are **professional-grade tools** used to capture traffic **without modifying it**.

---

### Dualcomm ETAP-2003 (Budget Passive TAP)

#### Key points:
- Supports **10/100/1000 Mbps**
- Passive (**no packet injection**)
- Reliable for **home lab + small enterprise**

👉 Best for: **students + practical labs**

---

### ProfiShark (High-End USB TAP)

#### Why it's powerful:
- USB-powered → connects directly to laptop
- High precision timestamping
- Used in **forensics + enterprise debugging**

👉 Best for: **professional packet analysis**

---

### LANProbe Network TAP

- Often includes **USB output + bypass features**
- Can act as:
  - TAP
  - Inline monitoring device

👉 Best for: **hybrid lab setups**

---

## 3. Red Team / Pentesting Tool

### Shark Jack (Hak5)

- Plug-and-play attack + sniffing device
- Runs payloads automatically when plugged into network

#### Capabilities:
- DHCP attacks
- Packet capture
- Recon scripts

👉 Best for: **red team / automation / quick drops**

# What Should YOU Choose?

## 🟢 If you're learning (recommended setup)

---

- Shark Jack

# Final Insight

- **Raspberry Pi ≠ TAP**
  - It captures traffic, but cannot passively split signals like hardware TAPs

- **TAP = visibility without detection**
- **Pi = analysis + flexibility**
