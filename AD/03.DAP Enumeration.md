# LDAP Enumeration

## LDAP (Lightweight Directory Access Protocol) – Enumeration

---

## Default Ports

| Port | Service |
|-----|------|
| **389** | LDAP (unencrypted) |
| **636** | LDAP over SSL (LDAPS) |
| **3268** | Global Catalog (LDAP for Active Directory) |
| **3269** | Global Catalog over SSL (LDAPS) |

---

## What is LDAP?

**LDAP** is a **lightweight protocol** used to access and manage directory information over a network.

It is commonly used for:

- **User authentication**
- **Centralized user management**
- **Directory services like Active Directory and OpenLDAP**

> LDAP is a simplified version of the **X.500 Directory Access Protocol (DAP)** and communicates over **TCP/IP**.

---

## Common LDAP Implementations

| Platform | Description |
|---|---|
| **Microsoft Active Directory** | LDAP backend for Windows domains |
| **OpenLDAP** | Open-source implementation |
| **Novell eDirectory** | Enterprise-grade LDAP directory |

---

## Authentication & Use Cases

LDAP is most commonly used for:

- **Single Sign-On (SSO)**
- **Central authentication (usernames, passwords)**

LDAP can integrate authentication across platforms like:

- Jenkins
- Docker
- Kubernetes
- OpenVPN
- Samba
- Linux PAM

> Admins can enforce **user/group-based permissions** using LDAP groups across systems.

## Enumeration & Tools

### Goal of LDAP Enumeration

- Gather **user, group, computer, and domain information**
- Identify potential **privilege escalation paths**
- Map the **Active Directory structure**

---

## CLI Tools for LDAP Enumeration

### ldapsearch (Linux-native, part of ldap-utils)

```bash
ldapsearch -x -h <IP> -b "dc=corp,dc=local"
```

**Parameters**

| Option | Description |
|------|-------------|
| `-x` | Simple authentication |
| `-h` | LDAP server IP / hostname |
| `-b` | Base DN (distinguished name) |

Example:

```bash
ldapsearch -x -h 10.10.10.5 -b "dc=corp,dc=local"
```

---

### Nmap LDAP Scripts

```bash
nmap -p 389 --script ldap-search <target-ip>
```

Useful for quick LDAP enumeration.

---

### enum4linux / enum4linux-ng

Legacy **SMB / LDAP enumeration tool**

```bash
enum4linux-ng <target-ip>
```

# Nmap LDAP Scripts

## RootDSE Enumeration

```bash
nmap -v -p 389 --script=ldap-rootdse.nse -Pn 192.168.2.100
```


## General LDAP Search

```bash
nmap -v -p 389,636 --script=ldap-search.nse -Pn 192.168.2.100
```

```bash
nmap -v -sV -sT --script=ldap* -p 389,636 192.168.2.100
```

# Authentication Methods

## Anonymous Bind

```bash
ldapsearch -x -H ldap://192.168.2.100 -D "" -w "" -b "DC=infosecwarrior,DC=local"
```

```bash
ldapsearch -x -H ldap://10.10.10.161 -x -b "DC=htb,DC=local"
```

Dump LDAP data to a file:

```bash
ldapsearch -x -H ldap://10.10.10.161 -D "" -w "" -b "DC=htb,DC=local" > htb.local-ldapsearch.txt
```

Extract usernames:

```bash
cat htb.local-ldapsearch.txt | grep sAMAccountName
```

---

## Authenticated Bind

```bash
ldapsearch -x -H ldap://192.168.2.100 -D 'infosecwarrior\jdoe' -w 'Password123!' -b "DC=infosecwarrior,DC=local"
```

```bash
ldapsearch -x -H ldap://192.168.2.100 -D 'jdoe@infosecwarrior.local' -w 'Password123!' -b "DC=infosecwarrior,DC=local"
```

---

# LDAP Object Extraction

## Full Directory Dump (base/rootDSE)

```bash
ldapsearch -H ldap://192.168.2.100 -x -s base -b "" "(objectClass=*)" "*" +
```

---

## Over LDAPS (Encrypted)

```bash
ldapsearch -H ldaps://192.168.2.100:636 -x -s base -b "" "(objectClass=*)" "*" +
```
