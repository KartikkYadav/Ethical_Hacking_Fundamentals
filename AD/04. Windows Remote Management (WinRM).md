
# Windows Remote Management (WinRM)

### Setting up Windows Remote Management (WinRM) for Remote PowerShell Execution

WinRM ek Windows service hai jo remote PowerShell execution allow karti hai. Ye default HTTP port **5985** par run karti hai (HTTPS = 5986).

---

## Configuring WinRM

WinRM service enable aur start karne ke liye:

```bash
winrm quickconfig
```

Ye:

* WinRM service start karta hai
* HTTP listener configure karta hai (default port 5985)

---

## Ensuring the WinRM Service Starts Automatically

- 1. Service immediately start karo

```bash
Start-Service WinRM
```


- 2. Port 5985 open hai ya nahi check karo

```bash
nmap -v -n -p 5985 -sV -sC -A -T4 192.168.1.61
```

- 3. Service ko boot par automatic set karo

```bash
Set-Service -Name WinRM -StartupType Automatic
```

---

## If PowerShell Method Doesn’t Work (Force via Registry)

Agar `Set-Service` kaam na kare to registry modify karo.

- Step 1: PowerShell as Administrator open karo

- Step 2: Registry modify karo

```bash
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\WinRM" -Name "DelayedAutoStart" -Value 0
```

```bash
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\WinRM" -Name "Start" -Value 2
```

Explanation:

* DelayedAutoStart = 0 → Delayed start disable
* Start = 2 → Automatic start

- Step 3: System restart karo

```bash
Restart-Computer -Force
```

---

## Use SC Config Command (Alternative)

Agar registry method bhi kaam na kare:

- Command Prompt as Administrator open karo

```bash
sc config winrm start= auto
```

System restart ke baad verify karo:

```bash
Get-Service -Name WinRM | Select-Object Name, StartType
```

---

## Enabling PowerShell Remoting

Remote PowerShell execution enable karne ke liye:

```bash
Enable-PSRemoting -Force
```

Ye firewall exceptions aur required permissions configure karta hai.

---

## Checking WinRM Service Status

```bash
Get-Service -Name WinRM
```

```bash
Get-Service -Name WinRM | Select-Object -Property Name, StartType
```

---

## Checking Configured Listeners

Configured HTTP/HTTPS listeners check karne ke liye:

```bash
winrm enumerate winrm/config/listener
```

---

## Adding Users to "Remote Management Users" Group

User ko remote access dene ke liye group me add karo.

```bash
net localgroup "Remote Management Users" mohan /add
```

```bash
net localgroup "Remote Management Users" rohan /add
```

Current users list dekhne ke liye:

```bash
net localgroup "Remote Management Users"
```

---

## Accessing Windows Machine via WinRM (Evil-WinRM)

Credentials ke through remote access:

```bash
evil-winrm -i 192.168.1.101 -u mohan -p password@12345
```

```bash
evil-winrm -i 192.168.1.101 -u piyush -p password@12345
```

Explanation:

* -i → Target IP
* -u → Username
* -p → Password

---

## Creating a Firewall Rule for WinRM

Inbound WinRM connection ke liye port 5985 open karo:

```bash
netsh advfirewall firewall add rule name="WinRM" dir=in action=allow protocol=TCP localport=5985
```

---

## Checking WinRM Service Configuration

Current configuration dekhne ke liye:

```bash
winrm get winrm/config/service
```

---

## Configuring PowerShell Remoting Security

Security settings configure karne ke liye GUI open karo:

```bash
Set-PSSessionConfiguration -Name Microsoft.PowerShell -ShowSecurityDescriptorUI
```

---

## Use HTTPS Instead of HTTP (Recommended)

Security ke liye HTTPS listener create karo:

```bash
winrm create winrm/config/Listener?Address=*+Transport=HTTPS @{CertificateThumbprint="THUMBPRINT"}
```

---

## Verify WinRM Connectivity

Dusre system se connectivity test karo:

```bash
Test-WSMan -ComputerName TargetMachine
```

---

## Short Practical Flow (Quick Checklist)

1. winrm quickconfig
2. Enable-PSRemoting
3. Firewall rule add karo
4. Service automatic set karo
5. evil-winrm se connect karo
