
# Remote Desktop Access to a Domain User

Remote Desktop Services (RDS) allow karta hai users ko remotely kisi system ka desktop access karne ke liye.
Active Directory environment me RDP access normally Group Policy (GPO) ke through centrally manage kiya jata hai.

### Group Policy se:

* RDP enable kiya ja sakta hai
* Firewall rule configure hoti hai
* Specific users/groups ko login right diya jata hai
* Multiple systems par ek saath apply hota hai

---

### Prerequisites

1. Active Directory environment configured ho
2. Target computers domain joined ho
3. Administrative privileges ho GPO create/modify karne ke liye
4. Windows Firewall properly configured ho

---

## Step-by-Step Guide to Enable Remote Desktop via Group Policy

### Step 1: Open Group Policy Management Console

Domain Controller ya GPMC installed machine par login karo.

Run:

```bash
gpmc.msc
```

---

### Step 2: Create New GPO

1. GPMC me domain ya specific OU par right-click karo
2. Select karo:

Create a GPO in this domain, and Link it here...

3. GPO ka naam do:

Enable Remote Desktop

4. Click OK

---

### Step 3: Edit GPO to Enable Remote Desktop

Newly created GPO par right-click karo → Edit

Navigate to:

Computer Configuration → Policies → Administrative Templates → Windows Components → Remote Desktop Services → Remote Desktop Session Host → Connections

Policy open karo:

Allow users to connect remotely using Remote Desktop Services

Set karo:

Enabled → Apply → OK

---

### Step 4: Configure Network Level Authentication (Recommended)

Navigate to:

Computer Configuration → Policies → Administrative Templates → Windows Components → Remote Desktop Services → Remote Desktop Session Host → Security

Policy open karo:

Require user authentication for remote connections by using Network Level Authentication

Set karo:

Enabled

Ye RDP session start hone se pehle authentication enforce karta hai.

---

### Step 5: Allow Remote Desktop Users (Restricted Groups)

Navigate to:

Computer Configuration → Policies → Windows Settings → Security Settings → Restricted Groups

Steps:

1. Right-click Restricted Groups → Add Group
2. Group field me type karo:

Remote Desktop Users

3. Click OK
4. Under "This group is a member of" required groups add karo
5. Apply → OK

---

### Step 6: Configure Windows Firewall for RDP

Navigate to:

Computer Configuration → Policies → Windows Settings → Security Settings → Windows Defender Firewall with Advanced Security → Inbound Rules

Steps:

1. Right-click Inbound Rules → New Rule
2. Select Port → Next
3. Select TCP
4. Specify port:

3389

5. Next → Select Allow the connection
6. Finish wizard

---

### Step 7: Link GPO to Appropriate OU

Agar GPO link nahi hua hai to:

1. OU par right-click
2. Select:

Link an existing GPO

3. Select your GPO
4. Click OK

---

### Step 8: Update Group Policy on Target Computers

Target computer par Command Prompt open karo:

```bash
gpupdate /force
```

By default GPO refresh interval approx 90 minutes hota hai.

## PowerShell Automation – Grant RDP Access to Domain User

### Step 1: Add User to Remote Desktop Users Group on All Computers

```bash
$DomainUser = "infosecwarrior.local\jdoe"

$Computers = Get-ADComputer -Filter * | Select-Object -ExpandProperty Name

foreach ($Computer in $Computers) {
    Invoke-Command -ComputerName $Computer -ScriptBlock {
        param($User)
        Add-LocalGroupMember -Group "Remote Desktop Users" -Member $User -ErrorAction SilentlyContinue
    } -ArgumentList $DomainUser -Credential (Get-Credential)
}
```

Ye script:

* Saare domain computers fetch karta hai
* Har machine par Remote Desktop Users group me user add karta hai

---

### Step 2: Enable Remote Desktop on All Computers

```bash
$Computers = Get-ADComputer -Filter * | Select-Object -ExpandProperty Name

foreach ($Computer in $Computers) {
    Invoke-Command -ComputerName $Computer -ScriptBlock {
        Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server" -Name "fDenyTSConnections" -Value 0
        Enable-NetFirewallRule -DisplayGroup "Remote Desktop"
    } -Credential (Get-Credential)
}
```

Ye:

* Registry key modify karta hai
* Firewall rule enable karta hai

---

### Step 3: Grant Remote Desktop Logon Rights via GPO

```bash
$User = "YourDomain\YourUser"
$GPO = "Allow RDP Access"

New-GPO -Name $GPO -ErrorAction SilentlyContinue

Set-GPRegistryValue -Name $GPO -Key "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -ValueName "UserAuthentication" -Type DWord -Value 1

New-GPLink -Name $GPO -Target "OU=Computers,DC=YourDomain,DC=com"
```

Note:
OU=Computers,DC=YourDomain,DC=com ko apne actual OU path se replace karo.

---

### Step 4: Force Group Policy Update on All Machines

```bash
Get-ADComputer -Filter * | ForEach-Object {
    Invoke-Command -ComputerName $_.Name -ScriptBlock { gpupdate /force }
}
```

Ye domain ke sabhi systems par GPO force update karega.

---

### Final Result

Agar sab steps sahi apply hue hain to:

* RDP enabled hoga
* Firewall allow karega port 3389
* User Remote Desktop Users group me hoga
* NLA enabled hoga
* GPO properly linked hoga

Ab domain user safely RDP login kar sakta hai domain joined machines par.
