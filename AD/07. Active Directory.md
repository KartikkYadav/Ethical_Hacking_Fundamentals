
# Active Directory 

## 1. Active Directory (AD) 

Active Directory (AD) Microsoft ka directory aur identity management service hai jo Windows domain networks me use hota hai. Ye Windows 2000 me introduce hua tha aur Windows Server ke sath included hota hai.

AD ek directory service hai jo:

* Users
* Computers
* Groups
* Printers
* Network resources

ko centrally manage karta hai.

---

## 2. AD Kya Manage Karta Hai

### 1. Windows-based networks manage karta hai

Active Directory basically ek database hai jo network objects ka data store karta hai.

Centralized management deta hai:

* User accounts → Create, modify, delete
* Computer accounts → Device permissions manage
* Security policies → Group Policy ke through control
* Network resources → File share, printer, applications

---

### 2. Network Objects Store karta hai

AD hierarchical structure me objects store karta hai:

* User objects → Username, password, group, email
* Computer objects → Device name, OS version, group membership
* Group objects → Shared permissions wale users/computers
* Organizational Units (OU) → Logical containers
* Printers & Shared folders → AD managed resources

Data replicated hota hai sabhi Domain Controllers (DCs) par consistency ke liye.

LDAP (Lightweight Directory Access Protocol) use hota hai AD ko query karne ke liye.

---

### 3. Centralized Secure Management

AD allow karta hai:

* Single administrative control
* Multi-site networks manage karna
* Authentication using Kerberos
* Single Sign-On (SSO)
* Replication across Domain Controllers

Key logical components:

* Forest
* Domain
* Trust
* Sites

---

### 3. Key Components of Active Directory

| Component                | Description                                            |
| ------------------------ | ------------------------------------------------------ |
| Domain Controller (DC)   | Server jo AD database store karta hai                  |
| Forest                   | Top level container jo multiple domains hold karta hai |
| Organizational Unit (OU) | Logical container for organizing objects               |
| Global Catalog           | Sab objects ka searchable subset                       |
| Group Policy             | User aur computer settings control karta hai           |
| Schema                   | AD object structure define karta hai                   |
| LDAP                     | Directory access protocol                              |
| Kerberos                 | Authentication protocol                                |
| Replication Service      | Data sync across DCs                                   |

---

### 4. Why Active Directory Matters

* Centralized security
* Easy user management
* Large networks support
* High availability through replication

---

### 5. AD Services

Active Directory multiple services provide karta hai:

* Active Directory Domain Services (AD DS) → Core service
* Active Directory Lightweight Directory Services (AD LDS) → Application directory service
* Active Directory Certificate Services (AD CS) → Digital certificates
* Active Directory Federation Services (AD FS) → Identity federation
* Active Directory Rights Management Services (AD RMS) → Document permission control

---

### 6. AD Data Structure

AD hierarchical structure follow karta hai:

### Domain

Objects ka collection jo same AD database share karta hai
Example: company.com

### Tree

Multiple domains with contiguous namespace
Example:
marketing.company.com
engineering.company.com

### Forest

Multiple trees ka collection
Forest security boundary act karta hai

---

### 7. Active Directory Trust

Trust allow karta hai ek domain ke users ko dusre domain ke resources access karne ke liye.

Trust Types:

| Type         | Characteristics              | Direction          | Authentication      |
| ------------ | ---------------------------- | ------------------ | ------------------- |
| Parent-Child | Transitive                   | Two-way            | Kerberos V5 or NTLM |
| Tree-Root    | Transitive                   | Two-way            | Kerberos V5 or NTLM |
| Shortcut     | Transitive                   | One-way or Two-way | Kerberos V5 or NTLM |
| Forest       | Transitive                   | One-way or Two-way | Kerberos V5 or NTLM |
| External     | Non-transitive               | One-way            | NTLM only           |
| Realm        | Transitive or Non-transitive | One-way or Two-way | Kerberos V5 only    |

---

### 8. AD Benefits

* Security → Access control improve karta hai
* Extensibility → Business structure ke according organize
* Simplicity → Centralized identity management
* Resiliency → Replication for high availability

---

### 9. NTDS.dit

NTDS.dit AD ka main database file hai.

Isme store hota hai:

* User objects
* Groups
* Password hashes

Default Location:

```bash
%SystemRoot%\NTDS\NTDS.DIT
```

Database Engine:
Extensible Storage Engine (ESE)
