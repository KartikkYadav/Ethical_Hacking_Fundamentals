
# Offensive AD – Attack Methodology 

### Attack Stages

Active Directory attack methodology generally multiple stages me divide hoti hai. Har stage ka clear objective hota hai.


### 1. Reconnaissance (Recon)

Target network ke baare me intelligence gather karna.

### Recon me kya collect karte hai:

* Open Ports (tools like Nmap)
* Services running (SMB, LDAP, RDP)
* Domain Information (DNS, AD structure)
* Employee Details (OSINT via LinkedIn, Social Media)
* Technology Stack (OS version, software version)

Practical mindset:
Pehle samjho network ka exposure kya hai. Kaunse ports open hai, kaunse services vulnerable ho sakti hai.

---

### 2. Domain Enumeration (Enum)

Target AD environment ko actively query karna.

Identify karte hai:

* Domain Controllers (e.g. using nltest)
* AD Objects (users, groups, permissions)
* Trust relationships (tools like BloodHound, PowerView)
* Group Policy Objects (GPO)

Practical goal:
Low-privilege access se maximum information extract karna.
Yahi stage me future privilege escalation paths milte hain.

---

### 3. Local Privilege Escalation

Compromised machine par higher privilege gain karna.

Common techniques:

* Exploiting misconfigurations (weak service permissions)
* Kernel exploits (token stealing, memory corruption)
* Stealing credentials from memory (e.g. mimikatz)
* DLL hijacking

Practical mindset:
Agar tumhare paas normal user access hai to target hota hai Local Admin ya SYSTEM banna.

---

### 4. Admin Recon (Administrative Reconnaissance)

High-value targets identify karna.

Target objects:

* Domain Admins
* Service accounts (especially SPNs wale)
* Group Policy Objects

Mapping:

* Administrative control paths
* Privilege escalation routes
* Trust relationships

Practical goal:
Kaun se users se Domain Admin tak pahunch sakte ho, ye identify karna.

---

### 5. Lateral Movement

Ek machine se dusri machine par move karna.

Techniques:

* Exploiting SMB, RDP, WMI
* Reusing credentials from compromised systems
* Abusing network shares
* Using administrative tools like PsExec, wmiexec
* Abusing Active Directory replication for stealth

Practical mindset:
Credentials reuse karna hi sabse common method hota hai.

---

### 6. Domain Admin Privileges

Final objective hota hai Domain Admin level access.

Common techniques:

* Kerberoasting → Service account hashes crack karna
* Pass-the-Hash → NTLM hash reuse karna
* DCSync → AD database replicate karke credentials extract karna
* Golden Ticket attack → Fake Kerberos ticket create karna
* Silver Ticket attack → Specific service ticket forge karna

Practical goal:
Domain Controller par full control lena.

---

### 7. Cross-Trust Attacks

Multiple trusted domains ya forests ke beech move karna.

Techniques:

* SID History abuse
* Trust relationship exploitation
* Kerberos delegation abuse

Practical mindset:
Ek domain compromise hone ke baad dusre trusted domain par pivot karna.

---

### 8. Persistence and Exfiltration

Access maintain karna aur data steal karna.

Persistence methods:

* Scheduled tasks create karna
* Startup items
* Registry key manipulation
* Group Policy modification

Exfiltration:

* Sensitive files copy karna
* Database dumps
* Credentials export

Practical goal:
Long-term access aur sensitive data extraction.

---

### Final Flow (Realistic Attack Chain)

1. Recon
2. Enumeration
3. Local Priv Esc
4. Admin Recon
5. Lateral Movement
6. Domain Admin takeover
7. Cross-domain pivot
8. Persistence + Data Exfiltration
