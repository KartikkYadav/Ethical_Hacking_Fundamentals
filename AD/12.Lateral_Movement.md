# 🔵 Lateral Movement

## 📌 What is Lateral Movement?

**Lateral Movement** ek **post-exploitation technique** hai jisme attacker ek compromised system se network ke andar dusre systems tak move karta hai.

👉 Goal:
- Access expand karna  
- Privileges escalate karna  
- High-value targets tak pahuchna  

### 🎯 Common Targets in Active Directory (AD)

- File servers  
- Application servers  
- Domain Controllers  
- Backup servers  
- Administrator workstations  

---

## 🟢 Why Lateral Movement Matters

Initial access ke baad attacker ka real game start hota hai 👇

1. Credentials harvest karta hai  
2. Valid accounts reuse karta hai  
3. Network me pivot karta hai  
4. Privilege escalate karta hai  
5. Sensitive assets target karta hai  

🎯 Final Goal:
- **Domain Admin access**
- **Sensitive data exfiltration**

---

# 🟢 Common Lateral Movement Techniques (Windows / AD)

---

## 1️⃣ SMB-Based Movement

Windows file-sharing protocol (SMB) ka use hota hai remote execution ke liye.

### 🔧 Tools:
- CrackMapExec  
- PsExec  

### ⚙️ Working:
- Valid credentials se authentication  
- Remote command execution  
- Payload deploy / hash dump  

---

## 2️⃣ Pass-the-Hash (PtH)

Password crack karne ki jagah attacker **NTLM hash reuse karta hai**.

### 🔧 Tools:
- Mimikatz  
- Impacket  

⚡ No password required — only hash needed  

---

## 3️⃣ WinRM Remote Execution

Windows Remote Management ka use karke remote PowerShell execution.

### 🔧 Tools:
- Evil-WinRM  
- CrackMapExec  

---

## 4️⃣ RDP Pivoting

Remote Desktop Protocol use karke attacker directly login karta hai.

👉 Interactive access milta hai (GUI access)

---

## 5️⃣ WMI Execution

Windows Management Instrumentation remote execution allow karta hai.

### 🔧 Tools:
- Impacket  
- PowerShell  

---

## 6️⃣ Token Impersonation

Attacker memory me existing privileged tokens ko impersonate karta hai.

### 🔧 Often used with:
- Mimikatz  

---

# 🔵 Typical Lateral Movement Flow (AD Environment)

```
1. Initial foothold (low-priv user)
2. Credential dumping
3. Local admin reuse identify
4. Pivot to another system
5. Privilege escalation
6. Move towards Domain Controller
7. Dump NTDS (domain hashes)
```

---

# 🔵 Indicators of Lateral Movement (Blue Team View)

### 🚨 Suspicious Activities:

- Multiple authentication attempts across hosts  
- Same account ka rapid login across systems  
- Unusual WinRM usage  
- Remote service creation  
- SMB authentication spikes  
- Access to admin shares (C$, ADMIN$)

# 🟢 Defensive Controls

### 🛡️ Hardening Measures:

- ✔ Enforce unique local admin passwords (**LAPS**)  
- ✔ Disable NTLM (where possible)  
- ✔ Restrict WinRM access  
- ✔ Implement tiered admin model  
- ✔ Enable Credential Guard  
- ✔ Monitor authentication patterns  
- ✔ Reduce local admin reuse  

---

# 🔵 Simple Definition

> Lateral movement is the process of using compromised credentials or access to move from one system to another inside a network.

---

# 🧠 Quick Revision (Exam / Interview)

- Post-exploitation phase technique  
- Uses valid credentials (not exploits mostly)  
- Focus: pivot + privilege escalation  
- Common protocols: SMB, WinRM, RDP, WMI  
- Final target: Domain Controller / Domain Admin
