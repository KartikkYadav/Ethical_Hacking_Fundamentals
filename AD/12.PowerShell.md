# 🔷 PowerShell

## 📌 Overview

**PowerShell** ek powerful **task automation + scripting framework** hai jo Windows environments me deeply integrated hota hai.

### 🧩 Components:
- Command-line shell  
- Scripting language  
- Configuration management framework  

👉 Widely used in:
- System administration  
- Automation  
- Security operations

# 🟢 PowerShell Logging & Detection

## 🛡️ Security Layers

### 1️⃣ Transcription Logging
- Input + Output logging  

### 2️⃣ Script Block Logging
- Full script logging (deobfuscated)  

### 3️⃣ AMSI (AntiMalware Scan Interface)
- Execution se pehle scanning  

### 4️⃣ Constrained Language Mode (CLM)
- Restricted execution environment  

---

# 🟢 Execution Policy

⚠️ Important: **Execution Policy is NOT a security boundary**

👉 Sirf accidental script execution prevent karta hai  

---

## 🔸 Execution Policies

| Policy | Description |
|--------|------------|
| Restricted | No scripts allowed |
| AllSigned | Only signed scripts |
| RemoteSigned | Downloaded scripts must be signed |
| Unrestricted | All scripts allowed (with warning) |
| Bypass | No restriction |
| Undefined | No policy |

---

## 🔸 Check Policy Scope

    Get-ExecutionPolicy -List

### Scopes:
- Process  
- CurrentUser  
- LocalMachine  

---

# 🟢 Execution Policy Bypass Techniques

⚠️ Ye methods **policy bypass karte hain**, but AMSI/logging ko bypass nahi karte

---

## 🔸 Start with Bypass

    powershell -nop -ep bypass

👉 Use cases:

## 🔸 Environment Variable (Session Only)

    $env:PSExecutionPolicyPreference = "bypass"

# 🟢 Security Perspective

- Attackers commonly use **ExecutionPolicy Bypass**  
- Policy ≠ real protection  

### ✅ Real Defense Depends On:
- AMSI  
- Script Block Logging  
- CLM  
- EDR monitoring  

---

# 🟢 Key Takeaways

1️⃣ PowerShell ko disable karna practical nahi hai  

2️⃣ Execution Policy ≠ security control  

3️⃣ Real protection = AMSI + Logging + CLM + EDR

5️⃣ Blue team ko behavior monitor karna chahiye, sirf commands nahi
