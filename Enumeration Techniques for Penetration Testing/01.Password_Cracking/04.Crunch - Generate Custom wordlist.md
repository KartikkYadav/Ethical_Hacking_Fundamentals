### What is RSMangler

- RSMangler is a Ruby tool that **takes a wordlist and “mangles”** it — i.e. applies transformations (permutations, leetspeak, case changes, adding suffixes/prefixes, etc.). (GitHub)
- It’s often used in the context of password cracking / penetration testing, to generate variants of dictionary words that reflect how humans modify passwords (e.g. “password” → “p@ssw0rd”, “password1”, “Password”, etc.) (Putorius)
- Its features include (by default many are ON):
    
    • permutations of input words (GitHub)
    
    • uppercase / lowercase / swap / capitalize (Kali Linux)
    
    • leetspeak / full-leet substitutions (GitHub)
    
    • adding “ed”, “ing” suffixes (Kali Linux)
    
    • reversing words (Kali Linux)
    
    • adding numbers, years, punctuation, etc. (GitHub)
    
    • acronym creation from multiple words (GitHub)
    
- RSMangler is part of Kali Linux’s toolset. (Kali Linux)
- The version listed in Kali’s docs is **1.5**. (Kali Linux)

# Key options you’ll use most

- `-file, -f <file>` — **input wordlist** (use  for stdin).
    
    Example: `--file words.txt` or `--file -` to read from a pipe. (GitHub)
    
- `-min, -m <n>` — **minimum length** for output words. Use to filter tiny entries and reduce noise. (shentoushi.top)
- `-max, -x <n>` — **maximum length** for output words. Critical to stop explosion. (shentoushi.top)
- `-perms, -p` — **permutations** mode: combines words in your list into ordered concatenations (can dramatically increase output). Use only when you really need combined words. (GitHub)
- `-output <file>` — write output to a file instead of stdout (handy for large runs). (Some docs show using redirection `>`, others include `-output`.) (netprojnetworks.com)
- `-help, -h` — show help/available options. Always check your installed version. (shentoushi.top)

> Note: Many RSMangler builds enable lots of mangles by default (case changes, leet, suffixes, prefixes). That’s powerful but can create massive lists — control size with -m, -x and selective options. (Putorius)

# Practical Examples 🛠️

- **Basic pipe with length limits to avoid massive output:** (Keeps generated passwords between 6 and 12 characters.) ✅

```bash
cat base.txt | rsmangler -m 6 -x 12 --file - > mangled.txt

```

- **Generate leet + years + numeric variants:** 🔢🔡

```bash
rsmangler.rb --file words.txt --leet --years --na --pna --punctuation --output mangled.txt

```

- **Permutations on a very small list (explosive output):** ⚠️💥

```bash
rsmangler.rb --file small.txt --perms --acronym --years > permuted_mangled.txt

```

- **Pipe directly into Hydra (authorized testing only):** 🔐➡️

```bash
rsmangler.rb --file target_usernames.txt --leet --na | hydra -L username -P - ssh://target.example.com

```

1. **Quick, safe mangle (moderate size)**
    
    Keep length reasonable and save to a file:
    
    ```bash
    rsmangler --file words.txt -m 6 -x 12 --output mangled.txt
    
    ```
    
    Or with stdin:
    
    ```bash
    cat words.txt | rsmangler --file - -m 6 -x 12 > mangled.txt
    
    ```
    
    Use when you want expanded variants but not monstrous lists. (Kali Linux)
    
2. **Permutations of a small, targeted list**
    
    Only do perms on a small input set (permutations explode combinatorially):
    
    ```bash
    cat small-target-list.txt | rsmangler --file - -p -m 6 -x 16 > perms.txt
    
    ```
    
    Use when you want combinations of names/keywords (e.g., `firstname`, `lastname`, `petname`). (GitHub)
    
3. **Sample/smaller output for testing**
    
    Generate then sample the top N to inspect before running big jobs:
    
    ```bash
    cat words.txt | rsmangler --file - -m 6 -x 12 | head -n 100 > test-sample.txt
    
    ```
    
    Or random sample:
    
    ```bash
    cat words.txt | rsmangler --file - -m 6 -x 12 | shuf | head -n 100 > sample.txt
    
    ```
    
4. **Pipe into cracking tool (on-the-fly)**
    
    Avoid saving a giant file — pipe directly into your cracker if it supports stdin:
    
    ```bash
    cat words.txt | rsmangler --file - -m 6 -x 12 | john --wordlist=-
    
    ```

# Tips & safety to control output size

- **Start small**: test with `head -n 20 words.txt` so you can see what mangles produce.
- **Use `m`/`x` aggressively** — this is the single best way to control list growth. (shentoushi.top)
- **Use `-perms` only for tiny sets** (permutation of N words → ~N! growth in concatenations). (GitHub)
- **Pipe through `sort -u`** to deduplicate if you expect overlaps.
- **Combine with `shuf`/`head`** for samples before full runs.
- **Monitor disk usage**: `wc -l` and `du -h` are your friends.

---

# Example “best practice” workflow

1. Build a *targeted* seed list (people, places, product names).
2. Run RSMangler with conservative length limits:
    
    ```bash
    cat seed.txt | rsmangler --file - -m 6 -x 12 --output staged-mangled.txt
    
    ```
    
3. Deduplicate and sample:
    
    ```bash
    sort -u staged-mangled.txt | shuf | head -n 100 > final-sample.txt
    
    ```
    
4. If the sample looks good, feed `staged-mangled.txt` into your next tool.

# 🕷️ **CeWL (Custom Word List generator / web crawler)**

CeWL crawls a target website and builds a wordlist from the collected content — useful to harvest names, phrases, and email addresses linked to a domain.

**Install and repo:**

- 🔗 GitHub: https://github.com/digininja/CeWL

**Examples:**

```bash
# Basic crawl (depth 2) and write wordlist

cewl -v https://www.armourinfosec.com -d 2 -w wordlist.txt

# Crawl and extract emails as well

cewl -v https://www.armourinfosec.com -e --email_file email_out.txt -d 2 -w wordlist.txt

# View extracted emails

cat email_out.txt

```

- `e` — extract emails
- `-email_file <file>` — save emails to file
