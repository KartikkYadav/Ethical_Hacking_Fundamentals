### **Offline Password Cracking**

Offline password cracking is an attack method where the attacker has obtained one or more password hashes and attempts to recover the original plaintext passwords without interacting with the authentication server.

---

### **How Offline Attacks Work**

- The attacker acquires hashed passwords from a compromised system or leaked database.
- Since hashes are non-reversible, the attacker guesses a password, hashes it, and compares it against the stolen hash.
- This process is repeated until a match is found or the attacker gives up.

---

### **Common Hashing Algorithms**

**MD (Message Digest) Family:** MD2, MD4, MD5, MD6

**SHA (Secure Hash Algorithm) Family:** SHA-1, SHA-256, SHA-512

**Windows Hashes:** LM (Lan Manager), NTLM (NT Lan Manager)

---

### **Characteristics of Hash Algorithms**

| **Algorithm** | **Output Length** | **Notes** |

| SHA1 | 160 bits | More secure than MD5 but weak |
| SHA256 | 256 bits | Modern, secure hash |
| SHA512 | 512 bits | More secure, slower |
| NTLM | 128 bits | Used in Windows authentication |

### 🧂 **Salted Password Hashing**

- **Salt** is random data added to passwords before hashing to prevent rainbow table attacks.
- **Password + Salt:** Combines password and salt, then hashes the result.
- **Salt + Password:** Salt is concatenated before the password, then hashed.
- This increases the **complexity** and **uniqueness** of each hash even for identical passwords.

---

### 💡 **Examples**

### 🔸 Example 1 – Without Salt

```
Password:  admin123
MD5 Hash:   0192023a7bbd73250516f069df18b500

```

👉 If someone else uses the same password (`admin123`), they’ll have **the exact same hash**, making it easy for attackers to identify common passwords.

---

### 🔸 Example 2 – With Salt

```
Password:  admin123
Salt:      XyZ!
Combined:  admin123XyZ!
MD5 Hash:  9a73b21eacb2d24c6027ef8a63a3df5a

```

👉 Now, even if another user has the same password but a **different salt**, the resulting hash will be **completely unique**.

---

### 🔸 Example 3 – Using SHA256 with Salt

```
Password:  welcome@123
Salt:      s4ltKey!
SHA256 Hash:  3f1a46e9e14e902fcd2f293ec11a7b3df47c72e4c14448a6f4b15a8d7b71c1a7

```

# 🧰 Password-Cracking Rack (ethical / lab use only)

## ⚖️ Legal / Safety first

Always test only on your own systems, intentionally vulnerable lab boxes, or where you have written permission. Unauthorized cracking is illegal and unethical.

---

## 🧩 Core components

- **Hash identification** — `hash-identifier`, `hashid` (identify hash type first).
- **Cracking engines** — `hashcat` (GPU-accelerated), `john` / `john –format` (CPU or OpenCL).
- **Wordlists & rules** — `rockyou.txt`, `SecLists`, `wordlists` + custom lists.
- **Mask / rule generators** — generate targeted masks for common patterns.
- **GPU(s)** — NVIDIA (preferred for hashcat), robust cooling, up-to-date drivers.
- **Sandbox lab** — isolated VM/container with sample hashes and target services.
- **Hash collection** — safe test files (e.g., /etc/shadow from test VM) — *not* real targets.

---

## 🔎 Workflow (high level)

1. **Identify** the hash type.
2. **Select** the cracking mode (dictionary, mask, rule, hybrid).
3. **Choose** wordlists + rules or mask.
4. **Tune** hashcat/john options (threads, workload, GPU, session).
5. **Run** against test hashes.
6. **Log** results and apply defenses.

---

## 🛠️ Attack modes explained (short)

- **Dictionary**: try words from a list (fast + effective).
- **Rule-based**: transform dictionary words (e.g., append digits, substitutions).
- **Mask / brute-force**: fixed pattern for unknowns (useful for short passwords).
- **Hybrid**: dictionary + mask (e.g., word + year).
- **Rainbow tables**: precomputed — ineffective vs salted hashes.
- **Slow-hash attacks**: bcrypt/scrypt/argon2 are expensive — slower cracking.

## 🛡️ Defenses to implement (what to do on real systems)

- Use **unique salts** + **slow hashing** (bcrypt, scrypt, Argon2) with appropriate cost/work factor.
- Enforce **long, complex passphrases** (length >12; passphrases better than common words).
- Implement **rate-limiting / account lockout** to slow online guessing.
- Use **MFA** to mitigate password reuse.
- **Rotate** and **monitor** password storage schemes; detect leaks promptly.
- Use password-hashing libraries (well-maintained libs) not custom crypto.

## Dictionary (straight) attack

- **Straight dictionary (MD5 + rockyou)**
    
    `hashcat -m 0 -a 0 md5-1.txt /usr/share/wordlists/rockyou.txt --show`
    
    - **`m 0`**: specifies the hash type as **MD5**.
    - **`a 0`**: specifies the attack mode as **Straight Dictionary Attack**.
    - `md5-1.txt`: the file containing the password hashes.
    - `/usr/share/wordlists/rockyou.txt`: the wordlist/dictionary file.
    - `-show`: displays the cracked passwords.
- **Straight dictionary (SHA1)**

        echo -n "(password1)" | sha1sum | cut -d ' ' -f 1 > sha1-hash.txt

  

             hashid  6d43e6473317023f2599cce6e5e930b1d42d73ff 

             hashid  -m 6d43e6473317023f2599cce6e5e930b1d42d73ff

      

       `hashcat -m 100 -a 0 sha1-1.txt /usr/share/wordlists/rockyou.txt`

- **`m 100`**: specifies the hash type as **SHA1**.
- **`a 0`**: specifies the attack mode as **Straight Dictionary Attack**.
- **Apply rules**
    
    `hashcat -m 0 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt -r rules/best64.rule`
    
    - **`r rules/best64.rule`**: applies the transformations from the specified rule file to the words in the dictionary (e.g., appending numbers, changing case).

## Hybrid attacks

- **Wordlist + mask (append mask to each word)**
    
    `hashcat -m 0 -a 6 hashes.txt wordlist.txt ?d?d?d`
    
    - **`a 6`**: specifies the attack mode as **Hybrid Wordlist + Mask**. The mask (`?d?d?d`) is applied **after** each word.
    - `?d`: a placeholder for a single digit (0-9). In this example, 3 digits are appended.
- **Mask + wordlist (prepend mask to each word)**
    
    `hashcat -m 0 -a 7 hashes.txt ?d?d?d wordlist.txt`
    
    - **`a 7`**: specifies the attack mode as **Hybrid Mask + Wordlist**. The mask (`?d?d?d`) is applied **before** each word.
