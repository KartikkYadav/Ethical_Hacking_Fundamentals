import hashlib

def md5(data):
return hashlib.md5(data.encode()).hexdigest()

def sha1(data):
return hashlib.sha1(data.encode()).hexdigest()

def sha256(data):
return hashlib.sha256(data.encode()).hexdigest()

def compute_hashes(password, salt):
results = {}
results["md5(password)"] = md5(password)
results["md5(md5(password))"] = md5(md5(password))
results["sha256(md5(password))"] = sha256(md5(password))
results["md5(password + salt)"] = md5(password + salt)
results["md5(salt + password)"] = md5(salt + password)
results["md5(salt + password + salt)"] = md5(salt + password + salt)
combined = sha1(password) + md5(password) + sha1(password)
results["md5(sha1(password) + md5(password) + sha1(password))"] = md5(combined)
return results

print(f"{desc}: {hsh}")

## Potfile & Session Tips

This section covers commands related to the Hashcat **potfile** (a file that stores successfully cracked hashes and their corresponding passwords) and managing cracking sessions.

- **Potfile path (default):**Bash
    
    `cat ~/.local/share/hashcat/hashcat.potfile`
    
    This command is used to **view the contents** of the default Hashcat potfile.
    
- **Remove potfile to avoid reusing cracked entries:**Bash
    
    `rm -f ~/.local/share/hashcat/hashcat.potfile`
    
    This command **deletes** the potfile. This is often done before a new cracking attempt to ensure Hashcat tries to crack all hashes, even ones it has previously cracked and stored.
    
- **Start named session and restore:**BashBash
    
    `hashcat --session=myrun -m 0 -a 3 hashes.txt ?d?d?d?d`
    
    This command **starts** a new Hashcat session named `myrun` for cracking.
    
    - `m 0`: Specifies the **hash mode** (in this case, MD5).
    - `a 3`: Specifies the **attack mode** (in this case, a brute-force mask attack).
    - `hashes.txt`: The input file containing the hashes.
    - `?d?d?d?d`: The **mask** (4 digits).
    
    `hashcat --session=myrun --restore`
    
    This command is used to **resume** the previously named and interrupted session (`myrun`) from where it left off.

## Performance Tips

This section offers advice on how to monitor system performance, specifically GPU usage, while Hashcat is running.

- **Monitor GPU temps & usage:**BashBash
    
    `nvidia-smi`
    
    This command (NVIDIA System Management Interface) provides a **snapshot** of the NVIDIA GPU status, including temperature, usage, memory, and running processes.
    
    `watch -n 0.5 nvidia-smi`
    
    This command runs `nvidia-smi` every **0.5 seconds** (`-n 0.5`), providing a **continuously updating** view of the GPU performance metrics.
    
- Use --backend-devices to target specific GPUs:`cat ~/.local/share/hashcat/hashcat.potfile`
    
    This tip suggests using the --backend-devices argument in Hashcat (e.g., hashcat ... --backend-devices 1,2) to specifically tell the program which GPU(s) to utilize for the cracking process, which is useful in multi-GPU systems.
