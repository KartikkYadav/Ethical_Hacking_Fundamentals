## 🐍 **Medusa**

Medusa is a fast, parallel, and modular brute-force tool used for penetration testing against remote authentication services such as **SSH, FTP, Telnet, and MySQL**.

The commands you provided are examples for using Medusa with different authentication protocols and credentials.

---

## 📘 **Overview of Medusa Usage**

Medusa automates brute-force password attacks by trying numerous **username and password combinations** against services.

It is used for **assessing security** by identifying accounts with **weak or default credentials**.

---

## ⚙️ **Sample Installation and Commands**

### 🛠️ **Installation**

To install Medusa on a Debian/Ubuntu system:

```
apt install medusa

```

This will download and set up Medusa on your machine.

---

## 🧩 **General Syntax**

Medusa accepts a variety of command-line options:

- **u** → specifies a single username.
- **U** → specifies a file containing usernames.
- **p** → specifies a single password.
- **P** → specifies a file containing passwords.
- **h** → specifies the target host.
- **M** → specifies the module (service, e.g., ssh, ftp, mysql).
- **n** → specifies the port (if not default).
- **t** → specifies threads (for parallel attempts).

## 🔐 **SSH Examples**

⭐ *These commands attempt various username/password combinations on an SSH server at **192.168.1.31**.*

```
medusa -u root -p root -h 192.168.1.31 -M ssh -t 10

```

```
medusa -U /opt/user.txt -P /opt/password.txt -h 192.168.1.31 -M ssh -t 10

```

```
medusa -u root -P /opt/password.txt -h 192.168.1.31 -M ssh -t 10 -n 22

```

---

## 📁 **FTP Examples**

```
medusa -u administrator -p @rmour123 -h 192.168.1.41 -M ftp -n 21

```

```
medusa -U /opt/username.txt -P /opt/password.txt -h 192.168.1.41 -M ftp -n 21

```

```
medusa -C /opt/user_pass.txt -h 192.168.1.41 -M ftp -n 21

```

⭐ *These commands target FTP authentication on **192.168.1.41**, using either direct credentials or files.*

```

```

🔸 **-C** uses a file containing lines in the `username:password` format.
