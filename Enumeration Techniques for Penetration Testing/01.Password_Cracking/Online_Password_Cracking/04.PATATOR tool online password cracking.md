# 🌑🔥 **PATATOR — Complete A-to-Z Guide (Full Description + Full Commands)**

Patator is a **multipurpose brute-force tool** designed for fast, flexible, and modular password-guessing attacks across multiple services (SSH, FTP, HTTP, SMTP, MySQL, etc.).

---

# 🌈🔵 **1. Overview**

Patator supports:

✔️ Parallel brute-force

✔️ Modular design (many attack modules)

✔️ Custom response conditions

✔️ Username fuzzing / password fuzzing

✔️ Automatic threading

✔️ Multiple input providers (FILE0, FILE1…)

✔️ Rich output filtering (`-x ignore`, `-x retry`, etc.)

---

# 🟣 **2. Global Syntax**

```
patator <module> <parameter=value> <parameter=value> ... [options]

```

Example generic form:

```
patator MODULE host=TARGET user=FILE0 password=FILE1 -x ignore:code=401

```

---

# 🟡 **3. Global Options**

```
--help             Show help for patator
MODULE --help      Show module-specific help

-t N               Number of threads
-x condition       Custom response rules
-l / -s            Logging options
--timeout          Request timeout

```

# 🔥 **6. Custom Response Conditions (Important)**

### Ignore failed responses:

```
-x ignore:code=401

```

### Stop on success:

```
-x success:code=200

```

### Ignore if response contains text:

```
-x ignore:fgrep="Invalid"

```
