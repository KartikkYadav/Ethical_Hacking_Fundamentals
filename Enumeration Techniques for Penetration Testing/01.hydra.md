# HYDRA – NOTES (Kali Linux)

**Tool Name:** THC Hydra  
**Purpose:** Fast and flexible online password-cracking tool for brute-forcing login credentials.

---

## 1. What Hydra Does
- Performs brute-force and dictionary attacks on login services.
- Supports 50+ protocols:
  - SSH  
  - FTP  
  - HTTP / HTTPS  
  - SMB  
  - Telnet  
  - RDP  
  - SMTP  
  - MySQL / PostgreSQL  
- Supports single or multiple targets.

> ⚠️ Use only on systems you own or have explicit permission to test.

---

## 2. When Hydra Is Used
- Testing weak passwords  
- Penetration testing login services  
- Checking password strength and policy  

---

## 3. Hydra Command Format

### Basic Syntax

    hydra -l username -P wordlist.txt service://target


### Common Parameters
| Option | Description |
|--------|-------------|
| `-l` | Single username |
| `-L` | Username list file |
| `-p` | Single password |
| `-P` | Password wordlist |
| `-s` | Custom port |
| `-o` | Save output to file |
| `-t` | Threads (speed) |
| `-v` / `-V` | Verbose modes |
| `-f` | Stop on first valid login |

---

## 4. Example Commands

### SSH Brute Force

    hydra -l admin -P rockyou.txt ssh://192.168.1.10


### FTP Brute Force

    hydra -L users.txt -P passlist.txt ftp://192.168.1.20

### HTTP POST Form Brute Force

    hydra -l admin -P rockyou.txt 192.168.1.5 http-post-form "/login:username=^USER^&password=^PASS^:F=Incorrect"

### RDP Brute Force

    hydra -L users.txt -P rockyou.txt rdp://192.168.1.15

---

## 5. Important Notes

### Speed Optimization
    -t 64

- Higher threads = faster, but may cause lockouts.

### Stop After First Hit

    -f

### Save Output

    -o results.txt

## 7. Hydra Limitations
- Works only on online services  
- Cannot crack offline hashes  
- Can trigger lockouts or rate limiting  

(For offline cracking → use **Hashcat** or **John the Ripper**)
