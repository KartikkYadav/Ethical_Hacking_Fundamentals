# Directory Brute Force

Directory Brute Force (also called **directory enumeration** or **directory fuzzing**) is a technique used to discover hidden files and directories on a web server.

It works by systematically trying many possible URLs to see which ones exist.

---

## Example

Suppose a website is:

    https://example.com/

A brute-force tool might try URLs like:

- https://example.com/admin/
- https://example.com/login/
- https://example.com/backup/
- https://example.com/.git/
- https://example.com/uploads/

…and thousands more.

---

## Tools Commonly Used for Directory Brute Forcing

| Tool | Description |
|------|-------------|
| Gobuster | Very fast Go-based scanner |
| Dirb | Older but still effective |
| Dirbuster | GUI-based scanner |
| FFUF | Fast web fuzzer |
| Wfuzz | Flexible fuzzing tool |
| Dirsearch | Python-based scanner |

---

## How It Works

1. The tool loads a **wordlist** containing possible directory names.
2. Each word is appended to the base URL.
3. The tool sends an HTTP request to the generated URL.
4. The server response is analyzed.

Common responses:

| Status Code | Meaning |
|-------------|--------|
| 200 | Directory exists |
| 301 / 302 | Redirect |
| 403 | Directory exists but restricted |
| 404 | Directory not found |

---

## Example Command

Example using Gobuster:

    gobuster dir -u https://example.com/ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt
