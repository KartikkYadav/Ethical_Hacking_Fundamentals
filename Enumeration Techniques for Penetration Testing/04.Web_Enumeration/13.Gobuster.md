# Gobuster

## Gobuster

Gobuster is a powerful and fast brute-forcing tool written in Go, designed for discovering hidden files, directories, DNS subdomains, and virtual hosts on a web server.

It is extremely useful during **web penetration testing, red teaming, and bug bounty hunting**.

Official repository:  
https://github.com/OJ/gobuster

---

# Installation

Install **gobuster** using the package manager:

    apt install gobuster

---

# Check Installed Version

Verify the installed version:

    gobuster --version

---

# View Help Menu

Display the general help menu:

    gobuster --help

Display help for specific modules:

    gobuster dir --help

    gobuster dns --help

---

# Basic Directory Scan

Perform a basic directory brute-force:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt -u http://192.168.1.51

Example with another target:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31

Hide content length of each response:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 --hide-length

---

# Using Multiple Concurrent Threads

Set the number of concurrent threads:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -t 50

Higher thread counts increase scanning speed but may trigger WAF or rate limiting.

---

# Filtering Specific HTTP Status Codes

Select specific HTTP status codes:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -t 50 -s 403

Only show successful responses:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -t 50 -s 200

Scan a specific port and multiple status codes:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31:81 -t 50 -s 200,302,403

Hide specific status codes:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -t 50 --no-status 403

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -t 50 --no-status 200

Hide multiple status codes:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -t 50 --no-status 200,302,403

---

# Searching for Specific File Types

Search for specific file types like `.php`:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -x php

Search multiple extensions:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -x php,html

Search additional extensions:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -t 10 -x php,html,sql

Massive extension search:

    gobuster dir -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -u http://192.168.1.31 -x php,inc,php.bak,asp,aspx,do,action,cgi,pl,html,htm,js,css,json,txt,tar.gz,tgz -t 100

---

# Common Gobuster Modes

Gobuster supports multiple scanning modes:

### Directory Bruteforce

    gobuster dir -u http://target.com -w wordlist.txt

### DNS Subdomain Enumeration

    gobuster dns -d target.com -w subdomains.txt

### Virtual Host Discovery

    gobuster vhost -u http://target.com -w vhosts.txt

### S3 Bucket Enumeration

    gobuster s3 -w buckets.txt

### Google Cloud Bucket Enumeration

    gobuster gcs -w buckets.txt

---

# Useful Pentesting Tips

- Always start with **medium wordlists** to avoid unnecessary noise.
- Increase threads gradually (`-t`) to avoid triggering WAF protections.
- Combine **extensions (`-x`) with directory scans** to discover hidden files.
- Use **custom status filtering** to remove false positives.
- Run scans through **Burp proxy** when analyzing responses.
- Always respect **bug bounty scope and authorization rules**.


# Feroxbuster

## Feroxbuster

Feroxbuster is a fast, simple, recursive content discovery tool written in Rust. It is designed to help find hidden files and directories on web servers by sending HTTP requests to different paths.

It supports multi-threaded scans, proxy integration, recursion control, link extraction, smart filtering, and more.

Official repository:  
https://github.com/epi052/feroxbuster

---

# Installation

Install **feroxbuster** using the package manager:

    apt install feroxbuster

---

# Basic Usage

Display the help menu and available options:

    feroxbuster -h

This command lists all available flags, arguments, and examples.

---

# Scanning a Target URL

Perform a basic scan on a target:

    feroxbuster --url http://192.168.1.211

Recursively searches directories and files.

---

# Filtering HTTP Status Codes

Use a custom wordlist and specify threads:

    feroxbuster --url http://192.168.1.31 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -t 10

Options explanation:

- `-w` sets the wordlist  
- `-t` sets the thread count

Higher thread example:

    feroxbuster --url http://192.168.1.31 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt -t 100

Scan a public website for hidden content:

    feroxbuster --url https://www.armourinfosec.com/

Scan and ignore specific status codes:

    feroxbuster --url https://www.armourinfosec.com/ --filter-status 403,429

Save scan output to a file:

    feroxbuster -u http://192.168.1.185 -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt -t 10 -o 192.168.1.211-feroxbuster.txt

---

# Controlling Recursive Depth

Limit recursion depth:

    feroxbuster --url http://192.168.1.31 --depth 2

Helps control scan scope.

---

# Setting Timeout for Requests

Set request timeout:

    feroxbuster --url http://192.168.1.31 --timeout 10

Route traffic through a proxy:

    feroxbuster --url http://192.168.1.31 --proxy http://127.0.0.1:8080

Set a custom User-Agent header:

    feroxbuster --url http://192.168.1.31 --user-agent "Mozilla/5.0 (Pentest Scanner)"

---

# Advanced Targeting

Specify multiple targets at once using a file:

    feroxbuster --stdin < targets.txt

Contents of **targets.txt** should list one URL per line.

Example `targets.txt`:

    http://192.168.1.1
    http://192.168.1.2
    http://192.168.1.3

---

# Resuming a Scan

Resume an interrupted scan using the saved output:

    feroxbuster --resume-from resume-state.json

Resumes the scan instead of starting over.

---

# Extracting Links from Responses

Enable link extraction from responses:

    feroxbuster --url http://192.168.1.31 --extract-links

Useful for finding hidden endpoints exposed by the web application itself.

---

# Handling Rate Limits

Limit the number of requests per second to avoid server bans:

    feroxbuster --url http://192.168.1.31 --rate-limit 50

Target specific file extensions like `.php` or `.html`:

    feroxbuster --url http://192.168.1.31 --extensions php,html,txt

---

# Full Example Command

A full command combining several options:

    feroxbuster --url http://192.168.1.31 -w /usr/share/seclists/Discovery/Web-Content/common.txt -x php,html,txt -t 30 --filter-status 403,404,500 --depth 3 --timeout 15 --rate-limit 100 --proxy http://127.0.0.1:8080 -o /tmp/feroxbuster-results.txt

---

# Troubleshooting Tips

- Use `-vvv` for detailed debugging output if something goes wrong.
- Always verify wordlist paths and proxy settings.
- Try `--no-recursion` if facing huge directories without meaningful results.
- Check firewall and WAF protections which may block scans silently.
- Respect scope boundaries and ethical hacking rules.
- Use smaller wordlists first to avoid unnecessary load.
- Consider scheduling scans during maintenance windows.
