# Wfuzz – Web Fuzzing Tool

## What is Wfuzz

Wfuzz is a **web application security tool** primarily used for **brute forcing and fuzz testing web applications**.

It is commonly used in penetration testing to discover:

- Hidden directories and files on a web server
- Login credentials (usernames/passwords)
- Parameters or injection points (for SQL injection, XSS, etc.)
- Custom header or cookie-based vulnerabilities

At its core, Wfuzz sends a **large number of HTTP requests based on wordlists or payloads** that you specify.  
It then filters and analyzes the responses to identify interesting behavior, such as differences in:

- Status codes
- Response size
- Word count
- Character count

---

# Main Features

- Directory and file fuzzing
- Login brute forcing (Basic, Digest, NTLM authentication)
- GET and POST parameter fuzzing
- Header fuzzing (User-Agent, Referer, custom headers)
- Proxy support (Burp Suite, OWASP ZAP)
- Advanced filtering based on response code, lines, words, and characters

---

# Simple Example

    wfuzz -w /usr/share/wordlists/dirb/common.txt -u http://target.com/FUZZ

This command tests all words in **common.txt** at the **FUZZ** position in the URL.

# Simple Directory Discovery

Use Wfuzz to discover directories by fuzzing a single level in the URL.

Example:

    wfuzz -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

Options:

- `-w` → Wordlist used for fuzzing  
- `FUZZ` → Injection point for payload  

Using a proxy:

    wfuzz -p 127.0.0.1:8080 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

---

# Multi-Level Directory Discovery

Discover nested directories by adding multiple **FUZZ points**.

    wfuzz -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ/FUZZ/FUZZ

Using proxy:

    wfuzz -p 127.0.0.1:8080 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ/FUZZ/FUZZ

---

# Fast Directory Discovery with Filters

Speed up fuzzing with multiple threads and response filtering.

    wfuzz -t 100 --hc 404 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

    wfuzz -t 100 --hh 274 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

    wfuzz -t 100 --hc 404 --hh 274 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

Options:

- `-t 100` → Use 100 threads
- `--hc 404` → Hide responses with HTTP status code 404
- `--hh 10701` → Hide responses with specific character count

---

# Directory Discovery with Specific Response Codes

Show only responses with specific HTTP status codes.

    wfuzz -t 100 --sc 200 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

`--sc 200` → Show only responses with status code **200**

# Fuzzing Specific Filenames with Proxy Support

Use a static list of filenames and proxy traffic for interception.

    wfuzz -z list,index.php-login.php-admin.php-test.php -p 127.0.0.1:8080 -t 1 http://192.168.1.31/FUZZ

Options:

- `-z list,items` → Static payload list
- `-p 127.0.0.1:8080` → Route traffic through proxy

# Filtered Response with Proxy

Display only successful responses (status 200) through proxy.

# Multi-File and List Payloads

Combine file and list payloads.

    wfuzz -t 100 --hc 404 -p 127.0.0.1:8080:HTTP \
    -z file,/usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt \
    -z list,index.php-admin.php-login.php \
    http://192.168.1.31/FUZZ/FUZZ

Using two wordlists:

    wfuzz -p 127.0.0.1:8080 \
    -z file,/usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt \
    -z list,index.php-index.html-admin.php \
    http://192.168.1.31/FUZZ/FUZZ

# Filtering the Results

Hide filters:

- `--hc` → Hide status code  
- `--hl` → Hide lines  
- `--hw` → Hide words  
- `--hh` → Hide characters  

Examples:

    wfuzz --hc=404 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

    wfuzz --hc=404,301,500 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

---

# Colored Output

Examples:

    wfuzz -c --hl=76 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

    wfuzz -c --hc=404 --hw=181 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

# Show Filters

Show filters instead of hiding.

- `--sc` → Show status code  
- `--sl` → Show lines  
- `--sw` → Show words  
- `--sh` → Show characters  

Examples:

    wfuzz -c -t 100 --sc=200 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

    wfuzz -c -t 100 --sl=9 -w /usr/share/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-medium.txt http://192.168.1.31/FUZZ

# HTTP Basic Authentication

    wfuzz -c --hc=401 \
    -z list,administrator-root-admin-armour-u1-u2-u3 \
    -z file,/opt/password.txt \
    --basic FUZZ:FUZ2Z \
    -t 500 \
    http://192.168.1.31/

---

# HTTP Digest Authentication

    wfuzz -c --sc=200 \
    -z file,/opt/username.txt \
    -z file,/opt/password.txt \
    --digest FUZZ:FUZ2Z \
    -t 500 \
    http://192.168.1.31/

Authentication options:

- `--basic`
- `--ntlm`
- `--digest`
