# IIS Tilde (~) Shortname Enumeration

## IIS Tilde (~) Shortname Enumeration

**IIS Tilde Shortname Enumeration** is a vulnerability that exists in misconfigured **IIS (Internet Information Services)** web servers.  
It allows attackers to enumerate directory and file names using the legacy **8.3 short filename format**, even when directory listing is disabled.

---

# Background

Windows file systems (**FAT, NTFS**) create **short names (8.3 format)** for long filenames for backward compatibility.

Example:

    applicationconfig.php → APPLIC~1.PHP

Because of this behavior, IIS may respond differently to crafted requests using these shortnames, which can reveal whether a file or directory exists.

---

# Local Shortname Viewing (Windows)

To view shortnames locally on Windows:

    dir /a /x

### Switch Explanation

| Switch | Description |
|------|-------------|
| /a | Displays hidden/system files |
| /b | Uses bare format (no heading) |
| /x | Shows 8.3 short names |

Example output:

    25-02-2026  17:36    <DIR>          APPLIC~1     applicationconfigdata
    25-02-2026  17:37    <DIR>          APPLIC~2     applicationconfigdata2
    25-02-2026  17:37    <DIR>          APPLIC~3     applicationconfigbackup
                   696                  iisstart.htm
                 98757                  iisstart.png
                   168  WEB~1.CON       web.config

## 2. IIS-ShortName-Scanner (Java)

Java-based tool to enumerate shortnames in IIS.

### Install & Run

    git clone https://github.com/irsdl/IIS-ShortName-Scanner.git

    cd IIS-ShortName-Scanner/release

    java -jar iis_shortname_scanner.jar 2 20 http://192.168.1.31/

Explanation:

- `2` → Prefix length (example: `ad*`)
- `20` → Maximum results per prefix

## 3. sns — IIS Shortname Scanner (Go)

`sns` is a **fast multithreaded tool** used to discover hidden files and directories on vulnerable IIS servers using the **8.3 shortname disclosure vulnerability**.

---

# Requirements

- Go **1.18+**
- Internet access to download dependencies

---

# Install Using Go

    go install github.com/sw33tLie/sns@latest

---

    sns -u http://TARGET

# Examples

## Basic Scan

    sns -u http://192.168.1.100

    sns -u http://192.168.1.100 -w custom-wordlist.txt

    sns -u http://192.168.1.100 -t 50

    sns -u http://192.168.1.100 -o results.txt

| Flag | Description |
|-----|-------------|

| -t, --threads | Number of concurrent threads (default: 20) |
| -o, --output | Write results to file |
| --timeout | Timeout per request (seconds) |
| --verbose | Enable verbose output |

# How It Works

`sns` sends specially crafted requests such as:

    GET /admin~1* HTTP/1.1

Based on the HTTP response (**403, 404, timing differences, etc.**), it determines whether a matching shortname exists on the server.

Even if directory listing is disabled, vulnerable IIS servers may leak this information.

# Wordlists

The default wordlist is built-in, but better results can be achieved using **SecLists**.

Example wordlist:

    /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt

---

# Mitigation (For Defenders)

Disable **8.3 shortname support** on NTFS.

    fsutil behavior set disable8dot3 1

Other security measures:

- Patch IIS to the latest version
- Block suspicious patterns such as `*~1*` or `::$DATA`
- Use URLScan or rewrite rules

# Mitigation & Hardening

| Mitigation Step | Command / Note |
|---|---|
| Disable 8.3 Name Creation (NTFS) | fsutil behavior set disable8dot3 1 |
| Use URLScan / Request Filtering | Block patterns like `*~1*`, `*.aspx::$DATA` |
| Install Latest IIS Updates | Always patch vulnerabilities |
| Configure strict rewrite rules | Deny regex patterns via `web.config` |

Example **web.config rule**:

    <system.webServer>
        <security>
            <requestFiltering>
                <hiddenSegments>
                    <add segment="~1" />
                </hiddenSegments>
            </requestFiltering>
        </security>
    </system.webServer>
