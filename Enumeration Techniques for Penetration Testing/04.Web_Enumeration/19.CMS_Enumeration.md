
# CMSeeK

## CMSeeK – CMS Detection and Enumeration Tool

**CMSeeK** is a powerful open-source tool for detecting **Content Management Systems (CMS)** and enumerating their components.

It supports detection for **100+ CMS platforms** and helps identify:

- CMS versions
- Themes
- Plugins
- Users
- Potential vulnerabilities

---

# Installation

Follow the steps below to install **CMSeeK**.

## Clone the Repository

    git clone https://github.com/Tuhinshubhra/CMSeeK.git

---

## Change to the CMSeeK Directory

    cd CMSeeK

---

## Install Required Dependencies

    pip install -r requirements.txt

This installs all Python libraries required to run CMSeeK.

---

# Running CMSeeK

## Start the CMSeeK Tool

    python3 cmseek.py

or

    cmseek

This launches the **interactive CMSeeK interface** where you can scan URLs or run batch scans.

---

# Command-Line Options

You can run CMSeeK directly with command-line arguments to streamline enumeration.

---

## Check CMSeeK Version

    python3 cmseek.py --version

or

    cmseek --version

Displays the currently installed version.

---

## Update CMSeeK

    python3 cmseek.py --update

or

    cmseek --update

Fetches and installs the latest version from GitHub.

---

# Scan a Target CMS

Scan a website to detect its CMS.

    python3 cmseek.py -u http://192.168.1.31

or

    cmseek -u http://192.168.1.31

Example with domain:

    python3 cmseek.py -u https://www.armourinfosec.test

CMSeeK scans the target and gathers CMS enumeration data.

---

# Enable Verbose Output

    python3 cmseek.py -v -u http://192.168.1.31

Verbose mode shows **detailed scanning output** useful for debugging.

---

# Use a Custom User-Agent

    python3 cmseek.py -u http://192.168.1.31 --user-agent "Mozilla/5.0"

This sets a custom **User-Agent header**.

Useful for:

- Bypassing basic filters
- Avoiding bot detection
- Mimicking browser traffic

---

# Example Use Cases

## Basic Scan

    python3 cmseek.py -u https://www.armourinfosec.test

Performs a simple CMS detection scan.

---

## Verbose Scan With Custom Headers

    python3 cmseek.py -v -u https://www.armourinfosec.test --user-agent "Mozilla/5.0 (X11; Linux x86_64)"

Useful for:

- Debugging
- Testing different browser fingerprints

---

## Scan CMS Installed in a Folder

    python3 cmseek.py -u https://www.armourinfosec.test/blog/

Detects CMS installations in subdirectories such as:

- `/blog`
- `/cms`

---

# Features

- Detection of **100+ CMS platforms**
- Enumeration of CMS version, plugins, themes, and users
- Optional verbose output
- Batch URL scanning
- Support for custom user agents
- Auto-update functionality
- Works on **Linux, macOS, and Windows**
