# Wordpress-Enumeration

## WordPress Enumeration

WordPress enumeration is a key part of the **information gathering phase** when assessing a web application's security.

The goal is to identify:

- WordPress presence
- Version
- Themes
- Plugins
- Users

This helps identify **known vulnerabilities and attack vectors**.

---

# Nmap With HTTP Enumeration Script

This Nmap command uses the built-in **http-enum NSE script** to detect common web application files and folders.

    nmap -v -p 80,443 --script=http-enum.nse 192.168.1.192

Explanation:

- `-v` : Verbose output  
- `-p 80,443` : Scan HTTP and HTTPS ports  
- `--script=http-enum.nse` : Runs HTTP enumeration script

---

# WordPress Enumeration Using Nmap Script

This script specifically targets WordPress installations.

    nmap -v -p 80,443 --script=http-wordpress-enum.nse 192.168.1.192

This gathers information about:

- WordPress core files
- Plugins
- Themes
- Version indicators

---

# WordPress Username Enumeration With Nmap

Attempts to enumerate valid WordPress usernames.

    nmap -v -p 80,443 --script=http-wordpress-users.nse 192.168.1.192

Helps identify valid users which can later be used in:

- Brute force attacks
- Privilege escalation testing

---

# Notes

- Only perform these techniques on systems where you **have explicit permission**.
- Always conduct scans within an **authorized scope**.

# WPScan – WordPress Security Scanner

WPScan is a powerful **WordPress vulnerability scanner written in Ruby**.

It is widely used by penetration testers to identify vulnerabilities in:

- WordPress core
- Plugins
- Themes
- User accounts

---

# Installation

Install WPScan using APT.

    apt install wpscan

---

# Version Check And Updates

## Display WPScan Version

    wpscan --version

---

## Update WPScan Database

    wpscan --update

This updates the vulnerability database.

---

# Scanning For Vulnerabilities

## Scan Target Using HTTP

    wpscan --url http://www.armourinfosec.test

---

## Scan Target Using HTTPS

    wpscan --url https://www.armourinfosec.test

---

## Disable TLS/SSL Certificate Check

    wpscan --disable-tls-checks --url https://www.armourinfosec.test

Useful when scanning systems with:

- Self-signed certificates
- Expired certificates

---

## Scan Internal HTTPS Host

    wpscan --disable-tls-checks --url https://192.168.1.214

---

# Aggressive Detection Mode

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --detection-mode aggressive

This enables deeper scanning which may reveal hidden elements.

---

# Aggressive Scan With Threading

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --detection-mode aggressive -t 200

Explanation:

- `-t` : Number of threads

This increases scanning speed.

---

# User Enumeration

## Enumerate WordPress Users

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --enumerate u

---

## Enumerate Range Of User IDs

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --enumerate u1-1000

---

## Range Scan With Threading

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --enumerate u1-1000 -t 200

---

# Password Guessing / Brute Force

## Attempt Login Using Wordlist

    wpscan --disable-tls-checks --url https://www.armourinfosec.test -U bob -P /opt/rockyou.txt -t 100

Explanation:

- `-U` : Username
- `-P` : Password list
- `-t` : Threads

---

# Checking For Vulnerable Plugins

## Enumerate Vulnerable Plugins

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --enumerate vp

`vp` = Vulnerable plugins

---

# Aggressive Plugin Enumeration

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --enumerate ap --plugins-detection aggressive

`ap` = All plugins

---

# Plugin Enumeration With Version Detection

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --enumerate ap --plugins-detection aggressive --plugins-version-detection aggressive

---

# Checking For Vulnerable Themes

# Aggressive Theme Enumeration

# Full Enumeration

## Plugins, Themes, Configs, Exports

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --enumerate ap,at,cb,dbe

Explanation:

- `cb` : Config backups
- `dbe` : Database exports

---

# Full Enumeration With Aggressive Plugin Version Detection

    wpscan --disable-tls-checks --url https://www.armourinfosec.test --enumerate ap,at,cb,dbe --plugins-detection aggressive --plugins-version-detection aggressive

---

- `-e` : Short for `--enumerate`
