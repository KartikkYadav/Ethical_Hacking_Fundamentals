# CMS Enumeration

CMS Enumeration is the process of detecting and gathering information about the **Content Management System (CMS)** used by a website.

This includes identifying:

- The CMS type (e.g., WordPress, Joomla, Drupal)
- The CMS version
- Installed plugins/modules/themes
- Exposed users or sensitive information

---

# Why Perform CMS Enumeration?

CMS enumeration helps security testers to:

- Identify known vulnerabilities based on CMS version
- Understand the application's security posture
- Choose the correct tools and exploits
- Map the attack surface of the web application

---

# Techniques & Tools for CMS Enumeration

| Method | Description | Tools / Notes |
|------|------|------|
| Passive Detection | Analyze HTML source, HTTP headers, cookies | Wappalyzer, BuiltWith, WhatWeb |
| Active Probing | Check for known CMS paths, plugins, admin panels | wpscan, joomscan, droopescan |
| Fingerprinting | Identify CMS via code structure, cookies, file paths | WhatWeb, CMSmap |
| Brute-force Enumeration | Guess usernames/plugins via automated scripts | wpscan user enum, droopescan |
| Manual Inspection | Explore site structure and source code manually | Browser + DevTools |

# Practical CMS Enumeration Commands

## Detect CMS Automatically

    whatweb http://target.com

---

## WordPress Enumeration

    wpscan --url http://target.com --enumerate u,p,t

Enumerates:

- Users
- Plugins
- Themes

---

## Joomla Enumeration

    joomscan -u http://target.com

---

## Drupal Enumeration

    droopescan scan drupal -u http://target.com

---

# Manual CMS Detection Techniques

## Check Page Source

Look for CMS indicators in:

- meta generator tag
- script paths
- stylesheet paths

Example:

    <meta name="generator" content="WordPress 5.8">

---

## Check Robots.txt

    http://target.com/robots.txt

Often exposes:

- admin directories
- backup files
- hidden paths

---

## Inspect HTTP Headers

    curl -I http://target.com

Look for headers such as:

- X-Powered-By
- Server
- Set-Cookie

# Security Impact

Successful CMS enumeration can lead to:

- Plugin vulnerability exploitation
- Admin panel brute force
- Known CMS version exploits
- Privilege escalation

1. Detect CMS technology
2. Identify CMS version
3. Enumerate plugins/themes
4. Discover admin panels
5. Enumerate users
6. Check for known vulnerabilities

# CMSeeK – CMS Detection and Enumeration Tool

Repository: https://github.com/Tuhinshubhra/CMSeeK

CMSeeK is a powerful open-source tool used for detecting **Content Management Systems (CMS)** and enumerating their components.

It supports detection of **100+ CMS platforms** and helps identify:

- CMS version
- Plugins
- Themes
- Users
- Possible vulnerabilities

CMSeeK is commonly used during **web reconnaissance and CMS fingerprinting**.

---

# 1. Installation

    Install CMSeeK by cloning the official GitHub repository.

## Clone the Repository

    git clone https://github.com/Tuhinshubhra/CMSeeK.git

## Change to the CMSeeK Directory

    cd CMSeeK

## Install Required Dependencies

    pip install -r requirements.txt

This installs all required Python libraries needed to run CMSeeK.

---

# 2. Running CMSeeK

Start the CMSeeK tool.

    python3 cmseek.py

or

    cmseek

This launches the **interactive CMSeeK interface**, where you can scan URLs or perform batch scans.

---

# 3. Command-Line Options

You can run CMSeeK directly with command-line arguments to streamline scanning.

## Check CMSeeK Version

    python3 cmseek.py --version

or

cmseek --version

    Displays the currently installed version of CMSeeK.

---

## Update CMSeeK

    python3 cmseek.py --update

or

    cmseek --update

This fetches and installs the **latest version** from the GitHub repository.

---

# 4. Scan a Target CMS

Scan a website to detect its CMS and gather enumeration data.

    python3 cmseek.py -u http://192.168.1.31

or

    cmseek -u http://192.168.1.31

### Example

python3 cmseek.py -u https://www.armourinfosec.test

This scans the target website and identifies the CMS used.

---

# 5. Enable Verbose Output

Verbose mode displays detailed output during scanning.

    python3 cmseek.py -v -u http://192.168.1.31

Verbose mode helps with:

- Understanding enumeration steps
- Debugging scanning issues
- Viewing additional detection information

---

# 6. Use a Custom User-Agent

Some websites block automated scanners. You can bypass filters using a custom User-Agent.

    python3 cmseek.py -u http://192.168.1.31 --user-agent "Mozilla/5.0"

This makes requests appear like they are coming from a normal browser.

---

# 7. Example Use Cases

## Basic CMS Scan

    python3 cmseek.py -u https://www.armourinfosec.test

Performs a simple CMS detection scan on the target domain.

---

## Verbose Scan with Custom Headers

python3 cmseek.py -v -u https://www.armourinfosec.test --user-agent "Mozilla/5.0 (X11; Linux x86_64)"

Useful for testing with different browser fingerprints and debugging response headers.

---

## Scan CMS Installed in a Folder

Some websites install CMS inside subdirectories such as `/blog` or `/cms`.

python3 cmseek.py -u https://www.armourinfosec.test/blog/

CMSeeK will attempt to detect CMS installed inside that folder.
