# Mail-Masta-1.0-Local-File-Inclusion (LFI)

## Mail Masta 1.0 – Local File Inclusion (LFI)

- Exploit Reference: https://www.exploit-db.com/exploits/40290
- Plugin Download: https://downloads.wordpress.org/plugin/mail-masta.zip

---

# Overview

- **Application:** WordPress Plugin
- **Plugin Name:** Mail Masta
- **Affected Version:** 1.0
- **Vulnerability Type:** Local File Inclusion (LFI)
- **Attack Vector:** Remote (Unauthenticated)

A Local File Inclusion vulnerability exists in the Mail Masta plugin due to improper handling of user-supplied input via the **pl** parameter in the following script:

```
/wp-content/plugins/mail-masta/inc/campaign/count_of_send.php
```

The application fails to validate or sanitize input before using it in file inclusion functions, allowing attackers to read arbitrary files from the server.

# Technical Details

The vulnerability arises from unsafe usage of user-controlled input in PHP file handling functions such as:

```
include($_GET['pl']);
require($_GET['pl']);
file_get_contents($_GET['pl']);
```

Because the **pl** parameter is directly passed without validation:

- No path restriction
- No input sanitization
- No whitelisting

## Impact

Successful exploitation may lead to:

- Disclosure of sensitive files (**/etc/passwd**, logs, configs)
- Exposure of database credentials via **wp-config.php**
- Access to API keys and secrets
- Potential escalation to **Remote Code Execution (RCE)** (e.g., via log poisoning or file upload chaining)

---

## What is LFI?

**Local File Inclusion (LFI)** is a vulnerability where an application dynamically includes local files based on user input.

**Common exploitation techniques:**

- Directory traversal (**../../../**)
- Log file inclusion
- Session file inclusion
- Wrapper exploitation (**php://filter**, etc.)

**Example:**

```
?pl=php://filter/convert.base64-encode/resource=wp-config.php
```

---

## Secure Coding Practices

- Implement **strict input validation:**

```
$allowed_files = ['file1.php', 'file2.php'];
if (!in_array($_GET['pl'], $allowed_files)) {
    die('Invalid request');
}
```

- Use **absolute paths with whitelisting**
- Avoid dynamic file inclusion where possible

---

## PHP Hardening

**Disable:**

```
allow_url_include = Off
allow_url_fopen = Off
display_errors = Off
```
