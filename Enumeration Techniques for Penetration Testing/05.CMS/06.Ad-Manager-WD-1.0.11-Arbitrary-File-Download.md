## 🔵 Ad-Manager-WD-1.0.11-Arbitrary-File-Download (WordPress Plugin)

---

## 📌 Vulnerability Information

- **Application:** WordPress Plugin
- **Plugin Name:** Ad Manager WD
- **Affected Version:** 1.0.11
- **Vulnerability Type:** Arbitrary File Download / Directory Traversal
- **Attack Vector:** Authenticated (Admin)

---

## 🟢 Vulnerability Summary

The plugin contains an insecure file export feature that allows attackers to supply arbitrary file paths via the **`path`** parameter.

Due to insufficient validation this leads to:

- 📂 **Directory Traversal**
- 📥 **Arbitrary File Download**
- ⚠️ Reading sensitive server files

The issue exists because the plugin does not properly validate user input before processing file paths.

---

## 🛠 Technical Details

The vulnerability exists in the export functionality endpoint:

```
/wp-admin/edit.php?post_type=wd_ads_ads&export=export_csv&path=
```

### ⚠️ Issue

The **path parameter** is used to read files from the server without:

- Input sanitization
- Path validation
- Directory restriction
- Access control checks

This allows reading unintended files if abused.

---

## 🔴 Root Cause

### Main security issues:

- ❌ No validation of user supplied file paths
- ❌ No restriction to allowed directories
- ❌ Direct use of user input in file handling functions
- ❌ Missing security checks in export feature

---

## ⚡ Security Impact

An attacker with admin access could potentially:

- Read sensitive files
- Access configuration files
- Download internal application data
- Traverse directories

Example sensitive targets (risk examples):

- Configuration files
- Application logs
- Backup files
- System files (if permissions allow)

---

## 🛡 Mitigation / Fix

### Developer side fixes:

- ✅ Validate file paths
- ✅ Use allow-listed directories
- ✅ Apply `realpath()` validation
- ✅ Block traversal sequences
- ✅ Implement proper access control

### Example secure practices:

```
Validate input path
Restrict to specific export directory
Remove ../ sequences
Check file permissions
```

---

## 🔐 Recommended Security Controls

- Input validation
- Path normalization
- Least privilege file permissions
- WordPress nonce verification
- Secure coding practices
