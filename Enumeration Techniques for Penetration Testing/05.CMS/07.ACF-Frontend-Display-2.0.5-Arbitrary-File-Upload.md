## 🔵 ACF-Frontend-Display-2.0.5-Arbitrary-File-Upload

### WordPress Plugin ACF Frontend Display 2.0.5 - Arbitrary File Upload

This vulnerability affects the **ACF Frontend Display plugin (version 2.0.5)** for WordPress and allows **unauthenticated arbitrary file uploads** through an exposed file upload handler. Exploiting this flaw could result in **remote code execution (RCE)**, such as uploading a PHP web shell or info-disclosure scripts.

---

## 🟢 Vulnerability Summary

- **Type:** Arbitrary File Upload → Remote Code Execution (RCE)
- **Plugin:** ACF Frontend Display
- **Affected Version:** 2.0.5 (and possibly others)
- **Exploit-DB ID:** 37514
- **Authentication Required:** ❌ Unauthenticated

---

## 🛠 Technical Details

The plugin includes a copy of the vulnerable **blueimp jQuery File Upload** script, located at:

```
https://www.armourinfosec.test/wp-content/plugins/acf-frontend-display/js/blueimp-jQuery-File-Upload-d45deb1/server/php/index.php
```

This endpoint does **not validate file types**, nor does it properly authenticate or restrict access. As a result, attackers can upload arbitrary files such as PHP scripts.

## 🟢 Vulnerable Plugin

- This vulnerability exists in **ACF Frontend Display v2.0.5**
- The root cause is an **embedded vulnerable version of the blueimp file uploader**

---

## 🔐 Mitigation & Recommendations

### 1. Remove or Replace the Plugin

- If not essential, immediately remove this plugin.
- If functionality is required, replace it with a secure and actively maintained alternative.

### 2. Delete Vulnerable File Upload Scripts

Remove the following path and its contents:

```
/js/blueimp-jQuery-File-Upload-d45deb1/server/php/
```

### 3. Implement Access Controls

Ensure the file upload endpoints require proper authentication and authorization.

### 4. Restrict File Types

- Only allow safe file extensions (e.g., **.jpg, .png, .pdf**).
- Block execution of uploaded files via **.htaccess** or web server rules.

### 5. Monitor Upload Directories

Regularly scan:

```
wp-content/uploads/
```

for unexpected or malicious files.

### 6. Use a WAF or Security Plugin

A Web Application Firewall can detect and block unauthorized upload attempts.
