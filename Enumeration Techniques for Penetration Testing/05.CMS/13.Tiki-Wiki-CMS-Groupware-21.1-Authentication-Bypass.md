## 🔐 Tiki-Wiki-CMS-Groupware-21.1-Authentication-Bypass

## Tiki Wiki CMS Groupware 21.1 Authentication Bypass (CVE-2020-15906)

---

## 📌 Overview

- **CVE ID:** CVE-2020-15906
- **Affected Software:** Tiki Wiki CMS Groupware ≤ 21.1
- **Vulnerability Type:** Authentication Bypass

This vulnerability allows an attacker to bypass authentication mechanisms and gain unauthorized access to the application without valid credentials.

## ⚡ Exploitation Insight (Auth Bypass)

The vulnerability (CVE-2020-15906) is caused by improper handling of authentication logic in Tiki Wiki.

### Key Idea:

- Certain requests can bypass login validation
- Attackers may directly access authenticated functionality

### Typical Targets:

```
/tiki-login.php
/tiki-admin.php
/tiki-user_preferences.php
```

### What to Test:

- Direct access without session
- Manipulated parameters
- Missing authentication checks

⚠️ Exact exploit payloads vary depending on configuration, but the core issue is insufficient authentication enforcement.

## 🛡️ Mitigation & Recommendations

## Patch Management

- Upgrade to the latest secure version of Tiki Wiki CMS

## Access Control

- Restrict access to sensitive endpoints (**/tiki-admin.php**)
- Enforce proper authentication checks

## Network Security

- Limit exposure of admin interfaces
- Use reverse proxies or VPN access

## Web Application Firewall (WAF)

- Detect abnormal authentication flows
- Block unauthorized direct access attempts

## Monitoring

- Log authentication attempts
- Alert on suspicious access patterns
