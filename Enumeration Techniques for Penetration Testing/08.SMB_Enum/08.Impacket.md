
# Impacket

Impacket is a collection of Python classes for working with network protocols.

It is widely used in penetration testing for interacting with:

- SMB
- MSRPC
- LDAP
- Kerberos
- Other network protocols

# 🔹 Common Impacket Tools

| Tool | Purpose |
| --- | --- |
| `smbclient.py` | SMB share interaction |
| `secretsdump.py` | Dump hashes and secrets |
| `wmiexec.py` | Remote command execution via WMI |
| `psexec.py` | Remote shell using SMB |
| `rpcdump.py` | Enumerate RPC endpoints |
| `lookupsid.py` | SID and user enumeration |
| `GetNPUsers.py` | AS-REP roasting |
| `GetUserSPNs.py` | Kerberoasting |
| `ntlmrelayx.py` | NTLM relay attacks |
| `atexec.py` | Execute commands via Task Scheduler |

---

# 🔹 Common Use Cases

- SMB Enumeration
- Active Directory Enumeration
- Pass-the-Hash Attacks
- Kerberoasting
- AS-REP Roasting
- Remote Command Execution
- NTLM Relay Attacks
- Credential Dumping

---

# 🔹 Security Recommendations

- Disable SMBv1 where possible.
- Enforce SMB signing.
- Use strong passwords and MFA.
- Monitor SMB authentication events.
- Restrict administrative share access.
- Keep systems patched against known SMB vulnerabilities.

---

# 🔹 Summary

Impacket is one of the most powerful toolkits for Windows network interaction and Active Directory exploitation.

It is heavily used during:

- Penetration Testing
- Red Team Operations
- Active Directory Assessments
- SMB Exploitation
- Post-Exploitation Activities

Its wide collection of tools enables attackers and security professionals to interact directly with Windows authentication, SMB, Kerberos, and RPC services.
