# MS17-010 (EternalBlue)

MS17-010, commonly known as **EternalBlue**, is a critical vulnerability in the Microsoft SMBv1 protocol that allows remote code execution on vulnerable Windows systems.

The exploit was leaked from NSA offensive tooling and later used in major ransomware attacks such as:

- WannaCry
- NotPetya

# 🔹 SMBv1 Overview

SMB (Server Message Block) is a protocol used for:

- File sharing
- Printer sharing
- Remote administration
- Named pipe communication

EternalBlue abuses a flaw in SMBv1 memory handling to execute arbitrary code in kernel context.

# ✅ Group Policy Path

Launch with:

```powershell
gpedit.msc
```

```text
Computer Configuration → Windows Settings → Security Settings → Local Policies → Security Options →
Network access: Restrict anonymous access to Named Pipes and Shares
```

---

When disabled, it allows anonymous (null session) users to access named pipes and shared folders that do not have explicit permissions.

This lowers the barrier for remote enumeration, exploitation, and privilege escalation over SMB.

---

# 🔹 Configure via PowerShell

# ✅ Check Current RestrictAnonymous Setting

```powershell
Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name RestrictAnonymous
```

---

# ✅ Enable Restrict Anonymous Access (Recommended)

```powershell
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name RestrictAnonymous -Value 1
```

---

# ✅ Disable Restrict Anonymous Access (Insecure)

```powershell
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name RestrictAnonymous -Value 0
```

---

# ✅ Verify Configuration

```powershell
Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa"
```

---

# 🔹 RestrictAnonymous Values

| Value | Meaning |
| --- | --- |
| `0` | No restrictions (anonymous/null sessions allowed) |
| `1` | Restrict anonymous access to SAM accounts and shares |
| `2` | No anonymous access without explicit permissions |

# 🔹 Security Recommendations

- Disable SMBv1 immediately.
- Apply MS17-010 security patches.
- Restrict access to SMB ports (`139/445`).
- Enable SMB signing where possible.
- Monitor SMB traffic and authentication logs.
- Disable anonymous/null session access.

# 🔹 Summary

MS17-010 (EternalBlue) is one of the most dangerous SMB vulnerabilities ever disclosed.

It enables unauthenticated remote code execution against vulnerable Windows systems through SMBv1 and has been used in large-scale cyberattacks worldwide.

Understanding SMB enumeration, null sessions, SMBv1 exposure, and patch management is essential for defending against EternalBlue-style attacks.
