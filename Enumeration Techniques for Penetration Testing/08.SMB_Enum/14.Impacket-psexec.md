# Impacket-psexec

`impacket-psexec` is a post-exploitation tool from the Impacket suite that allows remote command execution on Windows systems over SMB using PsExec-like functionality.

It creates a Windows service remotely, uploads a temporary executable, executes commands, and removes artifacts afterward — providing semi-interactive command execution with elevated privileges.

---

# 🔹 Use Cases

- Lateral movement
- Remote command execution
- Privilege escalation (from privileged contexts)
- Remote persistence setup
- Administrative automation
- Post-exploitation operations

---

# 🔹 How It Works

1. Authenticates to the target via SMB
2. Uploads a temporary service executable (`RemComSvc`-style)
3. Creates and starts a Windows service
4. Executes the supplied command
5. Cleans up:
   - Service
   - Executable
   - Temporary artifacts

---

# 🔹 Key Features

- SYSTEM-level command execution
- NTLM authentication support
- Kerberos authentication support
- Semi-interactive shell support
- Widely used in:
  - Red Teaming
  - Penetration Testing
  - Active Directory Assessments

# 🔹 Authentication Format

If the password contains special characters (`@`, `!`, etc.), wrap it in single quotes.

```bash
[domain/]username[:password]@<target>
```

# 🔹 Common Flags

| Flag | Description |
| --- | --- |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-dc-ip` | Specify Domain Controller IP |
| `-target-ip` | Specify target IP |
| `-codec` | Output encoding |
| `-debug` | Enable debug output |

# 🔹 Comparison : psexec vs wmiexec vs atexec

| Feature | `psexec` | `wmiexec` | `atexec` |
| --- | --- | --- | --- |
| Service Creation | Yes | No | No |
| Binary Upload | Yes | No | No |
| Interactive Shell | Yes | Semi | No |
| Noise Level | High | Medium | Medium |
| Privileges Required | Admin | WMI Access | Admin |

- Restrict SMB administrative access
- Monitor:

- Use:
  - MFA
  - Credential Guard
  - LAPS

---

# 🔹 Notes

- `impacket-psexec` is very effective but highly detectable.
- Creates artifacts on the target system.
- Commonly blocked or monitored by:
  - AV
  - EDR
  - SIEM solutions

- Often used after:
  - Credential dumping
  - Pass-the-Hash attacks
  - Lateral movement
