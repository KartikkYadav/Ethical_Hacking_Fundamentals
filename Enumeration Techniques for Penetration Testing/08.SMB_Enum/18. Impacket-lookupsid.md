# 💀 Impacket-lookupsid

## 🔹 Impacket-lookupsid

`impacket-lookupsid` is a tool from the Impacket suite used for SID (Security Identifier) enumeration on remote Windows systems.

It connects over SMB/RPC and maps Relative Identifiers (RIDs) to usernames and groups using the LSA RPC service.

This tool is commonly used during:

- Active Directory reconnaissance
- Internal penetration testing
- User enumeration
- Privilege discovery

---

# 🟢 Use Cases

- Enumerate valid usernames and groups on a remote host
- Perform user enumeration during internal penetration tests
- Identify:
  - Privileged accounts
  - Service accounts
  - Built-in users

- Map common RIDs such as:

```text
500  → Administrator
501  → Guest
502  → KRBTGT
```

- Assist with:
  - Password spraying
  - Brute-force planning
  - Privilege escalation
  - Lateral movement preparation

---

# ⚙️ How It Works

1. Authenticates to the target over SMB
2. Connects to the `lsarpc` named pipe
3. Queries the Security Account Manager (SAM) through RPC
4. Resolves RID ranges into:
   - Usernames
   - Groups
   - Built-in accounts

---

# 🔥 Key Features

- Enumerates:
  - Local accounts
  - Domain accounts
  - Groups

- Identifies:
  - Renamed administrator accounts
  - Service accounts
  - Privileged users

- Read-only enumeration
- No remote code execution
- Useful for stealthy Active Directory reconnaissance

---

# 🧾 Authentication Format

```bash
[domain/]username[:password]@<target>
```

---

# 🟣 Password with Special Characters

```bash
impacket-lookupsid armour.com/administrator:'@Pushbrite9435'@192.168.29.204
```

---

# 📝 Notes

- Requires valid SMB/RPC authentication
- Access to `lsarpc` must be allowed
- Hardened systems or firewalls may block enumeration
- Does not execute commands on the target
- Can sometimes work anonymously if null sessions are enabled

# 🔹 Common RID Values

| RID | Account |
| --- | --- |
| `500` | Administrator |
| `501` | Guest |
| `502` | KRBTGT |
| `512` | Domain Admins |
| `513` | Domain Users |
| `514` | Domain Guests |
| `515` | Domain Computers |
| `516` | Domain Controllers |

---

# 🔹 Common Flags

| Flag | Description |
| --- | --- |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-domain-sids` | Enumerate domain SIDs |
| `-debug` | Enable debug output |
| `-dc-ip` | Specify Domain Controller IP |

# 🔥 Why It Matters

RID enumeration helps attackers:

- Discover valid usernames
- Identify privileged accounts
- Identify service accounts
- Build password spraying lists
- Detect renamed administrator accounts

This significantly improves:

- Credential attacks
- Privilege escalation planning
- Active Directory reconnaissance

---

# ⚠️ Limitations

- Requires SMB/RPC connectivity
- Firewall rules may block RPC access
- Hardened environments may disable anonymous SID enumeration
- Does not provide:
  - Shell access
  - Command execution
  - Credential dumping

---

# 🛡️ Defensive Recommendations

- Disable anonymous SID enumeration
- Restrict SMB/RPC exposure
- Monitor:
  - LSA RPC access
  - RID cycling activity
  - Excessive SMB enumeration

- Enable:
  - Sysmon
  - Windows Event Logging
  - EDR monitoring

- Restrict:
  - Null sessions
  - Anonymous access
  - Remote SAM enumeration

# 🔹 Summary

`impacket-lookupsid` is a powerful reconnaissance utility used to enumerate users, groups, and Security Identifiers (SIDs) from Windows systems over SMB/RPC.

It is heavily used during:

- Internal Penetration Testing
- Active Directory Enumeration
- Red Team Operations
- Post-Exploitation Reconnaissance

Because it performs read-only SID enumeration without executing code, it is often quieter and stealthier than remote execution tools.
