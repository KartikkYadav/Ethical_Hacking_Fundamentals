# 💀 Impacket-samrdump

## 🔹 Impacket-samrdump

`impacket-samrdump` is a tool in the Impacket suite that interacts with the SAMR (Security Account Manager Remote) protocol to enumerate users, groups, and RIDs from remote Windows systems or Active Directory domains.

It is commonly used during:

- Active Directory reconnaissance
- Internal penetration testing
- User and group enumeration
- Privilege discovery
- Lateral movement planning

---

# 🟢 Use Cases

- Enumerate:
  - Local users
  - Domain users
  - Groups
  - Built-in accounts

- Identify:
  - Privileged accounts
  - Service accounts
  - Administrative groups

- Perform account discovery during internal assessments
- Gather reconnaissance data for:
  - Privilege escalation
  - Password spraying
  - Kerberoasting
  - SMB authentication testing

---

# ⚙️ How It Works

1. Connects to the target system over SMB
2. Accesses the `lsarpc` named pipe
3. Communicates with the SAMR service via RPC
4. Retrieves information about:

```text
• User accounts
• Group memberships
• Relative Identifiers (RIDs)
• Built-in groups
```

---

# 🔥 Key Features

- No remote command execution
- Works against:
  - Local systems
  - Domain Controllers

- Often works with low-privileged credentials
- More detailed than `lookupsid`
- Useful for stealthy Active Directory reconnaissance

---

# 🧾 Authentication Format

```bash
[domain/]username[:password]@<target>
```

---

# 🟣 Passwords with Special Characters

```bash
impacket-samrdump armour.com/administrator:'@Pushbrite9435'@192.168.29.204
```

---

# 📝 Notes

- Requires valid SMB credentials
- Access to the `lsarpc` pipe must be allowed
- Hardened environments may restrict SAMR enumeration
- Frequently blocked by:
  - Firewall policies
  - RPC restrictions
  - SMB hardening

- Does not provide:
  - Shell access
  - Command execution
  - Credential dumping

# 🔍 samrdump vs lookupsid

| Feature | `samrdump` | `lookupsid` |
| --- | --- | --- |
| Enumerates Users | Yes | Yes |
| Enumerates Groups | Yes | Limited |
| Detailed Account Info | Yes | Limited |
| RID Cycling | No | Yes |
| Requires Valid Credentials | Usually | Sometimes |
| Remote Code Execution | No | No |

---

# 🔐 Authentication Methods

| Method | Example |
| --- | --- |
| Password | `user:password@target` |
| NTLM Hash | `-hashes LMHASH:NTHASH` |
| Kerberos | `-k -no-pass` |
| AES Key | `-aesKey <AES_KEY>` |

---

# 🚩 Common Flags

| Flag | Description |
| --- | --- |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-dc-ip` | Specify Domain Controller IP |
| `-target-ip` | Specify target IP |
| `-debug` | Enable debug output |

# ⚠️ Limitations

- Requires SMB/RPC connectivity
- Firewall rules may block RPC access
- Some hardened environments disable SAMR enumeration
- May generate:
  - RPC logs
  - SMB authentication logs

---

# 🛡️ Defensive Recommendations

- Restrict SAMR access
- Disable anonymous enumeration
- Limit SMB/RPC exposure
- Monitor:
  - SAMR queries
  - RID enumeration activity
  - SMB reconnaissance

- Enable:
  - Sysmon
  - Windows Event Logging
  - EDR monitoring

- Restrict:
  - Null sessions
  - Anonymous access
  - Excessive RPC access

# 🔹 Summary

`impacket-samrdump` is a powerful reconnaissance utility for enumerating users, groups, and account information through the SAMR protocol over SMB/RPC.

It is heavily used during:

- Active Directory Enumeration
- Internal Penetration Testing
- Red Team Operations
- Post-Exploitation Reconnaissance

Because it performs read-only account enumeration without executing code, it is quieter and less intrusive than remote execution tools.
