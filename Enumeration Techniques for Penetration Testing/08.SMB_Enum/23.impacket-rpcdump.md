# 💀 impacket-rpcdump

## 🔹 impacket-rpcdump

`impacket-rpcdump` is a reconnaissance tool in the Impacket suite that enumerates registered RPC endpoints on remote Windows systems.

It communicates with the RPC Endpoint Mapper (EPM) service over TCP port `135` to identify exposed:

- RPC interfaces
- Named pipes
- Protocol bindings
- Listening services

It is commonly used for:

- Windows reconnaissance
- Internal network assessments
- Attack surface discovery
- RPC service enumeration

---

# 🟢 Use Cases

- Enumerate exposed RPC services on remote systems
- Identify accessible:
  - DCOM interfaces
  - RPC interfaces
  - Named pipes

- Detect potentially vulnerable services such as:
  - MS-EFSR
  - MS-SAMR
  - MS-RPRN
  - ATSVC

- Support:
  - Lateral movement research
  - Privilege escalation analysis
  - Attack path discovery

- Map Windows RPC attack surface during internal assessments

---

# ⚙️ How It Works

1. Connects to the target RPC Endpoint Mapper service
2. Queries registered RPC interfaces
3. Enumerates:
   - UUIDs
   - Protocol bindings
   - Named pipes
   - Listening ports

4. Displays endpoint information for analysis

---

# 🔥 Key Features

- Supports anonymous enumeration
- Enumerates RPC UUIDs and endpoints
- Reveals accessible named pipes
- Useful for RPC service discovery
- No command execution
- No direct exploitation functionality

# 📌 Requirements & Notes

- Uses RPC Endpoint Mapper on TCP port `135`
- Some endpoints require authentication
- Hardened systems may restrict enumeration
- Primarily a reconnaissance utility
- Does not directly exploit services

---

# 🚨 Detection Considerations

- RPC enumeration activity may be logged
- Excessive RPC probing can trigger monitoring alerts
- Named pipe enumeration may appear in endpoint telemetry
- Frequently observed during internal reconnaissance

---

# 🔐 Authentication Methods

| Method | Example |
| --- | --- |
| Password | `user:password@target` |
| NTLM Hash | `-hashes LMHASH:NTHASH` |
| Kerberos | `-k -no-pass` |
| AES Key | `-aesKey <AES_KEY>` |

---

# 🚩 Common Flags

| Flag | Description |
| --- | --- |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-target-ip` | Specify target IP |
| `-dc-ip` | Specify Domain Controller IP |
| `-debug` | Enable debug output |

---

# ⚠️ Limitations

- Requires RPC connectivity
- Firewall rules may block TCP/135
- Hardened systems may restrict anonymous enumeration
- Does not provide:
  - Shell access
  - Command execution
  - Credential dumping

---

# 🛡️ Defensive Recommendations

- Restrict RPC exposure
- Disable unnecessary RPC services
- Limit SMB and named pipe access
- Monitor:
  - RPC enumeration
  - Named pipe access
  - Excessive endpoint mapper requests

- Enable:
  - Windows Event Logging
  - Sysmon
  - EDR monitoring

- Restrict:
  - Anonymous enumeration
  - Unnecessary remote administration interfaces

# 🔹 Summary

`impacket-rpcdump` is a lightweight RPC reconnaissance utility used to enumerate exposed RPC services, named pipes, and protocol bindings on Windows systems.

It is heavily used during:

- Internal Penetration Testing
- Active Directory Reconnaissance
- Red Team Operations
- Attack Surface Mapping

Because it performs passive RPC enumeration without executing code, it is often quieter than remote execution tools while still revealing valuable attack surface information.
