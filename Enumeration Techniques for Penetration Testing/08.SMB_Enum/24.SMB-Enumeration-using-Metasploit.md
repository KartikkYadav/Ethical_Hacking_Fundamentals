# 🛠 SMB-Enumeration-using-Metasploit

## 📌 SMB Enumeration using Metasploit

---

# ✅ Prerequisites

- Metasploit Framework installed (`msfconsole`)
- Target host reachable (e.g., `192.168.1.52`)
- Network access to SMB port `445`
- Optional: Valid SMB credentials

---

# 🚀 Start Metasploit

```bash
msfconsole
```

---

# 📂 SMB Enumeration Modules

# 🌐 1. Enumerate SMB Shares (No Auth)

```bash
use auxiliary/scanner/smb/smb_enumshares
set RHOSTS 192.168.10.52
run
```

## 🎯 Purpose

- Lists available shared folders
- Detects guest-accessible shares

---

# 👤 2. Enumerate SMB Users

```bash
use auxiliary/scanner/smb/smb_enumusers
set RHOSTS 192.168.10.52
run
```

## 🎯 Purpose

- Discovers valid domain/local users
- Useful for password attacks (in lab scenarios)

---

# 🖥 3. Enumerate SMB Sessions

```bash
use auxiliary/scanner/smb/smb_enumsessions
set RHOSTS 192.168.10.52
run
```

## 🎯 Purpose

- Shows active SMB sessions
- Helps identify logged-in users

---

# 💻 4. Detect SMB Version & OS Info

```bash
use auxiliary/scanner/smb/smb_version
set RHOSTS 192.168.10.52
run
```

## 🎯 Purpose

- Detects SMB version
- Identifies operating system
- Finds potential vulnerabilities

---

# 👨‍💻 5. Enumerate Logged-in Users

```bash
use auxiliary/scanner/smb/smb_enum_logged_on_users
set RHOSTS 192.168.10.52
run
```

## 🎯 Purpose

- Lists currently logged-in users
- Helps map active sessions

---

# 🔐 Using SMB Credentials (Authenticated Mode)

If credentials are available:

```bash
set SMBUser <username>
set SMBPass <password>
```

## 🌐 Optional (If domain is required)

```bash
set SMBDomain <domain>
```

---

# 🎯 5. Enumerate Login Users

```bash
use auxiliary/scanner/smb/smb_login
set RHOSTS 192.168.10.52
run
```

## 🎯 Purpose

- Lists currently logged-in users
- Helps map active sessions

---

# ⚡ Post-Exploitation (Meterpreter Session)

> Requires an active Meterpreter session
> 

## 📂 Enumerate Shares

```bash
use post/windows/gather/enum_shares
set SESSION <session_id>
run
```

---

# 💡 Practical Tips

# 🛰 Start with reconnaissance

Always begin with:

```bash
smb_version
```

## 🔍 This helps identify

- OS type
- SMB version
- Potential vulnerabilities

---

# 🔗 Combine with Nmap for deeper insight

```bash
nmap -p 445 --script smb-enum* 192.168.10.52
```

## 📌 Useful Scripts

- `smb-enum-shares`
- `smb-enum-users`
- `smb-os-discovery`

---

# ⚠ Look for misconfigurations

- Anonymous SMB access
- Writable shares
- Old SMBv1 enabled
- Weak authentication policies

---

# 📋 Quick Workflow Summary

```
1. smb_version              → Identify system
2. smb_enumshares          → Find shared folders
3. smb_enumusers           → Identify users
4. smb_enumsessions        → Check active sessions
5. smb_enum_logged_on_users → Map live users
6. Combine with Nmap for validation
```
