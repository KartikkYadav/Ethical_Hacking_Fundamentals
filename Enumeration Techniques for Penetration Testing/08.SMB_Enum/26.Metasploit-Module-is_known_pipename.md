# 📌 Metasploit-Module-is_known_pipename

## 🛠 Metasploit Module: `is_known_pipename` (Samba)

- This Metasploit module exploits a vulnerability in Samba (`CVE-2017-7494`), allowing remote code execution when a malicious shared library is uploaded to a writable share and then loaded via a named pipe.

## ⚠ Affected Samba Versions

- `3.5.0` to `4.5.6`
- `4.6.0` to `4.6.4`

---

# 📝 Notes on Options

- `SMB::AlwaysEncrypt false`
    
    → disables forced encryption for compatibility
    
- `SMB::ProtocolVersion 1`
    
    → forces legacy SMBv1 behavior
    
- `SMB_SHARE_NAME`
    
    → must be a writable SMB share

# 🔄 Exploitation Flow

1. Identify writable SMB share
2. Upload malicious `.so` payload to share
3. Configure Metasploit module
4. Trigger exploit via named pipe access

---

# ✅ Requirements for Successful Exploitation

- Vulnerable Samba version (`CVE-2017-7494`)
- Writable SMB share access
- SMB service reachable (`port 445 open`)
- Correct architecture payload (`x64/x86 mismatch will fail`)

---

# 💡 Tips

- Always verify Samba version before attempting exploitation
- Check share permissions (`read/write`)
- Ensure payload matches target architecture
- Combine with SMB enumeration tools (`netexec` / Metasploit scanners) for discovery
