# 📘 MySQL Remote Access Configuration & Connection

This guide covers how to enable remote MySQL access, configure authentication, open the required network port, and troubleshoot common connection issues.

---

# 🟢 1. Login Locally

## 🟩 Using MySQL Client

```bash
mysql -u root -p
```

## 🟩 Using `sudo` (Ubuntu/Debian)

```bash
sudo mysql
```

---

# 🟢 2. Check Existing Root Access

## 🟩 View Root Account & Authentication Plugin

```sql
SELECT user, host, plugin
FROM mysql.user
WHERE user='root';
```

---

# 🟢 3. Create a Remote Root User

## 🟩 Allow Connections from Any Host

```sql
CREATE USER 'root'@'%' IDENTIFIED BY 'StrongPassword!';
```

---

## 🟩 Grant Administrative Privileges

```sql
GRANT ALL PRIVILEGES
ON *.*
TO 'root'@'%'
WITH GRANT OPTION;
```

---

## 🟩 Reload Privileges

```sql
FLUSH PRIVILEGES;
```

---

# 🟢 4. MySQL 8 Authentication Fix

> Required when `auth_socket` or `caching_sha2_password` prevents remote authentication.

## 🟩 Check Authentication Plugin

```sql
SELECT user, host, plugin
FROM mysql.user
WHERE user='root';
```

---

## 🟩 Switch to `mysql_native_password`

```sql
ALTER USER 'root'@'%'
IDENTIFIED WITH mysql_native_password
BY 'StrongPassword!';
```

---

## 🟩 Reload Privileges

```sql
FLUSH PRIVILEGES;
```

---

# 🟢 5. Enable Remote Listening

## 🟩 Ubuntu / Debian

```bash
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
```

---

## 🟩 CentOS / RHEL

```bash
sudo nano /etc/my.cnf
```

---

## 🟩 Find

```ini
bind-address = 127.0.0.1
```

---

## 🟩 Replace With

```ini
bind-address = 0.0.0.0
```

---

# 🟢 6. Restart MySQL

## 🟩 systemd

```bash
sudo systemctl restart mysql
```

---

## 🟩 SysV

```bash
sudo service mysql restart
```

---

# 🟢 7. Configure Firewall

## 🟩 UFW

```bash
sudo ufw allow 3306/tcp
```

---

## 🟩 Firewalld

```bash
sudo firewall-cmd --add-port=3306/tcp --permanent
sudo firewall-cmd --reload
```

---

## 🟩 iptables

```bash
iptables -A INPUT -p tcp --dport 3306 -j ACCEPT
```

---

# 🟢 8. Verify MySQL Is Listening

```bash
ss -antlp | grep 3306
```

### Expected Output

```text
LISTEN 0 80 0.0.0.0:3306
```

---

# 🟢 9. Connect to a Remote MySQL Server

## 🟩 Basic Login

```bash
mysql -h 192.168.29.70 -u root -p
```

### Parameters

| Option | Description |
|---------|-------------|
| `-h` | Target host/IP |
| `-u` | Username |
| `-p` | Prompt for password |

---

## 🟩 Login with Password Inline

> ⚠️ Avoid using inline passwords on shared systems because they may be visible in shell history or process listings.

```bash
mysql -h 192.168.29.70 -u root -proot
```

### Example

```bash
mysql -h 192.168.29.70 -u admin -pAdmin@123
```

---

## 🟩 Specify Port

```bash
mysql -h 192.168.29.70 -P 3306 -u root -p
```

---

## 🟩 Disable SSL

```bash
mysql -h 192.168.29.70 --ssl=FALSE -u root -p
```

---

## 🟩 Force TCP Connection

```bash
mysql --protocol=TCP -h 192.168.29.70 -u root -p
```

# 🟢 11. Troubleshooting

## 🔴 Access Denied

```text
ERROR 1045 (28000): Access denied for user 'root'@'192.168.29.150'
```

### Check Existing Permissions

```sql
SELECT user, host
FROM mysql.user;
```

---

## 🟩 Common Causes

- Incorrect username or password
- User is not permitted to connect remotely
- Host restrictions in MySQL grants
- Firewall blocking TCP port **3306**
- `bind-address` still set to `127.0.0.1`
- MySQL service not restarted after configuration changes
- Authentication plugin mismatch (`auth_socket` or `caching_sha2_password`)

---

# 🟢 12. Restrict Remote Root Access

Instead of allowing access from every host (`%`), restrict access to a specific network.

```sql
CREATE USER 'root'@'192.168.29.%'
IDENTIFIED BY 'StrongPassword!';
```

---

# 🟢 13. SSH Tunnel (Recommended)

> Using an SSH tunnel is generally more secure than exposing MySQL directly to the network.

## 🟩 Create SSH Tunnel

```bash
ssh -L 3306:127.0.0.1:3306 user@192.168.29.70
```

---

## 🟩 Connect Through Tunnel

```bash
mysql -h 127.0.0.1 -u root -p
```

# 🟢 15. Quick Workflow

```text
1. Login locally
2. Check existing root account
3. Create remote user (or modify existing user)
4. Grant required privileges
5. Change authentication plugin (if needed)
6. Set bind-address to 0.0.0.0
7. Restart MySQL
8. Open firewall port 3306
9. Verify MySQL is listening
10. Test remote connection
11. Troubleshoot authentication or networking issues
12. Prefer SSH tunneling for production environments
```
