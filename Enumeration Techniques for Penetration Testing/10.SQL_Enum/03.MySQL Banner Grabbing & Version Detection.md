# 📘 MySQL Banner Grabbing & Version Detection

Banner grabbing is the process of identifying a MySQL server by retrieving information exposed during the initial connection handshake. The handshake can reveal the server version, protocol version, authentication plugin, capabilities, and other useful metadata.

---

# 🟢 1. Basic Banner Grabbing

## 🟩 Netcat

Connect directly to the MySQL service and read the server banner.

```bash
nc 192.168.10.103 3306
```

---

## 🟩 Verbose Netcat

```bash
nc -v 192.168.10.103 3306
```

### Example Output

```text
Connection to 192.168.10.103 3306 port [tcp/mysql] succeeded!

5.7.29-0ubuntu0.18.04.1
```

---

## 🟩 Telnet

Use Telnet to view the MySQL handshake banner.

```bash
telnet 192.168.10.103 3306
```

### Example Output

```text
Trying 192.168.10.103...
Connected to 192.168.10.103.

5.7.40-3.28-MariaDB

Thread ID: 12
Capabilities flags: 65535
Status: Autocommit
```

# 🟢 2. MySQL Client-Based Detection

## 🟩 Connect to the Server

```bash
mysql -h 192.168.10.103 -u root -p
```

---

## 🟩 Display Server Version

```sql
SELECT VERSION();
```

---

## 🟩 Display Version Variables

```sql
SHOW VARIABLES LIKE '%version%';
```

### Example Output

```text
+-------------------------+------------------------+
| Variable_name           | Value                  |
+-------------------------+------------------------+
| version                 | 8.0.36                 |
| version_comment         | MySQL Community Server |
| version_compile_os      | Linux                  |
| version_compile_machine | x86_64                 |
+-------------------------+------------------------+
```

# 🟢 3. Nmap-Based Detection

## 🟩 Aggressive Version Detection

```bash
nmap -v -sT -sV -A -sC -p3306 192.168.10.103
```

---

## 🟩 MySQL Handshake Information

```bash
nmap -v -sT -p3306 --script=mysql-info.nse 192.168.10.103
```

### Example Output

```text
PORT     STATE SERVICE VERSION
3306/tcp open  mysql   MySQL 5.7.29

Protocol: 10
Thread ID: 12
Capabilities flags: 65535
Status: Autocommit
```

---

## 🟩 Enumerate MySQL Information

```bash
nmap -v -sT -p3306 --script=mysql-enum.nse 192.168.10.103
```

---

## 🟩 Enumerate MySQL Users

```bash
nmap -v -sT -sV -p3306 --script=mysql-users.nse 192.168.10.103
```

---

## 🟩 Check for Empty Passwords

```bash
nmap -v -p3306 --script=mysql-empty-password.nse 192.168.10.103
```

---

## 🟩 Dump Server Variables

```bash
nmap -v -p3306 --script=mysql-variables.nse 192.168.10.103
```

# 🟢 5. Manual Packet Inspection

## 🟩 Capture the MySQL Handshake

```bash
tcpdump -i eth0 -nn -s0 -A port 3306
```

---

## 🟩 Inspect with Wireshark

### Display Filter

```text
mysql
```

### Look For

```text
Server Greeting
```

### The Greeting Packet Typically Contains

- Protocol version
- MySQL server version
- Thread ID
- Authentication plugin
- Capability flags
- Character set
- Status flags

# 🟢 6. Mass Scanning Multiple Hosts

## 🟩 Scan a Network with Nmap

```bash
nmap -sV -p3306 192.168.1.0/24
```

---

## 🟩 Fast Discovery with Masscan

```bash
masscan 192.168.1.0/24 -p3306 --rate 1000
```

---

## 🟩 Save Results

```bash
masscan 192.168.1.0/24 -p3306 --rate 1000 -oL mysql.txt
```

---

## 🟩 Extract Live Hosts

```bash
cat mysql.txt | awk '/open/ {print $4}' > ips.txt
```

---

## 🟩 Detailed Enumeration

```bash
nmap -sV -p3306 -iL ips.txt
```

# 🟢 9. Useful Nmap NSE Scripts

| NSE Script | Purpose |
|------------|---------|
| `mysql-info.nse` | Display MySQL version and handshake details |
| `mysql-enum.nse` | Enumerate MySQL information |
| `mysql-users.nse` | Enumerate MySQL users |
| `mysql-empty-password.nse` | Check for blank passwords |
| `mysql-variables.nse` | Dump server variables |
| `mysql-databases.nse` | Enumerate databases |
| `mysql-brute.nse` | Password auditing (authorized environments only) |
| `mysql-query.nse` | Execute SQL queries using valid credentials |

# 🟢 11. Key Findings

- MySQL typically listens on **TCP port 3306**.
- The initial handshake often reveals the server version without authentication.
- Nmap NSE scripts provide detailed MySQL enumeration capabilities.
- MariaDB commonly presents compatibility banners such as:

```text
5.5.5-10.6.12-MariaDB
```

- Packet captures can reveal protocol details, authentication plugins, and capability flags without valid credentials.
- For accuracy, verify server information using multiple methods such as:
  - Banner Grabbing
  - Nmap Version Detection
  - MySQL Client
  - Packet Inspection
