# 📘 SQLCMD on Kali Linux

## 📖 Overview

`sqlcmd` is Microsoft's official command-line utility for interacting with Microsoft SQL Server. It can be used to connect to SQL Server instances, execute queries, run SQL scripts, and perform administrative tasks.

---

# 🟢 1. Install SQLCMD on Kali Linux

## 🟩 Import Microsoft GPG Key

```bash
curl https://packages.microsoft.com/keys/microsoft.asc | \
sudo tee /etc/apt/trusted.gpg.d/microsoft.asc
```

---

## 🟩 Add Microsoft Repository

```bash
echo "deb [arch=amd64] https://packages.microsoft.com/debian/12/prod bookworm main" | \
sudo tee /etc/apt/sources.list.d/microsoft-prod.list
```

---

## 🟩 Update Package Lists

```bash
sudo apt update
```

---

## 🟩 Install SQLCMD

```bash
sudo ACCEPT_EULA=Y apt install -y mssql-tools18 unixodbc-dev
```

---

# 🟢 2. Add SQLCMD to PATH

## 🟩 Zsh (Default Kali Shell)

```bash
echo 'export PATH="$PATH:/opt/mssql-tools18/bin"' >> ~/.zshrc

source ~/.zshrc
```

---

## 🟩 Bash

```bash
echo 'export PATH="$PATH:/opt/mssql-tools18/bin"' >> ~/.bashrc

source ~/.bashrc
```

---

# 🟢 3. Verify Installation

```bash
sqlcmd -?
```

or

```bash
which sqlcmd
```

### Expected Output

```text
/opt/mssql-tools18/bin/sqlcmd
```

---

# 🟢 4. Connect to SQL Server

## 🟩 SQL Authentication

```bash
sqlcmd -S 192.168.1.52 -U sa -P Password123
```

---

## 🟩 Specify Port

```bash
sqlcmd -S 192.168.1.52,1433 -U sa -P Password123
```

---

## 🟩 Connect to a Specific Database

```bash
sqlcmd -S 192.168.1.52 \
-U sa \
-P Password123 \
-d master
```

---

# 🟢 5. Execute Queries

## 🟩 Display SQL Server Version

```bash
sqlcmd -S 192.168.1.52 \
-U sa \
-P Password123 \
-Q "SELECT @@VERSION"
```

---

## 🟩 Current Login

```bash
sqlcmd -S 192.168.1.52 \
-U sa \
-P Password123 \
-Q "SELECT SYSTEM_USER"
```

---

## 🟩 List Databases

```bash
sqlcmd -S 192.168.1.52 \
-U sa \
-P Password123 \
-Q "SELECT name FROM sys.databases"
```

---

# 🟢 6. Interactive Mode

## 🟩 Connect

```bash
sqlcmd -S 192.168.1.52 -U sa -P Password123
```

---

## 🟩 Run Queries

```sql
SELECT @@VERSION;
GO

SELECT SYSTEM_USER;
GO

SELECT name
FROM sys.databases;
GO
```

---

## 🟩 Exit

```sql
QUIT
```

or

```sql
EXIT
```

---

# 🟢 7. Execute SQL Script

```bash
sqlcmd -S 192.168.1.52 \
-U sa \
-P Password123 \
-i script.sql
```

---

# 🟢 8. Save Query Output

```bash
sqlcmd -S 192.168.1.52 \
-U sa \
-P Password123 \
-Q "SELECT name FROM sys.databases" \
-o output.txt
```

# 🟢 10. SSL/TLS Certificate Issues

SQL Server 2022 and **sqlcmd 18** enable encryption by default.

---

## 🟩 Trust Server Certificate

```bash
sqlcmd -S 192.168.1.52 \
-U sa \
-P Password123 \
-C
```

---

## 🟩 Disable Encryption

```bash
sqlcmd -S 192.168.1.52 \
-U sa \
-P Password123 \
-No
```

# 🟢 13. Useful SQLCMD Options

| Option | Description |
|---------|-------------|
| `-S` | SQL Server hostname or IP |
| `-U` | SQL Login username |
| `-P` | Password |
| `-d` | Database name |
| `-Q` | Execute a query and exit |
| `-i` | Execute SQL script file |
| `-o` | Save output to a file |
| `-C` | Trust server certificate |
| `-No` | Disable encryption (when supported) |
| `-L` | List discoverable SQL Servers |

# 🟢 15. Key Takeaways

- `sqlcmd` is Microsoft's official CLI for SQL Server.
- SQL Server commonly listens on **TCP 1433**.
- SQLCMD supports both interactive and non-interactive query execution.
- SQLCMD 18 enables TLS encryption by default.
- Alternative clients include **Impacket's mssqlclient** and **SQSH**.
- Use SQLCMD to enumerate databases, users, server information, and permissions in **authorized** environments.
