## SSH Host Keys

SSH host keys are cryptographic keys used by an SSH server to prove its identity to connecting clients. They help prevent Man-in-the-Middle (MITM) attacks by allowing clients to verify that they are communicating with the legitimate server.

When a client connects to an SSH server for the first time, the server presents its host key. The client stores the key fingerprint and verifies it on subsequent connections.

---

## Purpose of SSH Host Keys

SSH host keys provide:

- Server authentication
- Protection against man-in-the-middle attacks
- Host identity verification
- Trust establishment between client and server

> Without host keys, an attacker could impersonate a legitimate SSH server and intercept credentials or session data.
> 

---

## Common SSH Host Key Types

| Key Type | Algorithm | Status |
| --- | --- | --- |
| RSA | RSA | Commonly used |
| ECDSA | Elliptic Curve Digital Signature Algorithm | Supported on modern systems |
| ED25519 | Edwards-Curve Digital Signature Algorithm | Recommended |
| DSA | Digital Signature Algorithm | Deprecated |

### Example Host Key Files

```bash
/etc/ssh/ssh_host_rsa_key
/etc/ssh/ssh_host_ecdsa_key
/etc/ssh/ssh_host_ed25519_key
```

### Public Key Files

```bash
/etc/ssh/ssh_host_rsa_key.pub
/etc/ssh/ssh_host_ecdsa_key.pub
/etc/ssh/ssh_host_ed25519_key.pub
```

# Enumerating SSH Host Keys

## Using Nmap

```bash
nmap --script ssh-hostkey -p 22 <target>
```

### Example Output

```
PORT   STATE SERVICE
22/tcp open  ssh

| ssh-hostkey:
|   256 SHA256:K5M+3mW7rN5d2b3nA8x4eJzj9h4fM4wT7jYkL6N7rJ8 (ECDSA)
|_  256 SHA256:p9XQwzYf8JvQ2mR7vLkF6dC8sA4tN9wX2eP3uR5bT1c (ED25519)
```

## Using ssh-keyscan

### Retrieve All Available Host Keys

```bash
ssh-keyscan <target>
```

### Retrieve Specific Key Types

```bash
ssh-keyscan -t rsa <target>
```

```bash
ssh-keyscan -t ecdsa <target>
```

```bash
ssh-keyscan -t ed25519 <target>
```

### Multiple Types

```bash
ssh-keyscan -t rsa,ecdsa,ed25519 <target>
```

### Example Output

```
192.168.1.10 ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQ...
192.168.1.10 ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIB...
```

# Viewing Local Known Host Keys

Linux and macOS store trusted host keys in:

```bash
~/.ssh/known_hosts
```

### View Entries

```bash
cat ~/.ssh/known_hosts
```

### Search for a Specific Host

```bash
ssh-keygen -F <hostname>
```

---

# Viewing SSH Server Host Keys

### On a Linux Server

```bash
ls -la /etc/ssh/ssh_host_*
```

### Display Fingerprints

```bash
ssh-keygen -lf /etc/ssh/ssh_host_ed25519_key.pub
```

### Example

```
256 SHA256:K5M+3mW7rN5d2b3nA8x4eJzj9h4fM4wT7jYkL6N7rJ8 root@server (ED25519)
```

### Display Fingerprint Using MD5 Format

```bash
ssh-keygen -E md5 -lf /etc/ssh/ssh_host_ed25519_key.pub
```

# Obtaining Host Key Fingerprints Remotely

Using `ssh-keyscan` and `ssh-keygen`:

```bash
ssh-keyscan <target> 2>/dev/null | ssh-keygen -lf -
```

### Example Output

```
256 SHA256:K5M+3mW7rN5d2b3nA8x4eJzj9h4fM4wT7jYkL6N7rJ8
```

# Security Assessment Value

SSH host keys can reveal:

## SSH Implementation

Different SSH servers may generate different host key types:

- OpenSSH
- Dropbear SSH
- Bitvise SSH
- Embedded SSH implementations

---

## Shared Infrastructure

Identical host keys across multiple systems may indicate:

- Cloned virtual machines
- Golden images
- Load-balanced SSH services
- Shared infrastructure

---

## Weak Cryptography

Enumeration may reveal:

- DSA host keys
- Small RSA keys
- Deprecated algorithms
- Legacy SSH implementations

---

## Asset Correlation

Matching host key fingerprints across networks can help identify:

- Previously discovered systems
- Renamed hosts
- Infrastructure relationships

### Identify Weak Algorithms

```bash
nmap --script ssh2-enum-algos -p 22 <target>
```

```bash
ssh-keyscan target > hostkeys.txt
```

### Compare Two Hosts

```bash
ssh-keyscan host1 > host1.keys
ssh-keyscan host2 > host2.keys

diff host1.keys host2.keys
```

## SSH Host Keys Kya Hain?

SSH host keys server ki identity prove karte hain. Jab pehli baar kisi server se connect karte hain toh server apni public key client ko bhejta hai taaki client verify kar sake ki woh sahi server se connect ho raha hai.

## Host Key Verification

Jab pehli baar connect karte hain:

```bash
ssh user@<target>
```

Yeh message aata hai:

```

ED25519 key fingerprint is SHA256:K5M+3mW7rN5d2b3nA8x4eJzj9h4fM4wT7jYkL6N7rJ8.
Are you sure you want to continue connecting (yes/no)?
```

Hamesha fingerprint verify karo trusted channel se pehle accept karne se.


---

## ssh-keygen Tool

```bash
# Key generate karo
ssh-keygen -t ed25519 -f mykey

# Key fingerprint check karo
ssh-keygen -lf /etc/ssh/ssh_host_ed25519_key.pub

# Known hosts verify karo
ssh-keygen -F 192.168.1.10
```

---

## Known Hosts File

```bash
# Location
~/.ssh/known_hosts

# Content dekho
cat ~/.ssh/known_hosts

# Entry add karo manually
ssh-keyscan 192.168.1.10 >> ~/.ssh/known_hosts

# Entry remove karo
ssh-keygen -R 192.168.1.10
```

## Best Practices

```
✅ ED25519 host keys prefer karo
✅ RSA use karna ho toh minimum 3072 bits
✅ Deprecated DSA keys remove karo
✅ Systems rebuild/compromise pe keys rotate karo
✅ New hosts ki fingerprints verify karo
✅ Unexpected host key changes monitor karo
✅ Large environments mein SSH certificates use karo
✅ Host key fingerprints ka inventory maintain karo
```
