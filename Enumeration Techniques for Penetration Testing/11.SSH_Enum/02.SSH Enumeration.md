## SSH Enumeration Kya Hai?

SSH enumeration penetration testing aur security assessments mein ek critical phase hai. Iska objective hai target SSH service ke baare mein information gather karna — version, configuration, supported authentication methods, cryptographic algorithms, aur potential vulnerabilities.

SSH default par TCP port 22 pe run karta hai lekin administrators isse alternative ports pe bhi configure karte hain:

```
22      → Default
2222    → Common alternative
22000   → Custom
22222   → Custom
```

SSH kisi bhi TCP port pe configure ho sakta hai.

## SSH Host Keys — Complete Details

SSH host keys cryptographic keys hain jo SSH server apni identity prove karne ke liye use karta hai. Yeh man-in-the-middle (MITM) attacks se bachate hain.

Jab client pehli baar connect karta hai, server apni host key present karta hai. Client key fingerprint store karta hai aur subsequent connections mein verify karta hai.

**Purpose of SSH Host Keys:**

- Server authentication
- Protection against MITM attacks
- Host identity verification
- Trust establishment between client and server

**Without host keys** ek attacker legitimate SSH server ko impersonate karke credentials ya session data intercept kar sakta hai.

**Common SSH Host Key Types:**

| Key Type | Algorithm | Status |
| --- | --- | --- |
| RSA | RSA | Commonly used |
| ECDSA | Elliptic Curve Digital Signature Algorithm | Supported on modern systems |
| ED25519 | Edwards-Curve Digital Signature Algorithm | Recommended |
| DSA | Digital Signature Algorithm | Deprecated |

---

## Best Practices

```
✅ ED25519 host keys prefer karo
✅ RSA use karna ho toh minimum 3072 bits
✅ Deprecated DSA keys remove karo
✅ Systems rebuild/compromise pe keys rotate karo
✅ New hosts ki fingerprints verify karo
✅ Unexpected host key changes monitor karo
✅ Large environments mein SSH certificates use karo
✅ Host key fingerprints ka inventory maintain karo
```

## Hardening Recommendations

- Use **SSH Protocol Version 2** only.
- Keep **OpenSSH and operating systems updated**.
- **Disable direct root login**.Plaintext
    
    # 
    
    ```
    PermitRootLogin no
    ```
    
- **Prefer public key authentication**.Plaintext
    
    # 
    
    ```
    PubkeyAuthentication yes
    PasswordAuthentication no
    ```
    
- **Restrict access** using `AllowUsers` or `AllowGroups`.Plaintext
    
    # 
    
    ```
    AllowUsers admin security
    ```
    
- **Remove weak ciphers, MACs, and key exchange algorithms**.
- **Implement multi-factor authentication** where possible.
- Use **fail2ban** or equivalent rate-limiting controls.
- **Configure centralized logging** and monitoring.
- **Monitor failed authentication attempts**.
- **Limit exposed SSH services** through firewall rules.
- Consider **moving SSH to a non-standard port** to reduce automated scanning noise.
