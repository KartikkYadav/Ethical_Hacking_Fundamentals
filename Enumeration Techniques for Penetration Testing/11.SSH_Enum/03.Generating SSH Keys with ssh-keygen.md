The `ssh-keygen` command is the standard way to generate SSH key pairs for secure authentication. SSH keys are more secure than passwords and are commonly used for accessing servers, automating scripts, and enabling passwordless logins.

---

# 1. Generate an SSH Key Pair (Interactive)

```bash
ssh-keygen
```

Prompts for:

- Key type
- File location
- Optional passphrase

---

# 2. Generate an Ed25519 Key (Recommended)

Ed25519 is the recommended key type for most environments.

```bash
ssh-keygen -t ed25519
```

---

# 3. Generate an Ed25519 Key with a Comment

The comment helps identify the key later.

```bash
ssh-keygen -t ed25519 -C "james@example.com"
```

---

# 4. Generate a 4096-bit RSA Key

```bash
ssh-keygen -t rsa -b 4096
```

### Options

- `t rsa` → Specifies RSA algorithm
- `b 4096` → Sets key size to 4096 bits

> While 2048-bit RSA is still supported, 4096-bit RSA is generally preferred when RSA is required.
> 

---

# 5. Generate a Key Without a Passphrase

```bash
ssh-keygen -t ed25519 -q -N ""
```

### Options

- `q` → Quiet mode
- `N ""` → Empty passphrase

> Useful for automation but not recommended for sensitive accounts.
> 

---

# 6. Generate a Key at a Custom Location

```bash
ssh-keygen -t ed25519 -f /home/james/.ssh/id_ed25519 -q -N ""
```

### Option

- `f` → Output file path

Public key will be created as:

```bash
/home/james/.ssh/id_ed25519.pub
```

---

# 7. Generate a Key with a Custom Passphrase

```bash
ssh-keygen -t ed25519 -f /root/.ssh/id_ed25519 -q -N "armour123"
```

The private key will be encrypted using the specified passphrase.

---

# 8. Display a Public Key

```bash
cat ~/.ssh/id_ed25519.pub
```

The output can be copied into a remote server's `authorized_keys` file.

---

# 9. Copy a Public Key to a Remote Server

## Using ssh-copy-id (Recommended)

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server
```

This automatically appends the public key to the remote user's `authorized_keys` file and sets proper permissions.

---

## Manual Method

```bash
cat ~/.ssh/id_ed25519.pub >> ~/.ssh/authorized_keys
```

Or copy the contents of the public key into:

```bash
~/.ssh/authorized_keys
```

on the target server.

---

# 10. Change a Key Passphrase

```bash
ssh-keygen -p -f ~/.ssh/id_ed25519
```

Prompts for:

- Current passphrase
- New passphrase

---

# 11. Show Key Fingerprint

```bash
ssh-keygen -lf ~/.ssh/id_ed25519.pub
```

### Example Output

```
256 SHA256:XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX user@example.com (ED25519)
```

---

# 12. Export a Public Key to RFC4716 Format

```bash
ssh-keygen -e -f ~/.ssh/id_ed25519.pub
```

---

# 13. Import an RFC4716 Public Key

```bash
ssh-keygen -i -f publickey.txt
```

---

# 14. Generate a 2048-bit RSA Key (Interactive)

```bash
ssh-keygen -b 2048 -t rsa
```

### Options

- `b 2048` → Key size
- `t rsa` → RSA algorithm

---

# 15. Generate a 2048-bit RSA Key Without Passphrase

```bash
ssh-keygen -b 2048 -t rsa -q -N ""
```

### Options

- `q` → Quiet mode
- `N ""` → Empty passphrase

---

# 16. Generate RSA Key at a Custom Location

```bash
ssh-keygen -b 2048 -t rsa -f /home/james/.ssh/id_rsa -q -N ""
```

---

# 17. Generate RSA Key with a Custom Passphrase

```bash
ssh-keygen -b 2048 -t rsa -f /root/.ssh/id_rsa -q -N "armour123"
```

The private key will be encrypted using the specified passphrase.

---

# 18. Authorize Public Key for SSH Login

```bash
cd ~/.ssh
```

```bash
cp id_rsa.pub authorized_keys
```

Place the public key contents into the target account's:

```bash
~/.ssh/authorized_keys
```

to allow login with the corresponding private key.

# 🔒 Recommended SSH Permissions

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
chmod 600 ~/.ssh/authorized_keys
```

---

# ✅ Best Practices

- Prefer **Ed25519** keys unless RSA is specifically required.
- Use a strong passphrase to protect private keys.
- Never share private keys.
- Use `ssh-copy-id` instead of manually editing `authorized_keys` whenever possible.
- Regularly rotate keys used for privileged or automated access.
- Verify host fingerprints before connecting to new systems.
- Store backup copies of important SSH keys securely.
- Use separate SSH keys for different environments (Lab, Work, Production).

---

**OSCP Tip:** During labs and exams, you'll commonly use:

```bash
ssh-keygen -t ed25519
ssh-copy-id user@target
ssh -i ~/.ssh/id_ed25519 user@target
```

These three commands cover most SSH key authentication scenarios.
