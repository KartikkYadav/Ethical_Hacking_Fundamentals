## Overview

SSH se OS commands run karna remote administration, post-exploitation, automation, ya CTF/red teaming exercises ke liye fundamental hai.

---

## .bashrc aur .zshrc Kya Hain?

`.bashrc` aur `.zshrc` dono configuration files hain jo user ke home directory mein hoti hain aur shell environment ka behaviour customize karti hain.

---

## .bashrc (Bash Shell Ke Liye)

**Purpose:**

- Bash har baar run karta hai jab tum new interactive non-login shell start karte ho (jaise new terminal window/tab)
- User-specific environment variables, shell options, aliases, prompt styles, aur shell functions set karne ke liye use hota hai
- Automatically execute hota hai taaki har naye Bash session mein settings effect mein aayein

**Typical Customizations:**

- Aliases (jaise `alias ll='ls -la'`)
- Prompt settings (`PS1`)
- Environment variables export karna (`export EDITOR=vim`)
- Colors, shell behaviour configure karna, aur other scripts load karna

```bash
vim .bashrc
```

**Example .bashrc:**

```bash
# .bashrc

# Source global definitions
if [ -f /etc/bashrc ]; then
    . /etc/bashrc
fi

# User specific environment
if ! [[ "$PATH" =~ "$HOME/.local/bin:$HOME/bin:" ]]
then
    PATH="$HOME/.local/bin:$HOME/bin:$PATH"
fi
export PATH
export MY_VAR="persistent value"

# User specific aliases and functions
alias rm='rm -i'
alias cp='cp -i'
alias mv='mv -i'

GRC_ALIASES=true
[[ -s "/etc/profile.d/grc.sh" ]] && source /etc/grc.sh
```

## .zshrc (Zsh Shell Ke Liye)

`.zshrc` tab load hota hai jab bhi new interactive Zsh shell start hoti hai. Yeh `.bashrc` ka equivalent hai Z shell ke liye lekin even more advanced customization ke saath, often Oh My Zsh jaise frameworks se enhanced.

**Common Uses:**

- Themes
- Plugins
- Aliases
- Environment variables
- Autocompletion settings
- Syntax highlighting

```bash
vim ~/.zshrc
```

**Typical .zshrc Configuration:**

```bash
export PATH="$HOME/bin:$PATH"
alias ll='ls -lah'
alias gs='git status'
ZSH_THEME="agnoster"
```

## Common Configuration Examples Table

| Purpose | Example |
| --- | --- |
| Alias | `alias gs='git status'` |
| Prompt customization | `PS1='[\u@\h \W]\$ '` |
| Environment variables | `export PATH="$HOME/bin:$PATH"` |
| Zsh theme | `ZSH_THEME="agnoster"` |
| Bash options | `shopt -s histappend` |
| Zsh completion | `autoload -Uz compinit && compinit` |

## Which Should You Use?

- Bash main shell hai toh `~/.bashrc` customize karo
- Zsh use karte ho toh `~/.zshrc` customize karo
- Shells switch karte ho toh relevant settings copy karo corresponding file mein, lekin syntax differences ka dhyan rakho

**Summary:** `.bashrc` aur `.zshrc` startup files hain jo shell environment setup automate karti hain. Yeh aliases define karne, variables set karne, functions load karne, aur features add karne deti hain — automatically har naye shell session ke start mein.

## Ways to Run Commands Remotely over SSH

---

## 1. Single Command Run Karo

Remote command execute karo aur output locally dikhao:

```bash
# Hostname retrieve karo
ssh root@192.168.1.41 hostname

# User info display karo
ssh root@192.168.1.41 id

# File read karo
ssh root@192.168.1.41 "cat /root/.bashrc"

# Legacy RSA key support ke saath
ssh -oHostKeyAlgorithms=+ssh-rsa bee@192.168.1.235 'cat /home/bee/.bashrc'
ssh -oHostKeyAlgorithms=+ssh-rsa bee@192.168.1.235 id
```

---

## 2. Multiple Commands Run Karo

```bash
# Semicolon se multiple commands
ssh user@192.168.1.10 "uname -a && whoami"
```

---

## Reading Remote Files

Configuration files remotely inspect karo:

```bash
# User database dekho
ssh armour@192.168.1.230 "cat /etc/passwd"

# Bash configuration read karo
ssh armour@192.168.1.230 "cat ~/.bashrc"

# Shell history dekho
ssh armour@192.168.1.230 "cat ~/.bash_history"
```

## Transferring Files

---

## Upload Files with SCP

```bash
# Local file remote system pe copy karo
scp local-file.txt user@192.168.1.10:/tmp/

# Bash configuration upload karo
scp bashrc.txt root@192.168.1.41:/root/.bashrc
```

---

## Download Files with SCP

```bash
# Remote file locally download karo
scp user@192.168.1.10:/etc/passwd ./passwd

# Directory download karo
scp -r user@192.168.1.10:/home/user/ ./backup/
```

---

## Execute Commands with Private Key

```bash
# Private key specify karke command run karo
ssh -i id_rsa user@192.168.1.10 "whoami"

# Private key se file read karo
ssh -i id_rsa user@192.168.1.10 "cat /etc/shadow"
```

---

## Pentesting Mein SSH Commands Ka Use

```bash
# Quick enumeration
ssh user@TARGET "id; whoami; uname -a; cat /etc/passwd"

# Sudo check
ssh user@TARGET "sudo -l"

# SUID files
ssh user@TARGET "find / -perm -4000 2>/dev/null"

# Network info
ssh user@TARGET "netstat -ano; ifconfig"

# Cron jobs
ssh user@TARGET "cat /etc/crontab; ls -la /etc/cron*"

# Interesting files
ssh user@TARGET "find / -name '*.txt' 2>/dev/null | head -20"
```
