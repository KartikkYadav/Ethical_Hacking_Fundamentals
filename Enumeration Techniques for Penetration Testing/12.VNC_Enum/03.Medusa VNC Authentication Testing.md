When using Medusa against a VNC service, you may encounter the following error:

```
ALERT: User logon information must be supplied.
```

Although VNC authentication typically does not use usernames, Medusa requires a username parameter to be provided for all modules.

---

## Basic Usage

Provide a dummy username along with the password list:

```
medusa -h 192.168.10.110 -u dummy -P /opt/password.txt -M vnc
```

You can use any placeholder username:

```
medusa -h 192.168.10.110 -u vnc -P /opt/password.txt -M vnc
```

---

## View VNC Module Options

Display help and supported options for the VNC module:

```
medusa -M vnc -q
```

---

## Specify a Non-Default VNC Port

- By default, VNC commonly listens on TCP port `5900`.
- If the service is running on a different port, specify it using the `n` option:

```
medusa -h 192.168.10.110 -u dummy -P /opt/password.txt -M vnc -n 5901
```

---

## Verify the VNC Service

Before testing authentication, confirm that a VNC service is running and identify the correct port:

```
nmap -sV -p 5900-5905 192.168.10.110
```

Example output:

```
PORT       STATE  SERVICE  VERSION
5900/tcp   open   vnc      VNC protocol 3.8
```

---

## Common Issue

The following command fails because no username was supplied:

```
medusa -h 192.168.10.110 -P /opt/password.txt -M vnc
```

Result:

```
ALERT: User logon information must be supplied.
```

---

## Correct Command

```
medusa -h 192.168.10.110 -u dummy -P /opt/password.txt -M vnc
```

---

## Summary

| Option | Description |
| --- | --- |
| `-h` | Target host |
| `-u` | Username (required by Medusa, even for VNC) |
| `-P` | Password wordlist |
| `-M vnc` | Use the VNC module |
| `-n` | Specify a non-default port |
| `-q` | Display module-specific help |

---

## Example Workflow

1. Identify the VNC service and port:

```
nmap -sV -p 5900-5905 192.168.10.110
```

1. Review module options:

```
medusa -M vnc -q
```

1. Run the authentication test:

```
medusa -h 192.168.10.110 -u dummy -P /opt/password.txt -M vnc
```
