# VNC Version Detection

Detecting the version of a VNC server is an important part of service enumeration and security assessment. The following techniques can be used to identify the VNC protocol version, authentication methods, and server implementation details.

**Example Target:** Assume a VNC server is running on:

```
192.168.1.235:5901
```

---

## 1. Banner Grabbing with Netcat

VNC servers typically send a protocol banner immediately after a TCP connection is established.

**Basic Connection:**

```
nc 192.168.1.235 5901
```

**Verbose Mode:**

```
nc -v 192.168.1.235 5901
```

**Example Output:**

```
Connection to 192.168.1.235 5901 port [tcp/*] succeeded!
RFB 003.008
```

**Common Protocol Versions:**

| Banner | Protocol Version |
| --- | --- |
| `RFB 003.003` | VNC 3.3 |
| `RFB 003.007` | VNC 3.7 |
| `RFB 003.008` | VNC 3.8 |

---

## 2. Nmap Service Detection

**Basic Version Detection:**

```
nmap -sV -p 5901 192.168.1.235
```

This attempts to identify:

- Service type
- VNC implementation
- Version information

---

## 3. Nmap VNC Information Script

Use the dedicated NSE script for VNC enumeration:

```
nmap -v -sT -sV --script=vnc-info.nse -p 5901 192.168.1.235
```

**Example Output:**

```
5901/tcp open  vnc
| vnc-info:
|   Protocol version: 3.8
|   Security types:
|       VNC Authentication (2)
|_
```

Information may include:

- Protocol version
- Authentication methods
- Security types
- Server capabilities

---

## 4. Aggressive Service Enumeration

Perform comprehensive fingerprinting:

```
nmap -v -sT -sV -A --script=vnc-info.nse -p 5901 192.168.1.235
```

**Options Explained:**

| Option | Description |
| --- | --- |
| `-sT` | TCP connect scan |
| `-sV` | Service version detection |
| `-A` | Aggressive scan |
| `--script=vnc-info.nse` | Run VNC information script |
| `-p` | Target port |

---

## 5. Run All VNC NSE Scripts

Execute every available VNC-related NSE script:

```
nmap -v -sV -sT --script=vnc* -p 5901-5910 192.168.1.235
```

Scan a single port:

```
nmap -v -sV -sT --script=vnc* -p 5903 192.168.1.31
```

---

## 6. Manual Testing with VNC Viewer

A VNC client can sometimes reveal additional information during connection attempts.

- Connect to a VNC Server:

```
vncviewer 192.168.1.235:5901
```

- Display Help:

```
vncviewer -h
```

- Use a Password File:

```
vncviewer -passwd vncpassword.txt 192.168.1.235:5901
```

This can help verify:

- Authentication prompts
- Connection behavior
- Encryption support
- Protocol negotiation

---

## 7. Additional Enumeration with Nmap

- Scan multiple common VNC ports:

```
nmap -sV -p 5900-5910 192.168.1.235
```

- Scan all open TCP ports and identify services:

```
nmap -sV -p- 192.168.1.235
```

---

## Summary

| Command | Purpose |
| --- | --- |
| `nc -v <target> <port>` | Grab VNC protocol banner |
| `nmap -sV -p <port> <target>` | Service version detection |
| `nmap --script=vnc-info.nse` | Enumerate VNC details |
| `nmap --script=vnc*` | Run all VNC NSE scripts |
| `vncviewer <target>:<port>` | Manual testing and validation |

---

## Key Findings

- Netcat provides a quick method for retrieving the VNC protocol banner.
- Nmap service detection can identify VNC implementations and versions.
- The `vnc-info.nse` script provides protocol and authentication details.
- Running all VNC-related NSE scripts may uncover additional server information.
- VNC viewers can be useful for validating server behavior and authentication mechanisms.

---

## Example Workflow

1. Verify the port is open:

```
nc -v 192.168.1.235 5901
```

1. Perform service detection:

```
nmap -sV -p 5901 192.168.1.235
```

1. Gather VNC-specific information:

```
nmap --script=vnc-info.nse -p 5901 192.168.1.235
```

1. Validate manually:

```
vncviewer 192.168.1.235:5901
```

Using these techniques together provides an effective approach to VNC service enumeration and version identification during security assessments.
