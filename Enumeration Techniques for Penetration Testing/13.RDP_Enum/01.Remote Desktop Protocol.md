**Remote Desktop Protocol (RDP)** enumeration is the process of identifying, analyzing, and collecting information about systems exposing the RDP service (typically TCP/3389). The objective is to discover configuration weaknesses, authentication settings, exposed services, and potential vulnerabilities.

---

## 1. Identify Open RDP Services

### Basic Service Detection

```
nmap -Pn -p 3389 -sV <target_ip>
```

```
nmap -v -sT -sV -sC -A -Pn -p 3389 192.168.10.52
```

Example:

```
nmap -Pn -p 3389 -sV 192.168.1.42
```

### Full TCP Scan

- Useful when RDP is configured on a non-standard port.

```
nmap -Pn -p- --min-rate 1000 <target_ip>
```

### Search for RDP Across a Network

```
nmap -Pn -p 3389 192.168.1.0/24
```

---

## 2. Gather RDP Configuration Information

### Enumerate Encryption Settings

```
nmap -p 3389 --script rdp-enum-encryption <target_ip>
```

Example Output:

```
Security layer
    CredSSP (NLA): SUCCESS
    Native RDP: SUCCESS
    SSL: SUCCESS
```

This reveals:

- Supported security layers
- TLS support
- CredSSP availability
- NLA status

### Obtain NTLM Information

```
nmap -p 3389 --script rdp-ntlm-info <target_ip>
```

Information disclosed may include:

- NetBIOS Name
- DNS Name
- DNS Domain

Example:

```
Target_Name: WIN-SERVER
NetBIOS_Domain_Name: LAB
DNS_Computer_Name: WIN-SERVER.lab.local
Product_Version: 10.0.17763
```

### RDP Screenshot Capture

Sometimes login screens reveal hostnames or domain names.

```
nmap -p3389 --script rdp-screenshot <target_ip>
```

---

## 3. Analyze TLS Configuration

### SSLScan

```
sslscan <target_ip>:3389
```

### TestSSL

```
testssl.sh <target_ip>:3389
```

Review for:

- Weak ciphers
- Deprecated TLS versions
- Self-signed certificates
- Certificate details

## 4. Vulnerability Detection

```
msfconsole
```

```
use auxiliary/scanner/rdp/cve_2019_0708_bluekeep
```

```
set RHOSTS <target_ip>
```

```
run
```

### MS12-020 Detection

```
nmap --script rdp-vuln-ms12-020 -p3389 <target_ip>
```

## 5. RDP Authentication Enumeration

### Check NLA Status

```
nmap -p3389 --script rdp-enum-encryption <target_ip>
```

Indicators:

| Result | Meaning |
| --- | --- |
| CredSSP Supported | NLA Enabled |
| Native RDP Only | NLA Disabled |
| SSL Only | TLS-Based Authentication |

### Using FreeRDP for Validation

```
xfreerdp /v:192.168.1.42 /cert-ignore
```

- If NLA is required, FreeRDP will indicate it before authentication.

---

## 6. Username Validation

Some environments expose information through authentication responses.

- Using Kerberos:

```
kerbrute userenum -d lab.local users.txt dc.lab.local
```

- Using SMB:

```
crackmapexec smb <target_ip> -u users.txt -p ''
```

> This is often safer and more reliable than directly targeting RDP.
>

## 7. Password Auditing (Authorized Testing Only)

### Hydra

```
hydra -l Administrator -P passwords.txt rdp://192.168.1.42
```

### Ncrack

```
ncrack -p 3389 -U users.txt -P passwords.txt 192.168.1.42
```

### Account Lockout Warning

Before any password auditing:

- Check lockout policy
- Use low request rates
- Coordinate with administrators
- Avoid production impact

## 9. Connect to RDP

### xfreerdp

- Standard Login:

```
xfreerdp /u:administrator /p:password /v:192.168.1.42
```

- Full Screen:

```
xfreerdp /u:administrator /p:password /v:192.168.1.42 /f
```

- Ignore Certificate Warnings:

```
xfreerdp /u:administrator /p:password /v:192.168.1.42 /cert-ignore
```

- Pass-the-Hash (when authorized and supported):

```
xfreerdp /u:administrator /pth:<NTLM_HASH> /v:192.168.1.42
```

### rdesktop

```
rdesktop -u administrator -p password 192.168.1.42
```

- Specify Resolution:

```
rdesktop -u administrator -p password -g 1920x1080 192.168.1.42
```

## 11. Post-Enumeration Review

Document:

| Item | Review |
| --- | --- |
| RDP Version | Current or obsolete |
| TLS Configuration | Strong or weak |
| NLA Enabled | Yes/No |
| Authentication Methods | NTLM, Kerberos, CredSSP |
| Certificate Status | Trusted or self-signed |
| Lockout Policy | Present or absent |
| Vulnerabilities | BlueKeep, MS12-020 |
| Exposure | Internet-facing or internal |

## Quick Enumeration Workflow

- Service Detection:

```
nmap -Pn -sV -p3389 <target>
```

- Encryption Enumeration:

```
nmap -p3389 --script rdp-enum-encryption <target>
```

- NTLM Information:

```
nmap -p3389 --script rdp-ntlm-info <target>
```

- TLS Review:

```
sslscan <target>:3389
```

- Vulnerability Checks:

```
nmap --script rdp-vuln-ms12-020 -p3389 <target>
```

This workflow provides a comprehensive RDP assessment suitable for authorized penetration testing, security auditing, and defensive validation.
