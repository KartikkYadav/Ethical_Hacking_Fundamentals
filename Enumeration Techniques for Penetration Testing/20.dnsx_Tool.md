# OneForAll
OneForAll is a powerful and fast subdomain enumeration tool designed for security professionals.
​

### Cloning the Repository
- To install OneForAll, first, clone the repository from GitHub:

       git clone https://github.com/shmilylty/OneForAll.git

### Navigating to the OneForAll Directory

- Change into the cloned directory:

      cd OneForAll

### Installing Dependencies

- Install all required Python dependencies using pip:
      
      pip install -r requirements.txt

### Verifying Installation

- Check if OneForAll is installed correctly by displaying the help menu:

      python oneforall.py --help

### Running OneForAll for Subdomain Enumeration

- To enumerate subdomains for a target domain (e.g., tesla.com), run:
​
      python3 oneforall.py --target tesla.com run

This command will scan for subdomains and save the results in the default results directory.

### Extracting Subdomains from Results

- After the scan, you can extract subdomains from the results using:

      cat /opt/OneForAll/results/tesla.com.csv | cut -d "," -f6 | grep -v "^subdomain$"

- cut -d "," -f6: Extracts the 6th column from the CSV file.

- grep -v "^subdomain$": Filters out the header row.

### Customizing the Output Path
- To save the results in a custom directory (e.g., /tmp), use:

      python3 oneforall.py --target tesla.com run --path /tmp

### Additional Usage Examples

1 Enumerate multiple domains from a file (domains.txt):

    python3 oneforall.py --target-file domains.txt run

2 Use verbose mode for more detailed output:

    python3 oneforall.py --target tesla.com run --verbose

3 Save output in JSON format:

    python3 oneforall.py --target tesla.com run --format json

# dnsx
dnsx is a fast and multi-purpose DNS toolkit that allows running multiple DNS queries using a list of user-supplied resolvers.
​

### Installing dnsx
Using go install:

    go install -v github.com/projectdiscovery/dnsx/cmd/dnsx@latest

### Using go get (for older Go versions)

    GO111MODULE=on go get -v github.com/projectdiscovery/dnsx/cmd/dnsx

### Moving the Binary to a System Path

    cp -v /root/go/bin/dnsx /usr/local/bin/

### Verifying Installation

- Check if dnsx is installed correctly

      dnsx

- Display dnsx version

      dnsx -version

### Running dnsx Queries
- Reverse DNS Lookup (PTR Records)

- Single IP Lookup

      ​echo "8.8.8.8" | dnsx -silent -resp-only -ptr

- Bulk Reverse Lookup from a File
​
      vim  ip-list.txt
  
      8.8.8.8
      8.8.4.4
      64.6.64.6
      1.1.1.1


      dnsx -l ip-list.txt -silent -resp-only -ptr

- Scanning CIDR Ranges

      echo "8.8.8.0/16" | dnsx -silent -resp-only -ptr
      echo "106.201.192.0/23" | dnsx -silent -resp-only -ptr
      echo "82.196.42.196/28" | dnsx -silent -resp-only -ptr
      echo "82.196.42.0/24" | dnsx -silent -resp-only -ptr

### Filtering Dead Record
     cat tesla_domain.txtt
     cat tesla_domain.txt | dnsx -resp

### Extracting A Records.

    cat tesla_domain.txt | dnsx -silent -a -resp
    dnsx -l tesla-all_domains.txt -silent -a -resp
    cat tesla_domain.txt | dnsx -silent -a -resp-only
    dnsx -l tesla-all_domains.txt -silent -a -resp-only
    dnsx -l airbnb.com.txt -silent -a -resp

### Querying Multiple Record Types

    dnsx -l airbnb.com.txt -silent -a -aaaa -ns -ptr -resp -cname
    dnsx -l tesla-all_domains.txt -silent -a -aaaa -ns -ptr -resp -cname

### Extracting CNAME Records
​   
      dnsx -l tesla-all_domains.txt -silent -cname -resp-only
      cat tesla_domain.txt | dnsx -silent -cname -resp
      cat tesla_domain.txt | dnsx -silent -cname -resp-only
      cat tesla_domain.txt | dnsx -silent -cname -rcode noerror,servfail,refused
      dnsx -l tesla-all_domains.txt -silent -cname -rcode noerror,servfail,refused
      dnsx -l tesla-all_domains.txt -silent -cname -rcode noerror,servfail,refused -r /opt/resolvers.txt
        
  ### Querying NS, MX, and TXT Records
  Specific queries for Name Server, Mail Exchange, and Text records:
​

      dnsx -l tesla_domain.txt -ns -resp
      dnsx -l tesla_domain.txt -mx -resp
      dnsx -l tesla_domain.txt -txt -resp

#### Extracting Subdomains from Network Ranges
Uses PTR records to identify hostnames associated with an IP range:

    ​echo 8.21.0.0/16 | dnsx -silent -resp-only -ptr

### Using Custom Resolvers
Custom resolvers improve accuracy, reduce rate limiting, and avoid poisoned DNS responses.
       
        Download Resolver List
        wget https://raw.githubusercontent.com/blechschmidt/massdns/master/lists/resolvers.txt

### Saving Output

    Stores resolved DNS data for later analysis or reporting.
​

### Saving CNAME Records


      dnsx -l tesla-all_domains.txt -silent -cname -r /opt/resolvers.txt -o tesla-cname.txt
      dnsx -l tesla-all_domains.txt -silent -cname -resp-only -r /opt/resolvers.txt -o tesla-cname.txt

### Output in JSON Format
    Exports results in structured JSON format for automation and parsing.
    ​dnsx -l tesla-all_domains.txt -silent -cname -resp-only -r /opt/resolvers.txt -json -o tesla-cname.json
