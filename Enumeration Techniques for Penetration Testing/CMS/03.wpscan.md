# WPScan — WordPress Security Scanner Notes

---

# 📌 Overview

WPScan is a WordPress security scanner used to:

- Detect WordPress version
- Enumerate users
- Find plugins & themes
- Identify vulnerabilities
- Perform brute-force attacks

---

# ⚙️ Installation & Setup

## Install

    apt install wpscan

---

## Check Version

    wpscan --version

---

## Update Database (IMPORTANT)

    wpscan --update

---

# 🌐 Basic Scanning

## HTTP Scan

    wpscan --url http://target

---

## HTTPS Scan

    wpscan --url https://target

---

## Disable SSL Check (Lab / Self-signed)

    wpscan --disable-tls-checks --url https://target

---

## Internal Host Scan

    wpscan --disable-tls-checks --url https://192.168.1.1

---

# 🔥 Detection Modes

## Aggressive Mode

    wpscan --url https://target --detection-mode aggressive

👉 Finds:
- Hidden plugins
- More vulnerabilities

---

## Aggressive + Threads

    wpscan --url https://target --detection-mode aggressive -t 200

👉 Faster scanning using threads

---

# 👤 User Enumeration

## Basic User Enum

    wpscan --url https://target --enumerate u

---

## Range Enumeration

    wpscan --url https://target --enumerate u1-1000

---

## With Threads

    wpscan --url https://target --enumerate u1-1000 -t 200

---

# 🔑 Password Brute Force

## Basic

    wpscan --url http://target -U admin -P rockyou.txt

---

## With Threads

    wpscan --url https://target -U admin -P rockyou.txt -t 100

---

## Example

    wpscan --url https://target -U bob -P /opt/rockyou.txt

---

## Parameters

| Flag | Meaning |
|------|--------|
| -U | Username |
| -P | Password list |
| -t | Threads |

---

# 🔌 Plugin Enumeration

## Vulnerable Plugins

    wpscan --url https://target --enumerate vp

---

## All Plugins

    wpscan --url https://target --enumerate ap

---

## Aggressive Plugin Detection

    wpscan --url https://target --enumerate ap --plugins-detection aggressive

---

## Plugin Version Detection

    wpscan --url https://target --enumerate ap \
    --plugins-detection aggressive \
    --plugins-version-detection aggressive

---

# 🎨 Theme Enumeration

## Vulnerable Themes

    wpscan --url https://target --enumerate vt

---

## All Themes

    wpscan --url https://target --enumerate at

---

# 🔥 Full Enumeration

## Combo Scan

    wpscan --url https://target --enumerate ap,at,cb,dbe

---

## With Aggressive Plugin Detection

    wpscan --url https://target --enumerate ap,at,cb,dbe \
    --plugins-detection aggressive \
    --plugins-version-detection aggressive

---

## Short Form

    wpscan --url https://target -e ap,at,u

---

## Flags Meaning

| Flag | Meaning |
|------|--------|
| ap | All plugins |
| at | All themes |
| u | Users |
| cb | Config backups |
| dbe | Database exports |

---

# 📁 Output Results

## Save Plugin Results

    wpscan --url https://target \
    --enumerate ap \
    --plugins-detection aggressive \
    --plugins-version-detection aggressive \
    -o plugins.txt

---

## Save Full Scan

    wpscan --url https://target \
    --enumerate ap,at,cb,dbe \
    --plugins-detection aggressive \
    --plugins-version-detection aggressive \
    -o full_scan.txt

---

# 🚀 Practical Workflow

1. Basic scan

    wpscan --url http://target

---

2. User enumeration

    wpscan --url http://target -e u

---

3. Plugin + theme enumeration

    wpscan --url http://target -e ap,at

---

4. Aggressive scan

    wpscan --url http://target \
    --detection-mode aggressive \
    -e ap,at,u

---

5. Brute force (if allowed)

    wpscan --url http://target -U admin -P rockyou.txt

---

# ⚠️ Important Notes

- XML-RPC may be used automatically for brute force
- Aggressive mode = noisy
- Threads too high = detection risk
- Always check scope before brute force

---

# 💣 Attack Chain Example

    User Enum → admin
    +
    XML-RPC enabled
    +
    Weak password
    =
    Account Takeover

# 🧠 Pro Tips

- Start small, then go aggressive
- Save outputs for reporting
- Focus on plugins (most vulnerable)
- Combine with manual testing
