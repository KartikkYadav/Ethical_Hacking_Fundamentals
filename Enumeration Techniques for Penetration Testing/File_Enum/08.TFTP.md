# 🟢 Directory Setup For TFTP Boot

## Navigate To TFTP Root Directory

```bash
cd /var/lib/tftpboot/
```

---

## Verify Directory Exists

```bash
ls -lha /var/lib/ | grep "tftpboot"
```

---

## Copy Test Files

```bash
cp -v /var/log/* /var/lib/tftpboot/
```

---

## Set Permissions (Testing Only)

```bash
chmod -R 777 /var/lib/tftpboot/
```

> ⚠️ `777` permissions are unsafe for production environments.

# 🟢 Systemd Configuration For TFTP

## Copy Default Service File

```bash
cp -v /usr/lib/systemd/system/tftp.service /etc/systemd/system/tftp-server.service
```

---

## Copy Default Socket File

```bash
cp -v /usr/lib/systemd/system/tftp.socket /etc/systemd/system/tftp-server.socket
```

---

## Review TFTP Daemon Documentation

```bash
man in.tftpd
```

---

# 🟢 TFTP Server Service File

## Edit Service File

```bash
vim /etc/systemd/system/tftp-server.service
```

---

## Service Configuration

```ini
[Unit]
Description=Tftp Server
Requires=tftp-server.socket
Documentation=man:in.tftpd

[Service]
ExecStart=/usr/sbin/in.tftpd -c -p -s /var/lib/tftpboot
StandardInput=socket

[Install]
WantedBy=multi-user.target
Also=tftp-server.socket
```

---

# 🟢 TFTP Server Socket File

## Edit Socket File

```bash
vim /etc/systemd/system/tftp-server.socket
```

---

## Socket Configuration

```ini
[Unit]
Description=Tftp Server Activation Socket

[Socket]
ListenDatagram=69
BindIPv6Only=both

[Install]
WantedBy=sockets.target
```

# 🟢 Starting And Enabling The TFTP Server

## Restart TFTP Socket

```bash
systemctl restart tftp-server.socket
```

---

## Enable TFTP Service On Boot

```bash
systemctl enable tftp-server
```

# 🟢 Firewall Configuration Using Firewalld

## Check Firewalld Status

```bash
systemctl status firewalld
```

---

## Start Firewalld

```bash
systemctl start firewalld
```

---

## Enable Firewalld

```bash
systemctl enable firewalld
```

---

## Allow TFTP Service

```bash
firewall-cmd --zone=public --add-service=tftp --permanent
```

---

## Or Open UDP Port 69 Directly

```bash
firewall-cmd --zone=public --add-port=69/udp --permanent
```

---

## Reload Firewall Rules

```bash
firewall-cmd --reload
```

---

## Verify Firewall Configuration

```bash
firewall-cmd --zone=public --list-all
```

# ✅ Expected Result

You should see:

```text
in.tftpd listening on 0.0.0.0:69
```

or:

```text
[::]:69
```

indicating that the TFTP server is successfully listening on UDP port 69.

# ⚠️ Security Notes

TFTP:

- Uses UDP
- Has no authentication
- Has no encryption
- Sends files in plaintext

Because of this, TFTP should only be used:

- Inside trusted/internal networks
- For PXE booting
- Firmware transfers
- Embedded devices

---

# 🛡 Recommended Security Practices

- Restrict access with firewall rules
- Avoid world-writable permissions
- Use isolated VLANs/subnets
- Monitor logs and transfers
- Prefer SCP/SFTP for sensitive transfers

---

# ⚖️ Legal Disclaimer

Use TFTP services responsibly and securely within authorized environments.

Misconfigured TFTP servers can expose sensitive files and create security risks.
