# 🟢 What Is TFTP?

TFTP (Trivial File Transfer Protocol) is a lightweight file transfer protocol commonly used in:

- PXE boot environments
- Network device management
- Firmware transfers
- Configuration backups

---

# 🟣 Key Characteristics

- Uses **UDP**
- Default port:

```text
69/UDP
```

> ⚠️ Port `8099` is non-standard and environment-specific.

---

# 🔹 Features

- ❌ No authentication
- ❌ No encryption
- ❌ No directory listing by default
- Lightweight and simple

---

# 🟢 Common Uses

- Firmware uploads/downloads
- Router/switch configuration backups
- PXE boot files
- Embedded device management

---

# 🔴 Security Note

TFTP is inherently insecure.

It should never be exposed to:

- Public internet
- Untrusted networks
- Sensitive environments

---

# 🟢 Enumeration Objectives

During penetration testing, TFTP may allow:

- Downloading sensitive files
- Uploading malicious files
- Accessing configuration backups
- Identifying firmware images
- Learning internal infrastructure details

# 🟢 Enumeration Tools & Commands

# 🟣 Nmap NSE Scripts

## 🔸 Basic UDP Scan

```bash
nmap -v -sU -p 69 <target-ip>
```

---

## 🔸 Detect TFTP Version

```bash
nmap -sU -p 69 --script=tftp-version <target-ip>
```

---

## 🔸 Enumerate Accessible Files

```bash
nmap -sU -p 69 --script=tftp-enum <target-ip>
```

---

## 🔸 Use Custom File List

```bash
nmap -sU -p 69 --script=tftp-enum \
--script-args tftp-enum.filelist=customlist.txt <target-ip>
```

---

# 🟣 TFTP CLI Client

## 🔸 Connect to Server

```bash
tftp <target-ip>
```

---

## 🔸 Enable Verbose Mode

```bash
tftp> verbose
```

---

## 🔸 Download File

```bash
tftp> get <filename>
```

---

## 🔸 Upload File

```bash
tftp> put <filename>
```

---

## 🔸 Example

```bash
tftp 192.168.1.100
tftp> get config
```

---

# 🟣 atftp (Advanced TFTP Client)

## 🔸 Download File

```bash
atftp --get -r <remote_file> -l <local_file> <target-ip>
```

---

## 🔸 Example

```bash
atftp --get -r backup.txt -l backup.txt 192.168.1.100
```

---

## 🔸 Upload File

```bash
atftp --put -r <remote_file> -l <local_file> <target-ip>
```

---

## 🔸 Example

```bash
atftp --put -r php-reverse-shell.php \
-l /opt/web-shells/php-reverse-shell.php 192.168.1.32
```

# ⚠️ Important Notes

- TFTP has no authentication
- File permissions are often misconfigured
- UDP scanning can be unreliable
- Some servers allow uploads by default

---

# 🧪 Useful Testing Commands

## Verify UDP Port

```bash
ss -uln | grep 69
```

---

## Alternative UDP Scan

```bash
nmap -sU -sV -p 69 <target-ip>
```

---

## Packet Capture

```bash
tcpdump -i eth0 udp port 69
```

---

# 🛡 Hardening Recommendations

- Disable TFTP if unnecessary
- Restrict access via firewall
- Use read-only directories
- Disable anonymous uploads
- Isolate TFTP services
- Prefer SCP/SFTP where possible

---

# ⚖️ Legal Disclaimer

TFTP enumeration and testing should only be performed:

- On systems you own
- With explicit authorization

Unauthorized access or file transfer attempts may violate laws and regulations.
