# 🌐 RPCbind Enumeration

## 🔹 What is RPCbind?

`rpcbind` is a service that maps Remote Procedure Call (RPC) services to the ports they are listening on.

It is commonly found on systems running:

- NFS (Network File System)
- RPC-based Unix services
- Distributed network services

---

# 📌 Default Port

RPCbind listens on:

```text
TCP/111
UDP/111
```

---

# 🔄 How rpcbind Works

## 🟢 1️⃣ Service Registration

When an RPC service starts:

- It registers itself with `rpcbind`
- Provides:
  - Program number
  - Protocol
  - Port number

---

## 🟢 2️⃣ Client Requests

Clients contact `rpcbind` to determine:

```text
Which port is a specific RPC service using?
```

---

## 🟢 3️⃣ Service Resolution

After retrieving the port information:

- Client communicates directly with the RPC service

---

# 🎯 Why RPCbind Enumeration Matters

RPCbind enumeration helps identify:

- Registered RPC services
- NFS exports
- Service ports
- Internal infrastructure information

This is especially useful during:

- NFS enumeration
- Unix/Linux penetration testing
- Network reconnaissance

---

# 🔍 Discover Registered Services & Ports

# ✅ 1️⃣ List RPC Services with rpcinfo

Displays all RPC services registered with `rpcbind`.

```bash
rpcinfo -p <target_IP>
```

---

## Example

```bash
rpcinfo -p 192.168.2.200
```

---

# 📌 Example Output

```text
program vers proto   port
100000    4   tcp    111  portmapper
100003    3   tcp   2049  nfs
100005    1   udp   20048 mountd
```

---

# ✅ 2️⃣ Use Nmap to Enumerate rpcbind

```bash
nmap -v -sT -sV --script=rpcinfo -p 111 <target_IP>
```

---

## Example

```bash
nmap -v -sT -sV --script=rpcinfo -p 111 192.168.2.200
```

---

# 📝 What This Does

- Detects rpcbind
- Enumerates RPC programs
- Displays:
  - Program numbers
  - Protocols
  - Ports

# ✅ 4️⃣ Attempt NFS Enumeration

If NFS is enabled:

```bash
showmount -e <target_IP>
```

---

## Example

```bash
showmount -e 192.168.2.200
```

---

# 📌 Example Output

```text
Export list for 192.168.2.200:
/home *
/shared 192.168.2.0/24
```

---

# 🧠 Why This Is Important

Exposed NFS shares may contain:

- Sensitive files
- Backups
- Credentials
- Internal data

---

# ✅ 5️⃣ Mount NFS Share (If Accessible)

```bash
mount -t nfs 192.168.1.214:/ /mnt/d1/ -nolock
```

---

# 📝 Explanation

| Option | Meaning |
|---|---|
| `-t nfs` | Mount as NFS filesystem |
| `/mnt/d1/` | Local mount directory |
| `-nolock` | Disable file locking |

---

# 📁 Example Workflow

```bash
# Enumerate RPC services
rpcinfo -p 192.168.2.200

# Scan rpcbind with Nmap
nmap -sV --script=rpcinfo -p 111 192.168.2.200

# Enumerate NFS exports
showmount -e 192.168.2.200

# Mount NFS share
mount -t nfs 192.168.2.200:/shared /mnt/nfs -nolock

# Browse files
ls -lah /mnt/nfs
```

---

# 🔍 Common RPC/NFS Ports

| Port | Service |
|---|---|
| 111 | rpcbind / portmapper |
| 2049 | NFS |
| Dynamic | mountd / other RPC services |

---

# ⚠️ Common Security Risks

- Exposed NFS shares
- Weak export permissions
- Anonymous access
- Information disclosure
- Sensitive file exposure

---

# 🛡 Hardening Recommendations

## Restrict Port 111

Allow only trusted systems.

---

## Secure NFS Exports

Use strict rules in:

```text
/etc/exports
```

---

## Disable Unused RPC Services

Reduce attack surface.

---

## Use Firewalls

Restrict:

```text
111/TCP
111/UDP
2049/TCP
2049/UDP
```
