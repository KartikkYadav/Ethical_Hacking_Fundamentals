# 🔹 Microsoft’s Remote Procedure Call (RPC)

Microsoft’s Remote Procedure Call (RPC) is Microsoft’s implementation of the RPC protocol for Windows operating systems.

It enables:

- Interprocess Communication (IPC)
- Remote service interaction
- Remote administration
- Distributed application communication

Programs can execute code or request services on remote systems as if they were local.

---

# 🔹 Key Features of Microsoft RPC

# ✅ Interprocess Communication (IPC)

Allows communication between:

- Applications on the same machine
- Applications across networks

---

# ✅ Client-Server Architecture

## 🟣 Client

Requests execution of a remote procedure.

---

## 🟣 Server

Hosts the service and returns results.

---

# ✅ Transport Protocols

Microsoft RPC commonly uses:

- TCP

depending on the service.

# 🔹 How Microsoft RPC Works

# ✅ 1️⃣ Endpoint Mapper (Port 135)

The RPC Endpoint Mapper acts similarly to:

```text
rpcbind
```

on Linux/Unix systems.

---

## 🔍 Role

When a client wants to communicate with an RPC service:

1. Client connects to port `135`
2. Queries the Endpoint Mapper
3. Receives the correct dynamic port
4. Connects directly to the RPC service

---

# ✅ 2️⃣ Dynamic Ports

After initial communication on:

```text
Port 135
```

actual RPC traffic moves to dynamically assigned high ports.

---

# 🔄 RPC Communication Flow

```text
Client Request
      ↓
RPC Endpoint Mapper (Port 135)
      ↓
Returns Dynamic Port Number
      ↓
Client Connects to RPC Service
      ↓
Remote Procedure Execution
```

```bash

```

---

# ✅ Enumerate RPC Services using rpcclient

```bash
rpcclient -U "" <target_IP>
```

# 🔹 Security Risks of RPC

RPC exposure may lead to:

- Unauthorized remote access
- Service enumeration
- Lateral movement
- Privilege escalation
- Remote Code Execution (RCE)
- DCOM exploitation

---

# ⚠️ Common Attack Surfaces

| Service | Risk |
|---|---|
| WMI | Remote command execution |
| DCOM | Lateral movement |
| Print Spooler | PrintNightmare-type attacks |
| SMB/RPC | Enumeration & relay attacks |

---

# 🔹 Mitigation / Security Recommendations

# ✅ Restrict RPC Access Using Firewall

```powershell
New-NetFirewallRule -DisplayName "Block RPC" -Direction Inbound -Protocol TCP -LocalPort 135 -Action Block
```

---

# ✅ Restrict Dynamic RPC Ports

Limit the RPC dynamic port range.

---

# ✅ Use Strong Authentication

Prefer:

```text
Kerberos
```

over:

```text
NTLM
```

where possible.

---

# ✅ Disable Unnecessary Services

Reduce attack surface by disabling unused RPC-dependent services.

---

# ✅ Enable SMB Signing

Helps mitigate relay attacks.
