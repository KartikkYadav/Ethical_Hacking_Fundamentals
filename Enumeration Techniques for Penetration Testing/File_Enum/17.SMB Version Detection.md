# SMB Version Detection

Detecting the SMB version running on a target system is essential for identifying potential vulnerabilities (e.g., EternalBlue in SMBv1) and selecting appropriate tools for enumeration or exploitation.

---

# Port Checking (Basic Connectivity)

Used to verify if SMB/NetBIOS services are accessible on the target.

```bash
nc -v 192.168.1.174 139
```

```bash
nc -vvv 192.168.1.174 139
```

```bash
telnet 192.168.1.174 139
```

---

# Nmap Service and Version Detection

```bash
nmap -v -sV -sT -sC -A -p 445,139 192.168.1.174
```

## Options

- `-sV` : Service and version detection
- `-sC` : Default NSE scripts
- `-A` : Aggressive scan (OS detection, versioning, scripts)

---

# Targeted SMB NSE Scripts

```bash
nmap -p 445 --script smb-protocols,smb-os-discovery 192.168.1.174
```

---

# SMBClient for Version Testing

These commands help determine supported SMB protocol versions, including legacy SMBv1 (NT1).

```bash
smbclient -L //192.168.1.174
```

```bash
smbclient -L //192.168.1.174 --option='client min protocol=NT1'
```

```bash
smbclient -L //192.168.1.174 --option='client min protocol=NT1' -N
```

```bash
smbclient -L //192.168.1.174 --option='client max protocol=NT1' -N
```

---

# Tcpdump SMB Version Fingerprinting

Captures SMB traffic to extract version strings such as `Unix Samba 4.x.x`.

```bash
tcpdump -s0 -n -i eth0 src 192.168.1.174 and port 139 -A -c 10 2>/dev/null | grep -i "samba\|s.a.m"
```

```bash
tcpdump -s0 -n -i eth0 src 192.168.1.174 and port 445 -A -c 10 2>/dev/null | grep -i "samba\|s.a.m"
```

# ✅ smbver.sh Script

A helper script that combines `tcpdump` and `smbclient` to fingerprint SMB versions.

## Create Script

```bash
vim smbver.sh
```

## Script Content

```bash
#!/bin/sh
# Author: rewardone
# Purpose: Detect SMB version using tcpdump and smbclient

if [ -z $1 ]; then
echo "Usage: ./smbver.sh RHOST {RPORT}" && exit
else
    rhost=$1
fi

if [ ! -z $2 ]; then
    rport=$2
else
    rport=139
fi

tcpdump -s0 -n -i eth0 src $rhost and port $rport -A -c 10 2>/dev/null \
    | grep -i "samba\|s.a.m" \
    | tr -d '.' \
    | grep -oP 'UnixSamba.*[0-9a-z]' \
    | tr -d '\n' &

echo -n "$rhost: "
echo "exit" | smbclient -L $rhost 1>/dev/null 2>/dev/null
echo "" & sleep .1
```

## Make Script Executable

```bash
chmod +x smbver.sh
```

## Run Script

```bash
./smbver.sh <target-ip>
```

---

# Common Findings

- SMBv1 enabled (legacy and vulnerable)
- SMB signing disabled (susceptible to relay attacks)
- Outdated Samba versions
- Misconfigured SMB services

---

# Tips

- Run packet capture while triggering SMB connections to improve fingerprinting accuracy.
- SMBv1 (NT1) is deprecated and highly insecure.
- Combine version detection with enumeration tools such as:
  - `enum4linux-ng`
  - `smbmap`
  - `netexec (nxc)`
  - Impacket tools

---

# Summary

SMB version detection is a foundational step in SMB enumeration.

Identifying the protocol version and implementation helps uncover vulnerabilities, guides exploitation strategies, and improves the effectiveness of further enumeration activities.
