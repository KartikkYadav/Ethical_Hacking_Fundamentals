
# Null Session ( Anonymous SMB Access )

A null session is an unauthenticated connection to a Windows system over SMB (Server Message Block).

It allows interaction with certain services without providing credentials.

If misconfigured, null sessions can expose sensitive information such as:

- Shared resources
- User accounts
- Group memberships
- System details

---

# 🔹 Common Windows Hidden Administrative Shares

| Share Path | Purpose |
| --- | --- |
| `\\192.168.1.52\C$` | Hidden administrative share providing access to the `C:\` drive |
| `\\192.168.1.52\ADMIN$` | Used for remote administration (maps to `C:\Windows`) |
| `\\192.168.1.52\IPC$` | Inter-Process Communication share used for named pipes and authentication |

---

# 🔹 Access Requirements

- Administrative privileges are required for full access to:
  - `C$`
  - `ADMIN$`

- Shares ending with `$` are hidden and not visible in standard network browsing.

- `IPC$` may allow limited interaction even without authentication if null sessions are enabled.

# 🔹 Key Notes

- Null sessions are disabled by default in modern Windows environments (`Windows Server 2016+`).

- They may still be enabled in:
  - Legacy systems
  - Misconfigured environments
  - Systems with weakened Group Policy settings

- `IPC$` is commonly the entry point for SMB-based enumeration.

- Even limited access can aid in:
  - Username enumeration
  - Share discovery
  - Attack surface mapping

---

# 🔹 Security Implications

Allowing null sessions can lead to:

- Information disclosure
- Easier lateral movement
- Increased attack surface for SMB-based exploits

---

# 🔹 Mitigation

## ✅ Disable Anonymous Access via Group Policy

```text
Network access: Do not allow anonymous enumeration of SAM accounts
```

---

## ✅ Restrict Null Session Access

```text
RestrictAnonymous = 1 or 2
```

---

## ✅ Disable SMBv1 Where Possible

---

## ✅ Monitor SMB Connections and Logs

---

# 🔹 Summary

Null sessions provide unauthenticated access to SMB services and can expose valuable information about a target system.

Although modern Windows systems disable null sessions by default, legacy or misconfigured environments may still allow anonymous enumeration through shares such as `IPC$`.

Proper SMB hardening, anonymous access restrictions, and continuous monitoring are essential to reduce the risk of SMB-based reconnaissance and attacks.
