
# Mount SMB Share

This guide covers how to mount and manage SMB (CIFS) shares on Linux systems using the `mount` command.

---

# 🔹 Basic Mount Commands

# ✅ Mount the VulnShare Share to `/mnt`

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt
```

---

# ✅ Verify Mounted File Systems

```bash
df -h
```

---

# ✅ Unmount a Share

```bash
umount /mnt
```

---

# ✅ Verify Unmount

```bash
df -h
```

---

# 🔹 Mount with Credentials

# ✅ Guest Access

Mount the `VulnShare` share using guest access:

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o username=guest,password=
```

---

# ✅ Authenticated Access (Administrative Share)

Mount the `C$` share with credentials:

```bash
mount -t cifs //192.168.2.207/C$ /mnt -o username=administrator,password=Anish@123
```

---

# 🔹 Additional Mount Options

# ✅ Disable File Locking

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt/d1 -o nolock
```

---

# ✅ Mount as Read-Write

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o rw
```

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o username=guest,password=,rw
```

---

# ✅ Mount as Read-Only

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o username=guest,password=,ro
```

---

# ✅ Alternative Read-Write Syntax

> Note: May not work on all Linux distributions.

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt --rw
```

---

# 🔹 SMB Mount Troubleshooting

# ✅ Error : Mounted as Read-Only

If you encounter:

```bash
cannot mount //192.168.2.207/VulnShare read-only
```

---

# ✅ Check Kernel Logs

```bash
dmesg | tail -20
```

---

# ✅ Specify SMB Version

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o vers=3.0
```

---

# ✅ Try Alternative SMB Versions

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o vers=2.1
```

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o vers=1.0
```

---

# ✅ Provide Credentials

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o username=guest,password=
```

---

# ✅ Force Read-Write Mode

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o rw
```

---

# ✅ Disable Unix Extensions / Fix Permissions

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o rw,uid=$(id -u),gid=$(id -g),nounix
```

---

# ✅ Set File and Directory Permissions

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o rw,file_mode=0777,dir_mode=0777
```

---

# ✅ Disable Locking (Lab Environments)

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o nolock
```

---

# ✅ Quick Working Combination

```bash
mount -t cifs //192.168.2.207/VulnShare /mnt -o username=guest,password=,vers=2.1,rw,nounix,file_mode=0777,dir_mode=0777
```

---

# 🔹 Possible Causes

- Share is configured as read-only on the server
- User has only read permissions
- SMB version mismatch
- Authentication failure

---

# 🔹 Setup Requirements

# ✅ Create Mount Point

```bash
mkdir -p /mnt/data
```

---

# ✅ Install CIFS Utilities

```bash
apt install cifs-utils
```

---

# 🔹 Secure Credential Handling

# ✅ Create Credentials File

```bash
vim /root/.smbcredentials
```

---

# ✅ Add the Following

```text
username=administrator
password=Anish@123
```

---

# ✅ Set Proper Permissions

```bash
chmod 600 /root/.smbcredentials
```

---

# ✅ Mount Using Credentials File

```bash
mount -t cifs //192.168.2.207/C$ /mnt/d1 -o credentials=/root/.smbcredentials
```

---

# 🔹 Auto-Mount on Boot (Optional)

# ✅ Add Entry to `/etc/fstab`

```bash
//192.168.2.207/data /mnt/d1 cifs credentials=/root/.smbcredentials,iocharset=utf8,sec=ntlm 0 0
```

---

# ✅ Apply Mounts

```bash
mount -a
```

---

# 🔹 Notes

- Ensure the target system allows SMB connections.
- Use `sec=ntlm`, `ntlmssp`, or `vers=3.0` depending on server configuration.
- Avoid hardcoding credentials directly in commands for security reasons.
- Administrative shares like `C$` require proper privileges.

---

# 🔹 Common SMB Mount Options

| Option | Description |
| --- | --- |
| `username=` | SMB username |
| `password=` | SMB password |
| `vers=` | SMB protocol version |
| `rw` | Read-write mode |
| `ro` | Read-only mode |
| `nolock` | Disable file locking |
| `nounix` | Disable Unix extensions |
| `uid=` | Set file owner UID |
| `gid=` | Set group owner GID |
| `file_mode=` | Set file permissions |
| `dir_mode=` | Set directory permissions |
| `credentials=` | Use credentials file |

---

# 🔹 Security Recommendations

- Use credentials files instead of inline passwords.
- Restrict permissions on credentials files (`chmod 600`).
- Disable SMBv1 where possible.
- Limit SMB share exposure using firewall rules.
- Monitor SMB access logs regularly.

---

# 🔹 Summary

Mounting SMB shares allows Linux systems to interact with Windows file shares using the CIFS protocol.

Proper configuration of:

- Authentication
- SMB version
- Permissions
- Mount options

ensures reliable and secure SMB share access during:

- Penetration Testing
- System Administration
- File Sharing Operations
- Active Directory Environments
