# 💀 Impacket-smbexec

## 🔹 Impacket-smbexec

`impacket-smbexec` is a post-exploitation tool from the Impacket suite that provides semi-interactive command execution on a remote Windows host by leveraging SMB and Windows services without dropping a standalone executable on disk.

It is commonly used for:

- Fileless lateral movement
- Stealthier command execution
- Post-exploitation in Active Directory environments

Compared to `psexec`, it reduces the need for uploading service binaries directly to the target.

---

# 🟢 Use Cases

- Remote command execution with reduced artifact generation
- Fileless lateral movement
- Situations where PsExec-style service binaries are blocked
- Pass-the-Hash authentication attacks
- Kerberos-authenticated remote execution
- Semi-stealthy post-exploitation

---

# ⚙️ How It Works

1. Authenticates over SMB using:
   - Passwords
   - NTLM hashes
   - Kerberos tickets

2. Writes command payloads as temporary batch files into SMB-accessible shares such as:
   - `ADMIN$`
   - `C$`

3. Executes commands through Windows service mechanisms

4. Retrieves command output from temporary files written back to the SMB share

5. Cleans up temporary artifacts after execution

---

# 🔥 Key Features

- No standalone custom executable uploaded
- Uses native Windows components
- Supports semi-interactive command execution
- Compatible with:
  - NTLM authentication
  - Pass-the-Hash
  - Kerberos authentication

- Works with:
  - Local accounts
  - Domain accounts

---

# 🧾 Authentication Format

```bash
[domain/]username[:password]@<target>
```

---

# 🟣 Pass-the-Hash

```bash
impacket-smbexec -hashes <LM>:<NT> user@target
```

---

# 🟣 Password with Special Characters

```bash
impacket-smbexec domain/user:'P@ssw0rd!'@target
```

---

# ⚠️ Requirements & Limitations

- Requires access to:
  - `ADMIN$`
  - `C$`

- Requires administrative privileges
- Uses Windows service execution internally
- Creates temporary batch/script files
- Temporary files may be forensically recoverable
- Not fully fileless because SMB-written artifacts are still generated

# 🔍 smbexec vs psexec

| Feature | `smbexec` | `psexec` |
| --- | --- | --- |
| Uploads EXE Service Binary | No | Yes |
| Uses SMB Shares | Yes | Yes |
| Creates Windows Service | Yes | Yes |
| Interactive Shell | Semi | Semi |
| Noise Level | Medium | High |
| Disk Artifacts | Temporary batch files | Uploaded executable |
| AV/EDR Detection | Lower | Higher |

---

# 🔐 Authentication Methods

| Method | Example |
| --- | --- |
| Password | `user:password@target` |
| NTLM Hash | `-hashes LMHASH:NTHASH` |
| Kerberos | `-k -no-pass` |
| AES Key | `-aesKey <AES_KEY>` |

---

# 🚩 Common Flags

| Flag | Description |
| --- | --- |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-dc-ip` | Specify Domain Controller IP |
| `-target-ip` | Specify target IP |
| `-debug` | Enable debug output |

# 🔹 Summary

`impacket-smbexec` is a powerful Impacket tool for semi-interactive remote command execution over SMB.

It is commonly used during:

- Penetration Testing
- Red Team Operations
- Active Directory Assessments
