# 💀 Impacket-getArch

## 🔹 Impacket-getArch

`getArch` is a Windows post-exploitation utility included in the Impacket toolkit that helps determine the architecture (`x86` or `x64`) of a remote Windows system.

It is commonly used during:

- Reconnaissance
- Post-exploitation
- Payload preparation
- Exploit selection

---

# 🟢 Use Cases

- Identify whether a remote system is:
  - 32-bit (`x86`)
  - 64-bit (`x64`)

- Ensure compatibility of:
  - Payloads
  - DLLs
  - Shellcode
  - Exploits

- Validate architecture before:
  - Privilege escalation
  - Process injection
  - Memory migration
  - Lateral movement

---

# ⚙️ How It Works

1. Connects to the remote target over SMB/RPC
2. Queries architecture-related information
3. Detects whether the target OS is:
   - `x86`
   - `x64`

4. Displays the identified architecture

---

# 🟣 Example Output

## ✅ 64-bit System

```text

```

---

## ✅ 32-bit System


---

# 🔥 Why It Matters

- 64-bit Windows systems can run:
  - 32-bit applications
  - 64-bit applications

- Many payloads require matching architecture:
  - DLL injection
  - Meterpreter payloads
  - Shellcode
  - Kernel exploits

- Incorrect architecture selection may cause:
  - Crashes
  - Failed execution
  - Injection failures

# 🔹 Common Flags

| Flag | Description |
| --- | --- |
| `-target` | Specify target IP or hostname |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-dc-ip` | Specify Domain Controller IP |
| `-debug` | Enable debug output |

# ⚠️ Limitations

- Requires reachable SMB/RPC services
- Firewall restrictions may block detection
- Some hardened systems may restrict remote queries
- Does not provide:
  - Command execution
  - Shell access
  - Privilege escalation

---

# 🛡️ Defensive Recommendations

- Restrict SMB/RPC exposure
- Limit administrative access
- Monitor remote RPC enumeration
- Use host-based firewalls
- Enable:
  - Sysmon
  - Windows Event Logging
  - EDR monitoring
