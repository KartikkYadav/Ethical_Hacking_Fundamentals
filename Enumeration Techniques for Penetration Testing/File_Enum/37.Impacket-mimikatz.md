# 💀 Impacket-mimikatz

## 🔹 Impacket-mimikatz

`impacket-mimikatz` is a wrapper-style tool based on the Impacket framework that enables remote execution of Mimikatz on Windows systems over SMB.

It automates credential dumping operations by:

- Uploading Mimikatz remotely
- Executing credential dumping commands
- Retrieving output from the target system

It is commonly used during post-exploitation to extract:

- Plaintext credentials
- NTLM password hashes
- Kerberos tickets
- Kerberos keys
- DPAPI secrets

---

# 🟢 Use Cases

- Extract credentials from LSASS memory
- Dump NTLM hashes and Kerberos tickets
- Harvest credentials after initial compromise
- Support:
  - Lateral movement
  - Privilege escalation
  - Active Directory compromise

- Automate remote Mimikatz execution without RDP access

---

# ⚙️ How It Works

1. Authenticates to the target over SMB
2. Uploads a Mimikatz executable or payload
3. Executes Mimikatz remotely
4. Runs selected modules such as:

```text
sekurlsa::logonpasswords
lsadump::sam
kerberos::list
```

5. Retrieves command output
6. Cleans up uploaded artifacts

# 🟣 Common Mimikatz Modules

| Module | Purpose |
| --- | --- |
| `sekurlsa::logonpasswords` | Dump logged-in credentials |
| `lsadump::sam` | Dump local SAM hashes |
| `kerberos::list` | List Kerberos tickets |
| `crypto::certificates` | Extract certificates |
| `vault::list` | Dump Windows Vault credentials |
| `sekurlsa::tickets` | Export Kerberos tickets |
| `sekurlsa::ekeys` | Dump Kerberos encryption keys |
| `token::elevate` | Token impersonation |

# 🚨 Detection Considerations

- Highly monitored by:
  - AV
  - EDR
  - SIEM platforms

- Common detections include:

```text
• LSASS memory access
• Process injection
• Credential dumping behavior
• Known Mimikatz signatures
• Suspicious PowerShell activity
```

---

# 🛡️ Defensive Recommendations

- Enable:
  - Credential Guard
  - LSASS protection (`RunAsPPL`)
  - Attack Surface Reduction (ASR) rules

- Monitor:
  - LSASS access
  - SMB execution
  - Remote service creation
  - WMI/DCOM execution

- Restrict:
  - Local administrator privileges
  - SMB administrative shares
  - Unnecessary RPC exposure

- Use:
  - MFA
  - LAPS
  - EDR solutions

# 🔍 Mimikatz vs Impacket Execution Methods

| Method | Uploads Binary | Fileless | Noise Level |
| --- | --- | --- | --- |
| `psexec` | Yes | No | High |
| `wmiexec` | No | Yes | Medium |
| `smbexec` | No | Partial | Medium |
| `atexec` | No | Partial | Medium |

# 🔹 Summary

`impacket-mimikatz` is a post-exploitation utility used to remotely execute Mimikatz through SMB-based Impacket tooling.

It enables attackers and security professionals to:

- Dump credentials
- Extract NTLM hashes
- Retrieve Kerberos tickets
- Harvest secrets from compromised Windows systems

It is heavily used during:

- Red Team Operations
- Active Directory Assessments
- Internal Penetration Tests
- Post-Exploitation Activities

Because credential dumping is highly monitored in modern environments, defenders frequently deploy EDR protections specifically targeting Mimikatz behavior.
