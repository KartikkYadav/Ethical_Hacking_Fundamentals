# 💀 impacket-reg

## 🔹 impacket-reg

`impacket-reg` is a tool from the Impacket suite that enables remote Windows Registry interaction over MS-RPC using the MS-RRP (Remote Registry Protocol).

It allows operators to remotely:

- Query registry keys and values
- Create registry keys
- Modify registry entries
- Delete keys and values
- Save registry hives

All without requiring RDP or interactive shell access.

---

# 🟢 Use Cases

- Query remote registry keys and values
- Modify Windows configuration settings remotely
- Enable or disable:
  - Windows Defender
  - UAC
  - RDP

- Create persistence using:
  - Registry Run keys
  - Startup entries

- Enumerate:
  - Installed software
  - Startup applications
  - Security policies

- Extract:
  - AutoLogon credentials
  - Configuration data

- Save registry hives for offline analysis

---

# ⚙️ How It Works

1. Authenticates to the target over SMB/RPC
2. Connects to the Remote Registry service using MS-RRP
3. Performs remote registry operations
4. Returns registry data or modifies registry values

---

# 🔥 Key Features

- Remote registry enumeration and modification
- Registry key/value management
- Works without interactive shell access
- Supports registry hive export
- Useful for:
  - Post-exploitation
  - Persistence
  - Configuration management

---

# 🟣 Add a Registry Value

```bash
impacket-reg user:pass@target add \
-keyName HKLM\\Software\\Test \
-valueName TestValue \
-valueData 1 \
-valueType REG_DWORD
```

---

# 🟣 Delete a Registry Value

```bash
impacket-reg user:pass@target delete \
-keyName HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run \
-valueName EvilApp
```

---

# 🟣 Create a New Registry Key

```bash
impacket-reg user:pass@target create \
-keyName HKLM\\Software\\NewKey
```

---

# 🟣 Save Registry Hive

```bash
impacket-reg administrator:Password123@192.168.29.204 save \
-keyName HKLM\\SAM \
-outputPath C:\\Windows\\Temp\\sam.save
```

---

# 🟢 Common Registry Locations

| Registry Path | Purpose |
| --- | --- |
| `HKLM\\SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Run` | Startup persistence |
| `HKLM\\SYSTEM\\CurrentControlSet\\Services` | Service configuration |
| `HKLM\\SOFTWARE\\Policies` | Security and policy settings |
| `HKLM\\SAM` | Local account database |
| `HKLM\\SECURITY` | LSA secrets and security settings |
| `HKLM\\SYSTEM` | System configuration |
| `HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run` | User startup programs |

# 📌 Requirements & Notes

- Remote Registry service must be enabled
- Requires Administrator privileges for most operations
- Registry paths require escaped backslashes (`\\`)
- Uses:
  - SMB (`TCP/445`)
  - RPC interfaces

- Activity may trigger:
  - Security monitoring
  - Event logs
  - EDR alerts

---

# 🚨 Detection Considerations

- Registry modifications are heavily monitored
- Persistence keys may trigger:
  - AV alerts
  - EDR detections

- Remote Registry service usage can be logged
- Security tools often detect:
  - Defender tampering
  - Run key persistence
  - Suspicious registry writes

# 🚩 Common Flags

| Flag | Description |
| --- | --- |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-dc-ip` | Specify Domain Controller IP |
| `-target-ip` | Specify target IP |
| `-debug` | Enable debug output |

# ⚠️ Limitations

- Requires:
  - SMB connectivity
  - RPC access
  - Remote Registry service

- Hardened environments may:
  - Disable Remote Registry
  - Restrict RPC access
  - Block registry modifications

- Modern EDR products monitor:
  - Registry persistence
  - Defender tampering
  - Remote registry access

---

# 🛡️ Defensive Recommendations

- Disable Remote Registry service if unnecessary
- Restrict SMB/RPC exposure
- Monitor:
  - Registry modifications
  - Run key persistence
  - Security policy changes

- Enable:
  - Sysmon
  - Windows Event Logging
  - EDR monitoring

- Restrict:
  - Local administrator privileges
  - Remote registry access

# 🔹 Summary

`impacket-reg` is a powerful remote registry management utility used for Windows administration and post-exploitation over SMB/RPC.

It enables operators to:

- Query registry keys
- Modify system settings
- Create persistence mechanisms
- Extract configuration data
- Save registry hives

It is commonly used during:

- Active Directory Assessments
- Penetration Testing
- Red Team Operations
- Post-Exploitation Activities

Because registry modifications are heavily monitored in modern environments, defenders frequently use EDR and SIEM solutions to detect suspicious registry activity.
