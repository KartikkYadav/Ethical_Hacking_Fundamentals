# 💀 impacket-services

## 🔹 impacket-services

`impacket-services` is a tool in the Impacket suite used to remotely manage Windows services over SMB and DCE/RPC.

It allows administrators or operators to:

- Enumerate services
- Start or stop services
- Modify service configurations
- Create or delete services remotely

The tool communicates with the Windows Service Control Manager (SCM) using the `svcctl` RPC interface.

---

# 🟢 Use Cases

- Enumerate services on remote Windows systems
- Start or stop remote services
- Query service status and configuration
- Create services for remote command execution
- Modify services for persistence or payload execution
- Perform administrative or post-exploitation tasks
- Abuse weak service permissions

---

# ⚙️ How It Works

1. Authenticates to the remote host over SMB
2. Connects to the Service Control Manager (SCM) using DCE/RPC
3. Sends service management requests
4. Retrieves service configuration and status information

---

# 🔥 Key Features

- Remote Windows service management
- Supports:
  - Local accounts
  - Domain accounts

- Service enumeration and configuration
- Service creation and modification
- Can be used for:
  - Remote execution
  - Persistence
  - Lateral movement

# 🧾 Authentication Format

```bash
[domain/]username[:password]@<target>
```

# 🟣 Passwords with Special Characters


# 🟢 Supported Actions

| Action | Description |
| --- | --- |
| `list` | List all services |
| `start` | Start a service |
| `stop` | Stop a service |
| `status` | Show service status |
| `config` | Display service configuration |
| `create` | Create a new service |
| `change` | Modify service configuration |
| `delete` | Delete a service |

| Technique | Description |
| --- | --- |
| Unquoted Service Path | Privilege escalation through path hijacking |

# 🚨 Detection Considerations

- Service creation and modification are heavily logged
- Frequently monitored by:
  - AV
  - EDR
  - SIEM solutions

- Generates:
  - Windows Event Logs
  - SCM activity
  - Service start/stop telemetry

- Uses:
  - SMB (`TCP/445`)
  - RPC service control interfaces

---

# 📌 Requirements & Notes

- Requires Administrator privileges
- Requires SMB and RPC access
- Targets the `svcctl` named pipe
- Commonly used during:
  - Lateral movement
  - Remote administration
  - Post-exploitation

---

# 🔐 Authentication Methods

| Method | Example |
| --- | --- |
| Password | `user:password@target` |
| NTLM Hash | `-hashes LMHASH:NTHASH` |
| Kerberos | `-k -no-pass` |
| AES Key | `-aesKey <AES_KEY>` |

---

# 🚩 Common Flags

| Flag | Description |
| --- | --- |
| `-hashes` | Use NTLM hashes |
| `-k` | Use Kerberos authentication |
| `-no-pass` | Do not prompt for password |
| `-dc-ip` | Specify Domain Controller IP |
| `-target-ip` | Specify target IP |
| `-debug` | Enable debug output |

---

# ⚠️ Limitations

- Requires elevated privileges
- Firewall restrictions may block RPC communication
- Modern EDR products often monitor service manipulation
- Excessive service creation may trigger alerts

---

# 🛡️ Defensive Recommendations

- Restrict service management permissions
- Limit SMB/RPC exposure
- Monitor:
  - Service creation
  - Service modification
  - Unexpected service binaries

- Enable:
  - Sysmon
  - Windows Event Logging
  - EDR monitoring

- Restrict:
  - Local administrator access
  - Remote service creation
  - Unnecessary administrative shares
