
# 🗂️ NFS Enumeration

## 🔹 Overview

NFS (Network File System) is a protocol used to share directories and files across a network.  
It is commonly deployed in Unix and Linux environments.

Misconfigured NFS shares may expose sensitive files, allow unauthorized access, or enable privilege escalation.

---

# 🔹 Common NFS Ports

Always enumerate both TCP and UDP services during NFS assessments.

| Protocol | Port | Service |
| --- | --- | --- |
| TCP | `111` | RPCBind / Portmapper |
| TCP | `2049` | NFS Service |
| TCP | `1110` | Nlockmgr |
| TCP | `4045` | Status Monitor (`statd`) |
| TCP | `20048` | `mountd` |
| UDP | `111` | RPCBind / Portmapper |
| UDP | `2049` | NFS Service |
| UDP | `1110` | Nlockmgr |
| UDP | `4045` | Status Monitor |
| UDP | `20048` | `mountd` |

> Always scan both TCP and UDP ports during NFS enumeration.


---

# 🏳️ Export Flags

| Flag | Description |
| --- | --- |
| `rw` | Read-write access |
| `ro` | Read-only access |
| `sync` | Immediate write synchronization |
| `async` | Faster but less secure writes |
| `no_root_squash` | Remote root remains root |
| `root_squash` | Maps remote root to nobody |
| `no_all_squash` | Preserve UID/GID mappings |
| `all_squash` | Map all users to anonymous |

# 🚨 Common Misconfigurations

| Misconfiguration | Risk |
| --- | --- |
| `no_root_squash` enabled | Root privilege escalation |
| World-readable exports | Information disclosure |
| Writable shares | Malicious file upload |
| Weak subnet restrictions | Unauthorized access |
| Sensitive home directories exported | Credential exposure |

---

# 🔥 no_root_squash Privilege Escalation

If `no_root_squash` is enabled:

- Remote root user remains root on the NFS server
- Attackers can create SUID binaries remotely
- Can lead to full root compromise

---

# 📌 Notes

- NFSv3 commonly relies on RPC services
- NFSv4 typically uses only TCP `2049`
- UDP enumeration is important on legacy systems
- Anonymous access may expose sensitive data
- Always verify export permissions carefully

---

# 🚨 Detection Considerations

- Excessive RPC or mount requests may trigger monitoring
- NFS mount operations can appear in:
  - Syslog
  - Audit logs
  - Network telemetry

- `no_root_squash` configurations are considered high risk

---

# 🛡️ Defensive Recommendations

- Disable `no_root_squash`
- Restrict exports to trusted hosts/subnets
- Use:
  - `root_squash`
  - `ro`
  - Strong firewall rules

- Avoid exporting:
  - Home directories
  - Sensitive configuration paths

- Monitor:
  - NFS mounts
  - Export changes
  - RPC activity

---

# 🔹 Summary

NFS enumeration is an important part of Linux and Unix network assessments.

Misconfigured NFS shares can lead to:

- Information disclosure
- Unauthorized file access
- Privilege escalation
- Remote root compromise

Careful enumeration of:

- Exports
- Permissions
- Export flags
- RPC services

can reveal significant attack surface during penetration testing and red team operations.
