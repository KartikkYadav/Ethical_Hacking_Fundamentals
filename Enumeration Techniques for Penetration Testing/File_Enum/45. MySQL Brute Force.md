# 📘 MySQL Authentication & Password Auditing

> **Note:** The techniques in this guide should only be used in environments where you have explicit authorization to perform security testing. Unauthorized password guessing or brute-force attacks against systems you do not own or administer is illegal and unethical.

This guide covers legitimate authentication testing, manual login verification, password auditing, credential discovery during authorized assessments, and MySQL authentication troubleshooting.

---

# 🟢 1. Manual Login

## 🟩 Login Without Password

```bash
mysql -h 192.168.10.155 --ssl=FALSE -u root
```

### Modern MySQL Clients

```bash
mysql -h 192.168.10.155 --ssl=0 -u root
```

---

## 🟩 Login With Password Prompt

```bash
mysql -h 192.168.10.155 --ssl=FALSE -u root -p
```

---

## 🟩 Alternative Format

```bash
mysql --host=192.168.10.155 --ssl-mode=DISABLED -u root -p
```

---

## 🟩 Specify Port

```bash
mysql -h 192.168.10.155 \
--ssl-mode=DISABLED \
-P 3306 \
-u root \
-p
```

---

## 🟩 Local Socket Login

```bash
mysql -u root@localhost
```

# 🟢 4. MySQL Connection Limits

Example configuration:

```ini
[mysqld]

pid-file=/var/run/mysqld/mysqld.pid
socket=/var/run/mysqld/mysqld.sock
datadir=/var/lib/mysql
log-error=/var/log/mysql/error.log
secure-file-priv=""

max_connections=1000
max_connect_errors=100000
```

# 🟢 13. Authentication Verification Workflow

## 🟩 Detect MySQL Service

```bash
nmap -sV -p3306 192.168.10.155
```

---

## 🟩 Check for Empty Passwords

```bash
nmap -p3306 \
--script=mysql-empty-password \
192.168.10.155
```

---

## 🟩 Enumerate Users (With Valid Credentials)

```bash
nmap -p3306 \
--script=mysql-users \
--script-args="mysqluser=root,mysqlpass=<password>" \
192.168.10.155
```

---

# 🟢 15. Important Notes

- MySQL commonly listens on **TCP 3306**.
- Some MySQL servers require **TLS/SSL** for client connections.
- Modern MySQL versions typically use the `caching_sha2_password` authentication plugin.
- Older deployments may still use `mysql_native_password`.
- Repeated failed authentication attempts can increase the `Aborted_connects` counter or trigger account lockouts and monitoring alerts.
- Database credentials are frequently stored in application configuration files, making secure file permissions important.
- Always perform authentication testing only with explicit authorization.
