# 📘 Microsoft SQL Server (MSSQL) Enumeration

## 📖 Overview

Microsoft SQL Server (MSSQL) enumeration is the process of identifying SQL Server instances, configurations, versions, exposed services, authentication methods, and accessible resources on a target system. Proper enumeration helps security professionals assess the attack surface and identify potential misconfigurations.

---

# 🟢 1. Default SQL Server Ports

| Service / Feature | Default Port | Purpose |
|-------------------|-------------:|---------|
| SQL Server Database Engine | TCP 1433 | Primary database communication |
| SQL Server Browser | UDP 1434 | Named instance discovery |
| SQL Server Analysis Services (SSAS) | TCP 2383 | Default Analysis Services |
| SSAS Redirector | TCP 2382 | Redirects named SSAS instances |
| SQL Server Reporting Services (SSRS) | TCP 80 / 443 | Reporting interface |
| SQL Server Integration Services (SSIS) | TCP 135 | DCOM/RPC management |
| Always On / Database Mirroring | TCP 5022+ | Replication & High Availability |

---

## 🟩 Port Summary

- **TCP 1433** → SQL Server Database Engine
- **UDP 1434** → SQL Server Browser
- **TCP 2382 / 2383** → Analysis Services
- **TCP 80 / 443** → Reporting Services
- **TCP 135** → SSIS DCOM/RPC
- **TCP 5022+** → Mirroring / Always On

# 🟢 2. Instance Discovery

## 🟩 Full Service Discovery

```bash
nmap -v -sT -sC -sV -A -O -p- <target>
```

---

## 🟩 Scan Default SQL Port

```bash
nmap -v -sT -sC -sV -A -O -p1433 <target>
```

---

## 🟩 SQL Server Information

```bash
nmap -v -sT -sV -p1433 \
--script ms-sql-info \
<target>
```

---

## 🟩 NTLM Information

```bash
nmap -v -sT -sV -p1433 \
--script ms-sql-ntlm-info \
<target>
```

---

## 🟩 Configuration Enumeration

```bash
nmap -v -sT -sV -p1433 \
--script ms-sql-config \
<target>
```

---

## 🟩 Empty Password Check

```bash
nmap -v -sT -sV -p1433 \
--script ms-sql-empty-password \
<target>
```

---

## 🟩 Run All MSSQL NSE Scripts

```bash
nmap -v -sT -sV -p1433 \
--script ms-sql-* \
<target>
```

---

### These Scripts Can Identify

- SQL Server instances
- SQL Server version
- Configuration information
- NTLM details
- Empty passwords
- Accessible databases (when authorized)

# 🟢 3. Authenticated Enumeration

Many MSSQL NSE scripts support authenticated enumeration.

## 🟩 General Syntax

```bash
nmap -v -sT -sV -p1433 \
--script=<script-name> \
--script-args \
'mssql.username=<USER>,mssql.password=<PASSWORD>' \
<target>
```

---

## 🟩 Configuration Enumeration

```bash
nmap -v -sT -sV -p1433 \
--script ms-sql-config \
--script-args \
'mssql.username=sa,mssql.password=Password123!' \
<target>
```

---

## 🟩 Enumerate Tables

```bash
nmap -v -sT -sV -p1433 \
--script ms-sql-tables \
--script-args \
'mssql.username=sa,mssql.password=Password123!' \

```

---

# 🟢 4. SQL Browser Enumeration

The SQL Browser Service listens on **UDP 1434**.

It may reveal:

- Instance names
- Dynamic ports
- SQL Server version
- Connection parameters

---

## 🟩 Query Browser Service

```bash
nmap -sU -p1434 \
--script ms-sql-info \
<target>
```

---

## 🟩 Discover SQL Servers

```bash
nmap --script broadcast-ms-sql-discover
```

# 🟢 5. SQLCMD Enumeration

Microsoft provides **sqlcmd** for SQL Server administration.

---

## 🟩 Discover SQL Servers

```bash
sqlcmd -L
```

---

## 🟩 Connect to SQL Server

```bash
sqlcmd -S <server> -U sa -P '<password>'
```

---

## 🟩 Execute a Query

```bash
sqlcmd -S <server> \
-U sa \
-P '<password>' \
-Q "SELECT @@VERSION"
```

---

## 🟩 List Databases

```sql
SELECT name
FROM sys.databases;
```

---

## 🟩 List Logins

```sql
SELECT name
FROM sys.server_principals;
```

# 🟢 6. PowerShell Enumeration

## 🟩 SQL Services

```powershell
Get-Service |
Where-Object {$_.Name -like "*SQL*"}
```

---

## 🟩 Installed SQL Instances

```powershell
Get-ItemProperty `
"HKLM:\SOFTWARE\Microsoft\Microsoft SQL Server\Instance Names\SQL"
```

---

## 🟩 Discover SQL Servers Using SMO

```powershell
[System.Reflection.Assembly]::LoadWithPartialName(
"Microsoft.SqlServer.Smo"
) | Out-Null

[Microsoft.SqlServer.Management.Smo.SmoApplication]::EnumAvailableSqlServers()
```

# 🟢 10. Common MSSQL NSE Scripts

| NSE Script | Purpose |
|------------|---------|
| `ms-sql-info` | SQL Server version & instance discovery |
| `ms-sql-ntlm-info` | NTLM information |
| `ms-sql-config` | Configuration enumeration |
| `ms-sql-tables` | Enumerate tables |
| `ms-sql-query` | Execute SQL queries |
| `ms-sql-hasdbaccess` | Check database permissions |
| `ms-sql-empty-password` | Detect blank passwords |
| `ms-sql-xp-cmdshell` | Check `xp_cmdshell` status |
| `ms-sql-dump-hashes` | Dump password hashes (requires privileges) |

---

# 🟢 11. Security Considerations

- Exposed SQL services increase the attack surface.
- SQL Browser can leak instance information.
- Weak or default credentials increase risk.
- Exposed SSRS, SSAS, or SSIS services may disclose sensitive information.
- Excessive privileges increase the impact of compromise.
- Unpatched SQL Server versions may contain known vulnerabilities.

# 🟢 13. MSSQL Enumeration Workflow

```text
1. Discover open ports
2. Identify SQL Server instances
3. Query SQL Browser (UDP 1434)
4. Detect SQL Server version
5. Enumerate configuration
6. Perform authenticated enumeration (if authorized)
7. Enumerate databases
8. Review permissions
9. Inspect registry and services
10. Review SSRS / SSIS / SSAS exposure
11. Assess network exposure
```

# 🟢 15. Key Takeaways

- SQL Server commonly listens on **TCP 1433**.
- SQL Browser typically uses **UDP 1434**.
- Nmap provides extensive MSSQL NSE scripts for discovery and enumeration.
- SQLCMD is Microsoft's native command-line administration tool.
- PowerShell and the Windows Registry provide valuable local enumeration capabilities.
- SSRS, SSIS, and SSAS should be reviewed during security assessments.
- Minimize exposed services, enforce strong authentication, apply regular patching, and continuously monitor SQL Server deployments.
