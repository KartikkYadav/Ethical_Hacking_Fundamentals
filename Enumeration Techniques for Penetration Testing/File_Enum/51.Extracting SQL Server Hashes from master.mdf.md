# 📘 SQL Server Password Hashes & `master.mdf`

## 📖 Overview

The SQL Server **master** database is the primary system database and contains critical server-level metadata, including SQL login information, authentication configuration, server roles, and instance settings.

During incident response, forensic investigations, backup analysis, or authorized security assessments, investigators may encounter password hash data stored within the `master.mdf` database.

> **Note:** SQL Server does **not** store passwords in plaintext. SQL-authenticated accounts are stored using cryptographic password hashes.

---

# 🟢 1. The `master` Database

The **master** database stores server-wide metadata including:

- SQL Server configuration
- SQL login information
- Server-level security principals
- Server roles and permissions
- Database catalog metadata
- Linked server configuration
- Endpoint configuration

---

## 🟩 Default Location

```text
C:\Program Files\Microsoft SQL Server\<Instance>\MSSQL\DATA\master.mdf
```

---

## 🟩 Example

```text
C:\Program Files\Microsoft SQL Server\
MSSQL16.MSSQLSERVER\
MSSQL\DATA\master.mdf
```

---

# 🟢 2. Authentication Methods

SQL Server supports two primary authentication mechanisms.

---

## 🟩 Windows Authentication

Uses:

- Active Directory
- Local Windows accounts

### Benefits

- Centralized identity management
- Kerberos support
- Password policy enforcement
- Reduced password storage inside SQL Server

---

## 🟩 SQL Server Authentication

Uses SQL-specific accounts.

### Examples

```text
sa
app_user
reporting_user
service_account
```

SQL Server stores password hashes internally for these accounts.

---

# 🟢 3. Security Relevance

Access to the **master** database may reveal information such as:

- SQL login names
- Login status
- Authentication configuration
- Server roles
- Password hash metadata

For this reason, the **master** database should be treated as highly sensitive.

# 🟢 5. Protecting Authentication Data

## 🟩 Restrict File Access

Limit access to:

```text
master.mdf
mastlog.ldf
```

Only SQL Server service accounts and administrators should have access.

---

## 🟩 Encrypt Storage

Use technologies such as:

- BitLocker
- Self-encrypting drives
- Storage-level encryption

---

## 🟩 Secure Backups

Protect backup files using:

- Backup encryption
- Secure storage
- Access controls
- Retention policies

---

## 🟩 Monitor File Access

Monitor access to:

```text
*.mdf
*.ldf
*.bak
```

Unexpected access may indicate:

- Data theft
- Insider activity
- Malware
- Unauthorized backup extraction

# 🟢 6. Audit SQL Logins

## 🟩 Review SQL Logins

```sql
SELECT
    name,
    type_desc,
    is_disabled
FROM sys.server_principals
WHERE type_desc='SQL_LOGIN';
```

---

## 🟩 Review Server Roles

```sql
SELECT
    p.name AS LoginName,
    r.name AS ServerRole
FROM sys.server_role_members rm
JOIN sys.server_principals p
    ON rm.member_principal_id = p.principal_id
JOIN sys.server_principals r
    ON rm.role_principal_id = r.principal_id;
```

---

# 🟢 7. Defensive Recommendations

## 🟩 Enforce Password Policies

Enable:

- Password complexity
- Password expiration
- Account lockout

Example:

```sql
CREATE LOGIN AppUser
WITH PASSWORD='StrongPassword!',
CHECK_POLICY=ON,
CHECK_EXPIRATION=ON;
```

---

## 🟩 Review SQL Logins

```sql
SELECT
    name,
    is_disabled
FROM sys.sql_logins;
```

---

## 🟩 Disable Unused Accounts

```sql
ALTER LOGIN OldAccount DISABLE;
```

---

## 🟩 Review Sysadmin Membership

```sql
SELECT name
FROM sys.server_principals
WHERE IS_SRVROLEMEMBER('sysadmin', name)=1;
```

---

## 🟩 Prefer Windows Authentication

Whenever practical:

- Use Active Directory accounts
- Avoid shared SQL logins
- Implement MFA for administrative access

# 🟢 8. SQL Server System Databases

| Database | Purpose |
|----------|---------|
| `master` | Server configuration and login metadata |
| `model` | Template for new databases |
| `msdb` | SQL Agent jobs, alerts, maintenance plans |
| `tempdb` | Temporary objects and workspace |
| `resource` | Read-only system objects |

---

# 🟢 9. Best Practices

- Restrict access to SQL Server system database files.
- Encrypt storage and backups.
- Audit SQL logins regularly.
- Minimize membership in privileged server roles.
- Prefer Windows Authentication over SQL Authentication whenever possible.
- Monitor access to `.mdf`, `.ldf`, and `.bak` files.
- Keep SQL Server updated with the latest security patches.

---

# 🟢 10. Key Takeaways

- `master.mdf` contains critical SQL Server configuration and security metadata.
- SQL-authenticated accounts are stored using cryptographic password hashes—not plaintext passwords.
- Access to system database files should be tightly controlled.
- Protect backups with encryption and strict access controls.
- Regularly review logins, permissions, and privileged roles.
- Windows Authentication generally provides stronger centralized security than SQL Authentication.
