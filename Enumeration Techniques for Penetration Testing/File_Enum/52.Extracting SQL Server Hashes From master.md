# 📘 SQL Server Authentication Metadata (`master.mdf`)

## 📖 Overview

The SQL Server **`master`** database contains critical server metadata, including SQL login information and password hashes for SQL-authenticated accounts.

> **Note:** SQL Server stores **cryptographic password hashes**, not plaintext passwords. Accessing or analyzing these hashes should only be performed for legitimate administrative, forensic, incident response, backup recovery, or other authorized purposes.

---

# 🟢 1. Requirements

- PowerShell 7
- OrcaMDF Libraries
- PowerShell Post-Exploitation Toolkit
- Hashcat (for authorized password auditing)
- Text Editor (Vim, Nano, etc.)

# 🟢 2. SQL Server Authentication Metadata Analysis Workflow

## 🟩 Step 1 — Obtain Required Tools

Clone the toolkit:

```bash
git clone https://github.com/xpn/PowerShell-PostExploitation.git

cd PowerShell-PostExploitation
```

---

## 🟩 Step 2 — Start PowerShell

```powershell
pwsh
```

Import the module:

```powershell
Import-Module ./Get-MDFHashes.ps1
```

---

Load the OrcaMDF libraries:

```powershell
Add-Type -AssemblyName .\OrcaMDF.RawCore.dll

Add-Type -AssemblyName .\OrcaMDF.Framework.dll
```

---

If required, re-import the module:

```powershell
Import-Module ./Get-MDFHashes.ps1
```

> Repeat the import if module loading fails because of dependency ordering.

---

## 🟩 Step 3 — Extract SQL Login Hashes

Extract SQL login hashes from `master.mdf`:

```powershell
Get-MDFHashes -mdf ./master.mdf
```

---

Save the results:

```powershell
Get-MDFHashes -mdf ./master.mdf |
Out-File -FilePath ./master_hash.txt
```

## 🟩 Step 5 — Password Auditing

Place extracted hashes into:

```text
master_hash.txt
```

Run Hashcat:

```bash
hashcat -m 1731 -a 0 master_hash.txt /usr/share/wordlists/rockyou.txt
```

Display recovered results:

```bash
hashcat -m 1731 -a 0 master_hash.txt /usr/share/wordlists/rockyou.txt --show
```

> Use password auditing only on systems and backups you are authorized to assess.

# 🟢 4. Protecting SQL Authentication Data

## 🟩 Restrict File Access

Protect:

```text
master.mdf
mastlog.ldf
```

Only SQL Server service accounts and administrators should have access.

---

## 🟩 Encrypt Storage

Recommended controls:

- BitLocker
- Self-encrypting drives
- Storage encryption

---

## 🟩 Secure Backups

Protect backup files using:

- Backup encryption
- Secure storage
- Access controls
- Retention policies

Monitor access to:


Unexpected access may indicate:

- Malware activity

# 🟢 5. Audit SQL Logins

## 🟩 Review SQL Logins

```sql
SELECT
    name,
    type_desc,
    is_disabled
FROM sys.server_principals
WHERE type_desc='SQL_LOGIN';
```

---

## 🟩 Review Server Role Membership

```sql
SELECT
    p.name AS LoginName,
    r.name AS ServerRole
FROM sys.server_role_members rm
JOIN sys.server_principals p
    ON rm.member_principal_id = p.principal_id
JOIN sys.server_principals r
    ON rm.role_principal_id = r.principal_id;
```

# 🟢 6. Defensive Recommendations

## 🟩 Enforce Password Policies

Enable:

- Password complexity
- Password expiration
- Account lockout

Example:

```sql
CREATE LOGIN AppUser
WITH PASSWORD='StrongPassword!',
CHECK_POLICY=ON,
CHECK_EXPIRATION=ON;
```

---

## 🟩 Review Disabled Accounts

```sql
SELECT
    name,
    is_disabled
FROM sys.server_principals
WHERE is_disabled=1;
```

---

## 🟩 Review Sysadmin Membership

```sql
SELECT name
FROM sys.server_principals
WHERE IS_SRVROLEMEMBER('sysadmin', name)=1;
```

---

## 🟩 Prefer Windows Authentication

Whenever practical:

- Use Active Directory accounts
- Avoid shared SQL logins
- Enable MFA for administrative access

# 🟢 8. Key Takeaways

- `master.mdf` contains critical SQL Server security metadata.
- SQL-authenticated accounts are stored using cryptographic password hashes.
- Protect system database files with strong access controls.
- Encrypt database storage and backups.
- Regularly audit SQL logins and privileged accounts.
- Prefer Windows Authentication whenever possible.
