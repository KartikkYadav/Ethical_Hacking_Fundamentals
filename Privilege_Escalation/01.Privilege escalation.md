Privilege escalation is the process where a user or attacker gains higher privileges than originally assigned on a system.

---

# 1️⃣ Vertical Privilege Escalation

Vertical privilege escalation occurs when a user increases their privilege level from a lower-privileged account to a higher-privileged account.

### Example

- Standard User ➜ Administrator (Windows)
- Normal User ➜ Root (Linux)

---

# 2️⃣ Horizontal Privilege Escalation

Horizontal privilege escalation occurs when a user accesses another user's resources while remaining at the same privilege level.

### Example

- User A accesses User B's private files.
- Accessing another user's session without becoming Administrator or Root.

# 🐧 Linux Privilege Escalation Techniques

### 1. Kernel Exploits

Exploit vulnerable Linux kernel versions to execute code with Root privileges.

Example tools

- Linux Exploit Suggester
- CVE Exploits
- Dirty COW
- Dirty Pipe

---

### 2. Misconfigurations

Exploit weak system configurations such as

- Weak file permissions
- Writable scripts
- Cron jobs
- PATH hijacking

---

### 3. Password Cracking

Obtain password hashes and crack them.

Common files

```bash
/etc/shadow
/etc/passwd
```

Tools

- John the Ripper
- Hashcat

---

### 4. Sudo Misconfiguration

Check allowed sudo commands.

```bash
sudo -l
```

Common abuse

- NOPASSWD
- GTFOBins
- PATH Hijacking

Reference

https://gtfobins.github.io/

---

### 5. Service Misconfiguration

Abuse services running as Root.

Examples

- Writable service files
- Weak permissions
- Vulnerable daemons

# 🪟 Windows Privilege Escalation Techniques

### 1. Kernel Exploits

Exploit vulnerable Windows kernel vulnerabilities.

Examples

- PrintNightmare
- HiveNightmare
- CVE-based exploits

---

### 2. Service Misconfiguration

Abuse Windows services.

Examples

- Unquoted Service Path
- Weak Service Permissions
- Writable Service Executables

Useful Commands

```powershell
sc qc <service>
accesschk.exe
```

---

### 3. Registry Exploits

Abuse registry-based service configurations.

Registry Location

```
HKLM\SYSTEM\CurrentControlSet\Services
```

---

### 4. Password Mining & Credential Dumping

Extract credentials from

- SAM
- LSA Secrets
- LSASS Memory

Tools

- Mimikatz
- secretsdump.py
- reg save

---

### 5. Scheduled Tasks

Enumerate scheduled tasks.

```powershell

Abuse writable scheduled task binaries.

---

### 6. Token Impersonation

Tools

- Incognito
- JuicyPotato
- PrintSpoofer

### 7. Startup Applications

Common Locations

```
C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup

%AppData%\Microsoft\Windows\Start Menu\Programs\Startup

# 🖥️ Enable WinRM on Windows 11

## Step 1 — Open PowerShell as Administrator

- Right-click **Start**
- Select **Windows PowerShell (Admin)** or **Windows Terminal (Admin)**

---

## Step 2 — Enable WinRM

```powershell
winrm quickconfig
```

This command will

- Start the WinRM service
- Set Startup Type to Automatic
- Create a WinRM listener
- Configure Windows Firewall
- Prompt for confirmation (Type **Y**)

---

## Step 3 — Verify WinRM Configuration

### Check WinRM Listeners

```bash
winrm enumerate winrm/config/listener
```

---

### Verify WinRM Port

```bash
nmap -v -sT -sV -sC -A -O -p 5985 <TARGET_IP>
```

Example

```bash
nmap -v -sT -sV -sC -A -O -p 5985 192.168.1.71
```

---

### Check WinRM Service Status

```powershell
Get-Service -Name WinRM
```

Expected Output

- **Status:** Running
- **Startup Type:** Automatic

---

### Connect Using Evil-WinRM

```bash
evil-winrm -i <TARGET_IP> -u <USERNAME> -p <PASSWORD>
```

Example

```bash
evil-winrm -i 192.168.1.71 -u rahul -p 123
```

---

# 🔒 (Optional) Configure WinRM for HTTPS

For encrypted WinRM communication over HTTPS:

- Create or install an SSL certificate.
- Configure a WinRM HTTPS listener.
- Open TCP port **5986** in Windows Firewall.
- Connect with Evil-WinRM using SSL:

```bash
evil-winrm -i <TARGET_IP> -S -u <USERNAME> -p <PASSWORD>
```
