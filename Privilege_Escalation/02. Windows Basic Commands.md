## Windows Basic Commands

Windows Command Prompt (`cmd.exe`) provides many built-in commands to view system information, navigate directories, manage files, environment variables, and perform administrative tasks.

---

## Display System Information

Displays detailed information about the system, including OS version, hardware specifications, network configuration, and uptime.

```bash
systeminfo
```

### Example Output

| Property | Value |
| --- | --- |
| Host Name | SRV01 |
| Operating System | Microsoft Windows Server 2025 Standard |
| Version | 10.0.26100 |
| Build Number | 26100 |

```
Hostname          : SRV01
Operating System  : Microsoft Windows Server 2025 Standard
Kernel Version    : 10.0
Build             : 26100 (Windows Server 2025 RTM)
Edition           : Standard
```

# Copying Files and Folders in Windows

---

# 🌟 Basic Copying with `COPY`

Displays help for the `COPY` command.

```bash
copy /?
```

Copy a file from one location to another.

```bash
copy /a c:\data\file.txt e:\data /v /y
```

### Options

- **/a** – Specifies the source file (ASCII mode).
- **/v** – Verifies the copied file for accuracy.
- **/y** – Suppresses confirmation prompts when overwriting existing files.

### Example

```bash
copy /a C:\inetpub\wwwroot\iisstart.htm C:\data\ /v /y
```

---

# 🚀 Advanced Copying with `XCOPY`

`XCOPY` is a more powerful alternative to `COPY` for copying directories, files, and subdirectories.

Display help.

```bash
xcopy /?
```

Copy all files with multiple options.

```bash
xcopy d:\data\*.* c:\test\ /a /d /p /s /v /w
```

## Options

- **/a** – Copies only files with the archive attribute.
- **/d** – Copies files modified after a specified date.
- **/p** – Prompts before copying each file.
- **/s** – Copies directories and subdirectories, except empty ones.
- **/v** – Verifies copied files.
- **/w** – Waits for confirmation before starting.

## Examples

Copy everything from **C:\inetpub** (including subdirectories).

```bash
xcopy C:\inetpub\ /s /y C:\data\
```

Verify copied files.

```bash
xcopy C:\inetpub\ /s /y C:\data\ /v
```

Copy only HTML files.

```bash
xcopy C:\inetpub\*.htm C:\htmlfiles\ /s /v /y
```

Copy EXE files from Downloads.

```bash
xcopy C:\Users\Rahul\Downloads\*.exe C:\NewFolder\ /s /v /y
```

Copy all files from Downloads.

```bash
xcopy C:\Users\Rahul\Downloads\*.* C:\NewFolder\ /s /v /y
```

# ⚡ Robust Copying with `ROBOCOPY`

`ROBOCOPY` (Robust File Copy) is a more powerful and reliable file-copying tool introduced in later Windows versions.

Display help.

```bash
robocopy /?
```

Copy all files and subdirectories.

```bash
robocopy C:\data C:\data1 /s
```

Copy a specific file.

```bash
robocopy C:\data C:\data2 iisstart.htm /s
```

## Copy Specific File Types

Copy JavaScript files.

```bash
robocopy C:\data C:\js *.js /s
```

Copy CSS files.

```bash
robocopy C:\data C:\css *.css /s
```

Copy HTML files.

```bash
robocopy C:\data C:\html *.html /s
```

## Copy Multiple File Types

```bash
robocopy C:\data\ C:\allfiles *.jpg *.png *.html *.txt /s
```

```bash
robocopy C:\data\ C:\all *.txt *.css *.js /s
```

Copies multiple specified file types from **C:\data** to the target directory.

### Additional Options

| Option | Description |
| --- | --- |
| `/r:1` | Retries once if a file fails to copy. |
| `/w:1` | Waits one second between retries. |
| `/ndl` | Prevents directories from being listed in the output. |
| `/xjd` | Excludes junction points (to avoid infinite loops). |

# 📁 Changing File and Folder Attributes Using `ATTRIB`

The **ATTRIB** command in Windows allows users to modify file and folder attributes such as **Hidden**, **System**, **Read-Only**, and **Archive**.

---

# 📖 Basic Syntax

Display attributes of a file.

```bash
attrib [filename]
```

Example:

```bash
attrib C:\swapfile.sys
```

Display help.

```bash
attrib /?
```

---

# 🏷️ Attribute Flags

| Flag | Description |
| --- | --- |
| `+H / -H` | Adds/Removes the Hidden attribute |
| `+S / -S` | Adds/Removes the System attribute |
| `+R / -R` | Adds/Removes the Read-Only attribute |
| `+A / -A` | Adds/Removes the Archive attribute |
| `/S` | Applies changes to all subdirectories |
| `/D` | Applies changes to directories |

---

# ✏️ Modifying File Attributes

### Make a file Hidden, System, and Read-Only

```bash
attrib +s +h +r test.txt
```

### Remove System, Hidden, and Read-Only attributes

```bash
attrib -s -h -r test.txt
```

### Apply attributes to all files in a directory

```bash
attrib +s +h +r C:\data\* /s
```

```bash
attrib +s +h +r C:\data\* /s /d
```

```bash
attrib +s +r +h E:\* /s /d
```

### Remove attributes from all files

```bash
attrib -s -h -r C:\data\* /s
```

```bash
attrib -s -h -r C:\data\* /s /d
```

---

# 💡 Common Use Cases

### Hide a sensitive file

```bash
attrib +h +s +r secret.docx
```

### Unhide all hidden files on a drive

```bash
attrib -h -s -r *.* /s /d
```

### Protect a file from accidental deletion

```bash
attrib +r important.docx
```

# 🔐 Managing File and Folder Permissions in Windows with `ICACLS`

`ICACLS` is the modern command-line tool in Windows for managing file and folder permissions. It replaces `CACLS` and provides more granular control over **Access Control Lists (ACLs)**.

---

# 📋 Permission Levels

| Permission | Description |
| --- | --- |
| **F** | Full Control |
| **M** | Modify |
| **RX** | Read & Execute |
| **R** | Read |
| **W** | Write |

---

# 🧾 Understanding the ACL Entries

| Entry | Meaning |
| --- | --- |
| `DESKTOP-SQPBPON\Armour:(F)` | User **Armour** has Full Control (**F**) |
| `BUILTIN\IIS_IUSRS:(RX)` | Group **IIS_IUSRS** has Read & Execute (**RX**) |
| `NT SERVICE\TrustedInstaller:(F)` | **TrustedInstaller** service has Full Control |
| `NT AUTHORITY\SYSTEM:(F)` | **SYSTEM** account has Full Control |
| `BUILTIN\Administrators:(F)` | **Administrators** group has Full Control |
| `BUILTIN\Users:(RX)` | All Users have Read & Execute |
| `DESKTOP-SQPBPON\Administrator:(F)` | Administrator has Full Control |
| `BUILTIN\IIS_IUSRS:(I)(RX)` | **(I)** indicates inherited permission |
| `NT SERVICE\TrustedInstaller:(I)(F)` | Inherited Full Control |
| `NT AUTHORITY\SYSTEM:(I)(F)` | Inherited Full Control |
| `BUILTIN\Administrators:(I)(F)` | Inherited Full Control |
| `BUILTIN\Users:(I)(RX)` | Inherited Read & Execute |
| `DESKTOP-SQPBPON\Administrator:(I)(F)` | Inherited Full Control |

---

# 🔍 Key Observations

- **(I)** indicates the permission is **Inherited** from the parent folder.
- Changes made to the **parent folder's permissions** automatically affect child files and folders unless inheritance is disabled.
- Typically, **Administrator**, **SYSTEM**, **TrustedInstaller**, and **Administrators** have **Full Control**.
- **Users** and **IIS_IUSRS** generally have **Read & Execute (RX)** permissions.

---

# 🔑 Basic Access Permissions

| Flag | Meaning |
| --- | --- |
| **D** | Delete Access |
| **F** | Full Access |
| **N** | No Access |
| **M** | Modify (Read, Write, Delete) |
| **RX** | Read & Execute |
| **R** | Read Only |
| **W** | Write Only |
