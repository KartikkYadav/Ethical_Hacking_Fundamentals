## Windows Shell

- The **Windows Shell** serves as the primary user interface for the Microsoft Windows operating system, enabling users to interact with the system through both graphical and command-line elements.
- It provides access to core functions like launching applications, managing files, and configuring system settings.

---

# Types of Windows Shells

## 1. Graphical Shell

- **Executable:** `explorer.exe`
- **Location:** `C:\Windows\explorer.exe`
- **Description:** The default Windows interface that provides the desktop environment, taskbar, Start Menu, and File Explorer.

---

## 2. Command-Line Interfaces (CLI)

### Command Prompt (`cmd.exe`)

- **Location:** `C:\Windows\System32\cmd.exe`
- **Purpose:** Traditional text-based command interpreter used for executing batch commands and administrative tasks.

### PowerShell (`powershell.exe`)

- **Location:** `C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe`
- **Purpose:** A modern, scriptable shell designed for automation and configuration management using cmdlets.

---

# Types of Commands in CLI

Commands in Windows command-line environments fall into two main categories:

## 1. Internal Commands

Built directly into the command interpreter. These do not rely on external executable files.

| Command | Description |
| --- | --- |
| `echo` | Displays messages or variable output. |
| `cls` | Clears the command window screen. |
| `dir` | Lists files and directories. |
| `del` | Deletes specified files. |

---

## 2. External Commands

Stored as independent executables on the system. They rely on external binaries and are often associated with installed software.

| Command | Description |
| --- | --- |
| `firefox.exe` | Launches Mozilla Firefox (if installed). |
| `notepad.exe` | Opens Notepad. |
| `ping.exe` | Tests network connectivity to a specific host. |

---

# Command Prompt vs PowerShell

| Feature | Command Prompt (`cmd.exe`) | PowerShell (`powershell.exe`) |
| --- | --- | --- |
| **Scripting Language** | Batch (`.bat`) scripts | PowerShell (`.ps1`) scripts |
| **Object-Oriented** | No (text-based output) | Yes (outputs objects) |
| **Functionality** | Basic command execution | Advanced automation and scripting |
| **Administrative Control** | Limited to surface-level commands | Granular control over system management |
