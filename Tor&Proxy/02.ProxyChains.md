# ProxyChains

ProxyChains is a **Linux/Unix tool that forces network connections made by an application to pass through one or more proxy servers**.

It works by intercepting network calls and redirecting them through:

- **SOCKS4**
- **SOCKS5**
- **HTTP proxies**

ProxyChains is commonly used in:

- Network security testing
- Anonymity setups
- Bypassing network restrictions

---

# How ProxyChains Works

ProxyChains uses **LD_PRELOAD** to hook networking functions (such as `connect()`) used by applications.

It intercepts these calls and routes them through configured proxy servers.

## Traffic Flow

Application → ProxyChains → Proxy Server(s) → Destination Server

This allows applications that **do not natively support proxies** to still use them.

---

# Basic Syntax

    proxychains command

## Examples

    proxychains curl http://example.com/

    proxychains nmap -sT target.com

---

# Configuration File

The configuration file defines the **proxies and chaining method**.

Common locations:

    /etc/proxychains.conf

or

    ~/.proxychains/proxychains.conf

---

# Proxy Configuration Example

    [ProxyList]

    socks5 127.0.0.1 9050
    http 192.168.1.10 8080

Example:

- Tor SOCKS proxy running on **127.0.0.1:9050**
- HTTP proxy running on **192.168.1.10:8080**

---

# Proxy Chain Modes

ProxyChains supports different chaining modes.

---

## 1. Dynamic Chain

    dynamic_chain

Features:

- Uses proxies in order
- Skips dead proxies automatically
- Most commonly used mode

---

## 2. Strict Chain

    strict_chain

Features:

- Uses proxies **in the exact order**
- If one proxy fails, the connection fails

---

## 3. Random Chain

    random_chain

Features:

- Selects proxies randomly
- Rotates proxies for each connection
- Useful for **IDS testing and load balancing**

---

# Running Applications Through ProxyChains

## Firefox

    proxychains firefox www.google.com

## Nmap

    proxychains nmap www.google.com

⚠ Some Nmap scan types may **not work properly through proxies**.

---

# Example Use Case

## Check IP without proxy

    curl icanhazip.com

## Check IP through proxy

    proxychains curl icanhazip.com

If the proxy is working correctly, the returned IP address will be **different from your real IP**.
